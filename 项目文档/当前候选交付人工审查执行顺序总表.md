# 当前候选交付人工审查执行顺序总表

状态：`review-execution-sequence-only`

本总表用于把当前候选交付的审查前包、Reviewer 工作台、输出接收门禁、离线汇总模板、会后汇总、冲突审查、风险台账和转换预检串成不可跳步的人工执行顺序。它不是 reviewer 输出，不是会后真实汇总，不是 live submission，不是 release approval，不是 Phase 29 exit，不是 Phase 30 entry，也不启用 runtime execution 或 third-party execution。

本总表的材料、命令和上下游关系由 `当前候选交付人工审查链完整性索引.md` 做断链检查。

正式人工审查会议前，可以使用 `当前候选交付人工审查演练Dry-run记录模板.md` 做一次命令和材料 dry-run；该 dry-run 不产生 reviewer evidence。

## 二、不可跳步原则

- 任何步骤通过，都不能解释为 release approval。
- `npm.cmd run check` 通过，只表示结构和红线检查通过。
- 真实 reviewer 输出必须在线下产生，并先通过接收门禁。
- 接收门禁未通过时，不得进入离线汇总模板。
- 离线汇总未完成分流时，不得进入会后证据汇总。
- 存在冲突时，必须进入人工证据冲突审查。
- 存在 `accepted-with-risk`、`deferred-with-owner` 或 `blocked` 时，必须进入风险处置后续台账。
- 未完成冲突审查、风险后续和维护者显式转换授权时，不得进入 live submission 创建。

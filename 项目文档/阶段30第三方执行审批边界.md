# 阶段30第三方执行审批边界
更新日期：2026-07-06

Baseline: `2.0.26 / phase30-third-party-execution-approval-boundary`

Phase 30 third-party execution approval boundary: active

本边界包用于定义真实第三方插件执行前必须具备的插件级审批记录、证据输入和反级联规则。它只是 third-party-execution-approval-boundary-only package，不是 runtime go，不是 third-party execution approval，也不允许执行真实插件代码。

- `thirdPartyExecutionDecision=not-recorded`
- `runtimeGoNoGoDecision=not-recorded`
- `runtimeGoNoGoDecisionPacket=active`
- `perPluginApprovalRequired=true`
- `blanketPluginApprovalAllowed=false`
- `pluginScopedApprovalRequired=true`

- `phase29ExitReady=false`

This package is not runtime execution approval.

This package is not third-party execution approval.

This package does not execute real third-party plugin code.

## 2. 允许决策结论

第三方执行审批边界只允许为单个插件记录以下结论：
- `approve`
- `deny`
- `hold`
- `blocked`

当前默认结论必须保持 `blocked` / `no-execution-blocked`，因为 runtime owner Go/No-Go 尚未记录，插件级审批尚未记录，人工证据仍 pending。

## 3. 审批输入

第三方执行审批只能在以下输入全部可审阅后记录：
- `阶段30运行时GoNoGo决策包.md`
- `阶段30入口准备红线.md`
- `阶段30运行时沙箱验收框架.md`
- `阶段30密钥边界计划.md`
- `阶段30审计持久化演练.md`
- `阶段30人工复核证据包.md`
- `阶段30风险处置后续台账.md`
- 目标插件 manifest、permission profile、secret access request、private memory access request、audit plan、rollback strategy 和 plugin owner 决策记录

如果 runtime Go/No-Go 未被独立记录为有效 go，本边界只能保持 `thirdPartyExecutionApprovalStatus=no-execution-blocked`。

## 4. 必需字段

未来单插件第三方执行审批记录必须包含以下字段：
- `thirdPartyExecutionDecision`
- `thirdPartyExecutionOwner`
- `thirdPartyExecutionReviewedAt`
- `thirdPartyExecutionEvidenceRef`
- `thirdPartyExecutionDecisionReason`
- `thirdPartyExecutionResidualRisk`
- `thirdPartyExecutionRollbackCondition`
- `thirdPartyExecutionScope`
- `thirdPartyExecutionExclusions`
- `thirdPartyExecutionFollowupOwner`
- `pluginId`
- `pluginManifestRef`
- `pluginPermissionProfile`
- `pluginSecretAccessDecision`
- `pluginPrivateMemoryAccessDecision`
- `pluginAuditPlanRef`

缺少任一字段时，不允许把记录解释为第三方执行批准。

## 6. 与 runtime Go/No-Go 的关系

本边界必须继承以下规则：
- `gateCascadeAllowed=false`
- `runtime-go-to-third-party-execution` remains blocked
- `third-party-approval-to-runtime-execution` remains blocked
- runtime go/no-go 不自动触发 third-party execution
- third-party execution approval 不自动触发 runtime execution
- third-party execution approval 不自动授予 secret access
- third-party execution approval 不自动授予 private memory access
- third-party execution approval 必须绑定单个 `pluginId`

即使未来 `runtimeGoNoGoDecision=go`，也只能表示 runtime owner 对运行时启用作出独立记录；它不能自动设置 `thirdPartyExecution=true`，不能执行真实第三方插件，也不能绕过插件级审批边界。

## 7. 禁止事项

- 不伪造 plugin owner 决策。
- 不把 runtime go/no-go 当作 third-party execution approval。
- 不把 release approval、Phase 29 exit 或 Phase 30 entry 当作第三方执行批准。
- 不把 sandbox dry-run 当作真实插件执行。
- 不把 secret boundary review 当作 secret access approval。
- 不把 private memory boundary review 当作 private memory access approval。
- 不设置 `runtimeExecution=true`。
- 不设置 `thirdPartyExecution=true`。
- 不执行真实第三方插件代码。
- 不持久化真实 runtime state。

## 8. 完成判定

本边界包完成时，只表示项目具备插件级第三方执行审批记录的结构、字段和反级联边界。它不代表 runtime execution 已经启用，也不代表 third-party execution 已经通过。

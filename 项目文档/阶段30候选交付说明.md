## 3. 当前不可交付事项

以下事项不得作为本候选交付的一部分宣布完成：

- release approval；
- Phase 29 exit；
- Phase 30 implementation entry；
- runtime execution；
- third-party execution；
- secret access；
- private memory access by plugins；
- persisted runtime mutation；
- automatic blocker clearance；
- automatic human signoff。

## 4. 必须人工确认的事项

### 4.1 Git 目录形态

当前 `项目工程` 是实际 git 仓库根目录，人工已经确认接受当前扁平化目录形态，详见 `仓库扁平化确认记录.md`。

git 索引仍保留旧的一层目录路径。候选交付前仍必须由维护者明确授权路径迁移提交，把旧索引路径替换为当前仓库根文件。

在索引迁移提交完成前，不制作正式 release commit。

### 4.2 Phase 30 人工证据

10 个 evidence slots 仍为 `pending` 和 `evidenceRef=missing`。候选交付只能说明证据槽位已经建模和暴露，不能说明人工签核已经完成。

### 4.3 Release / Runtime 分离

即使未来 release approval 完成，也不能自动启用 runtime execution。runtime owner go/no-go 必须独立完成。

## 5. 候选交付结论

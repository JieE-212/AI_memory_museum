# 第三阶段数据库设计

目标：把浏览器 `localStorage` 中的展品迁移到后端 SQLite，保留第二阶段已经稳定的结构化 JSON 字段。

当前状态：已完成第三阶段并完成多轮收尾优化。实现文件为 `项目工程/database.js`，数据库默认路径为 `项目工程/data/memory-museum.sqlite`，可通过 `DB_PATH` 环境变量覆盖。运行环境要求 Node.js 24 或更高版本，因为当前实现使用 Node 内置 `node:sqlite`。

## 1. 数据库选择

第一版使用 SQLite。

原因：

- 单文件数据库，适合本地 Agent 原型
- 不需要额外数据库服务
- 后续可以平滑迁移到 PostgreSQL

## 2. 表结构

### halls

展厅表。

```sql
CREATE TABLE halls (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  description TEXT NOT NULL DEFAULT ''
);
```

初始化展厅：

- youth：青春展厅
- friends：朋友展厅
- family：家庭展厅
- low：低谷展厅
- strange：奇怪瞬间展厅
- daily：日常展厅

### memories

展品表。

```sql
CREATE TABLE memories (
  id TEXT PRIMARY KEY,
  schema_version INTEGER NOT NULL DEFAULT 1,
  title TEXT NOT NULL,
  hall_id TEXT NOT NULL,
  source_type TEXT NOT NULL,
  raw_content TEXT NOT NULL,
  exhibit_text TEXT NOT NULL,
  memory_date TEXT NOT NULL DEFAULT '',
  location TEXT NOT NULL DEFAULT '',
  emotion_intensity INTEGER NOT NULL DEFAULT 3,
  importance INTEGER NOT NULL DEFAULT 1,
  favorite INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL DEFAULT '',
  FOREIGN KEY (hall_id) REFERENCES halls(id)
);
```

### memory_people

人物表。

```sql
CREATE TABLE memory_people (
  memory_id TEXT NOT NULL,
  name TEXT NOT NULL,
  PRIMARY KEY (memory_id, name),
  FOREIGN KEY (memory_id) REFERENCES memories(id) ON DELETE CASCADE
);
```

### memory_tags

标签表。

```sql
CREATE TABLE memory_tags (
  memory_id TEXT NOT NULL,
  tag TEXT NOT NULL,
  PRIMARY KEY (memory_id, tag),
  FOREIGN KEY (memory_id) REFERENCES memories(id) ON DELETE CASCADE
);
```

### memory_emotions

情绪表。

```sql
CREATE TABLE memory_emotions (
  memory_id TEXT NOT NULL,
  emotion TEXT NOT NULL,
  PRIMARY KEY (memory_id, emotion),
  FOREIGN KEY (memory_id) REFERENCES memories(id) ON DELETE CASCADE
);
```

## 3. 建议索引

```sql
CREATE INDEX idx_memories_hall_id ON memories(hall_id);
CREATE INDEX idx_memories_created_at ON memories(created_at);
CREATE INDEX idx_memories_importance ON memories(importance);
CREATE INDEX idx_memories_favorite ON memories(favorite);
CREATE INDEX idx_memory_tags_tag ON memory_tags(tag);
CREATE INDEX idx_memory_emotions_emotion ON memory_emotions(emotion);
CREATE INDEX idx_memory_people_name ON memory_people(name);
```

## 5. localStorage 迁移策略

第一版做手动迁移，当前已经实现：

1. 前端读取当前 `localStorage`。
2. 用户点击“迁移到数据库”。
3. 前端调用 `POST /api/memories/import`。
4. 后端规范化并写入 SQLite。
5. 前端重新从 `GET /api/memories` 拉取数据。
6. 迁移成功后保留 localStorage 作为本地备份，不自动删除。

迭代补充：

- 如果 SQLite 为空但浏览器本地有展品，页面先显示本地备份并提示迁移。
- 如果用户没有手动迁移就直接保存新展品，前端会先自动迁移当前本地备份，再保存新展品。
- 如果 SQLite 和本地备份都有数据但 ID 不完全一致，前端会合并显示两边数据，并提示用户补写入 SQLite。
- `POST /api/memories/import` 使用整批事务，避免导入过程中出现半成功状态。

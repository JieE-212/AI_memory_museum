## 2. 敏感边界分类

| Boundary | 内容 | 默认策略 |
| --- | --- | --- |
| plugin-secret-boundary | 插件 API key、签名私钥、安装密钥、第三方服务凭据 | deny |
| app-secret-boundary | `.env`、AI API key、部署 token、运维凭据 | deny |
| sqlite-memory-boundary | `memory-museum.sqlite` 中的原始记忆、人物、地点、附件线索、Agent run | deny |
| redacted-export-boundary | 脱敏导出包和演示包 | allow-only-if-explicit-fixture |
| audit-metadata-boundary | 不含私人内容的审计 ID、fixture ID、dry-run 结果 | allow-dry-run |
| runtime-state-boundary | runtime 状态、release 状态、Phase 30 entry 状态 | deny-write |

任何默认策略为 `deny` 或 `deny-write` 的边界都不能被 fixture pass 自动覆盖。

## 3. 拒绝规则

Phase 30 第一轮必须固定以下拒绝规则：

- deny-env-secret-read
- deny-plugin-secret-read
- deny-signing-private-key-read
- deny-ai-api-key-read
- deny-sqlite-raw-memory-read
- deny-sqlite-agent-run-read
- deny-attachment-metadata-read
- deny-host-filesystem-secret-scan
- deny-runtime-state-write
- deny-release-state-write
- deny-phase30-entry-state-write

拒绝规则命中时，结果必须是 `blocked`，输出只能进入 dry-run evidence，不得进入可执行运行时。

## 4. 允许的 dry-run 信息

以下信息可以作为 dry-run evidence 出现：

- `fixtureId`
- `boundaryId`
- `policy`
- `requestedCapability`
- `decision`
- `blockedReason`
- `manualReviewRequired`
- `redactionApplied`
- `runtimeExecution=false`
- `thirdPartyExecution=false`
- `releaseReady=false`
- `phase30EntryReady=false`

dry-run evidence 不得包含：

- 原始记忆正文
- 展品说明全文
- 人物、地点、联系方式明文
- 附件原始内容
- AI API key 或任何 secret value
- SQLite 文件路径以外的可读取数据内容
- 签名私钥或安装密钥

## 5. Fixture 边界用例

| Fixture | 请求 | 预期结果 |
| --- | --- | --- |
| deny-env-secret-read-fixture | 读取 `.env` 或 AI key | blocked |
| deny-plugin-secret-read-fixture | 读取插件密钥 | blocked |
| deny-private-key-read-fixture | 读取签名私钥 | blocked |
| deny-sqlite-raw-memory-read-fixture | 读取原始记忆正文 | blocked |
| deny-agent-run-read-fixture | 读取 Agent run 明细 | blocked |
| deny-attachment-metadata-read-fixture | 读取附件原始元数据 | blocked |
| allow-audit-id-dry-run-fixture | 读取 fixture ID 和审计 ID | pass-dry-run |
| allow-redacted-export-fixture | 读取显式脱敏演示包 | pass-dry-run |

所有 fixture 都是描述性测试用例，不包含真实第三方插件代码。

## 6. 人工复核要求

以下情况必须人工复核：

- fixture 请求任何 secret 或 SQLite 私人记忆字段。
- fixture 请求从 redacted-export-boundary 升级到 sqlite-memory-boundary。
- fixture 试图把 dry-run evidence 写入 runtime state。
- fixture 试图把 blocked 结果标记为 pass。
- fixture 试图把 pass-dry-run 推导为 release approval。

人工复核只能记录 reviewer、reason、decision 和 evidenceRef，不能写入 secret value 或私人记忆内容。

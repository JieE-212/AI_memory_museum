## 3. Scope Lock

Phase 30 第一轮只覆盖 controlled runtime enablement readiness。

包含范围：

- Phase 30 入口基线
- sandbox acceptance harness 设计
- secret boundary 设计
- audit persistence dry-run 设计
- runtime go/no-go board 设计
- 人工签核和风险登记结构

排除范围：

- 真实第三方插件执行
- 插件网络访问
- 插件密钥读取
- 私人记忆数据直通插件
- release approval 自动化
- blocker 自动清除
- redline 自动解决
- 不得设置 `releaseReady=true`
- 不得设置 `phase29ExitReady=true`
- 不得设置 `phase30EntryReady=true`
- 不得启用 `runtimeExecution=true`
- 不得启用 `thirdPartyExecution=true`

## 7. 下一版验收标准

`2.0.1 / phase30-runtime-sandbox-acceptance-harness` 应该满足：

- 使用 fixture 插件，不执行真实第三方代码。
- 覆盖允许、拒绝、超时、输出隔离和异常退出场景。
- 不开放网络访问。
- 不读取插件密钥。
- 不读取 SQLite 私人记忆数据。
- 不写入运行时状态。
- 输出只进入 quarantine 或 dry-run evidence。
- 任何 pass 结果都不能推出 `runtimeExecution=true` 或 `thirdPartyExecution=true`。

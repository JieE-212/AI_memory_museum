# 仓库索引迁移预检清单

更新日期：2026-07-05

Preflight: `git-flattening-preflight`

## 1. 当前决策

已接受当前扁平化形态：`项目工程` 作为当前 git 仓库根目录继续推进。

当前状态仍是预检，不是索引迁移执行：

- `gitPathShape=accepted-flattened-project-root`
- `gitIndexMigration=committed-flattened-project-root`
- `releaseCommit=blocked-until-human-release-approval`
- `indexMutation=committed-path-migration`

## 2. 预检观察

当前 git index 仍保留旧的一层目录路径，因此 `git status` 会同时出现：

- 旧索引路径删除：`项目工程/...`、`项目文档/...`
- 当前仓库根文件未跟踪：`README.md`、`app.js`、`server.js`、`scripts/...`、`src/...`
- 本地运行产物被忽略：`data/`、`*.log`、`operations-debug.json`、`operations-export-debug.json`

这表示路径形态尚未完成迁移提交，不表示业务代码被确认删除。

## 3. 本轮不会执行的动作

本轮预检明确不执行：

- `git add -A`
- `git commit`
- `git reset --hard`
- `git checkout --`
- `git clean`
- release approval
- Phase 29 exit
- Phase 30 entry
- runtime execution enablement
- third-party execution enablement

## 4. 后续授权后建议步骤

需要维护者明确授权后，才能进入真正的索引迁移提交准备。建议顺序：

1. 再次运行 `node scripts/git-flattening-preflight.js`。
2. 审阅 `git status --short` 和 `git diff --stat`。
3. 只做路径迁移相关的 index 更新，避免混入 release、runtime 或 Phase 30 entry 状态变化。
4. 迁移后重新运行 `npm.cmd run check`。
5. 只有在迁移提交完成后，再讨论正式 release commit 或 Phase 29 exit。

## 5. 当前结论

仓库索引迁移提交已完成。当前仍不是 release commit，release approval 继续阻断。

# 仓库扁平化确认记录

更新日期：2026-07-05

Decision: `accepted-flattened-project-root`

人工决策结果：接受当前 `项目工程` 作为 git 仓库根目录的扁平化形态。

## 1. 当前事实

- `项目工程` 是当前实际 git 仓库根目录。
- 项目运行、检查、文档和脚本均已围绕该仓库根稳定工作。
- git 索引仍保留旧的一层路径，例如旧的 `项目工程/...` 和 `项目文档/...` 路径。
- 当前 git status 会表现为旧路径 deleted、新根路径 untracked。

## 2. 已确认方向

接受当前目录扁平化。后续路径迁移应把旧索引路径替换为当前仓库根文件。

这只是目录形态决策，不代表已经执行索引迁移，不代表正式 release，也不改变 Phase 29 / Phase 30 guardrail。

## 3. 迁移前保护线

在维护者明确授权迁移提交前，仍禁止：

- `git reset --hard`
- `git checkout --`
- `git clean`
- `git add -A`
- 自动提交 release commit

## 4. 建议的非破坏性迁移准备

维护者后续可以先审阅：

```powershell
git status --short
git diff --stat
git ls-files
```

确认无误后，再由维护者显式授权路径迁移提交。迁移提交应只表达目录扁平化，不应混入 release approval、Phase 30 entry 或 runtime enablement。

## 5. 当前结论

`gitPathShape=accepted-flattened-project-root`

`gitIndexMigration=committed-flattened-project-root`

`indexMutation=committed-path-migration`

`releaseCommit=blocked-until-human-release-approval`

# V5–V7.2 扩展路线：让记忆从保存走向重逢与讲述

> 状态（2026-07-18）：V5、V5.1、V6、V6.1、V7 与 V7.1 已发布；V7.2 已在本地完成为 `V7.2.0 / schema 10` 基线，真实 HTTP smoke 固定为 186 条断言，尚未提交发布。线上公开 Demo 仍为 `V7.1.0 / schema 9`，不能把本地能力描述成已经上线。

## 不变的产品主轴

时屿仍是本地优先的私人记忆策展工具，而不是通用笔记、待办或社交相册。所有扩展都服务于同一条路径：记录 → 整理 → 关联 → 重现 → 讲述 → 长期保存。

- 顶层导航始终只有“展品库、记录记忆、讲解与回顾、数据与项目”四项。
- 新能力默认折叠或懒加载，结果进入独立阅读层；默认页面保持清爽。
- 原文、图片、录音和人物关系都以用户确认为最终边界，模型不得补写事实。
- 本地规则、本地存储和可解释引用是默认路径；任何外部 AI 请求都必须由用户主动触发。
- 每个阶段都必须进入完整备份、脱敏边界、清空流程、Demo 保护和自动门禁。

## 基础护栏（本地完成）

在 V5 业务功能前补齐正式迁移账本，并把后续功能拆成独立数据库、API、前端和检查模块。归档升级采用兼容 V1 的 V2 section 清单；未知 required section 必须整包拒绝，避免出现“恢复成功但新数据丢失”。

## V5：AI 主题展览（本地完成）

### 目标

从用户明确选择的 2–12 件展品生成带开场、章节、图片和引用依据的主题展览。预览不落库，只有用户确认后才保存。

### 产品边界

- 入口位于“回顾”，不增加顶层导航或第五个回顾 Tab。
- AI 只接收当前选择的展品证据；未配置模型时由本地可解释规则生成。
- 每个策展判断都保留原文 quote、起止偏移和来源展品。
- 公开 Demo 可以预览，但不能保存或删除展览。

### 阶段完成条件

- 迁移、CRUD、引用失效、Demo 保护、备份与恢复均有独立门禁。
- 320px、390px、650px、651px 和桌面布局通过目视验收。

## V5.1：记忆回访（本地完成）

### 目标

提供“往年今日、很久没见、随机漫游”三种回访方式，但默认一次只呈现一件记忆。

### 产品边界

- “往年今日”只使用用户明确填写的记忆日期，不把保存日期冒充周年。
- 客户端显式传入本地日期和时区，保证闰日与跨时区行为可测试。
- 浏览状态只用于排序和避免重复推荐，不生成心理判断。

## V6：语义线索检索与人物档案（本地完成）

### 目标

在现有单一搜索框中加入 FTS、实体和可解释线索召回，并提供人物、地点、主题档案。

### 产品边界

- 未接入真实 embedding 时，页面明确称为“语义线索检索”，不宣称向量搜索。
- 旧人物、地点字段继续兼容；别名、合并和敏感关系必须由用户确认。
- 搜索结果展示匹配字段、置信度和原因，不只返回排序结果。

## V6.1：语音记忆（本地完成）

### 目标

保存原始录音、人工确认转写和可回放音频，让记忆从图文扩展为多模态。

### 产品边界

- 音频使用独立存储与数据表，不泛化图片资产模型。
- 首版支持浏览器录音和文件上传；转写以手动或明确离线引擎为主。
- 不把浏览器 SpeechRecognition 冒充本地转写。
- 公开 Demo 保持麦克风关闭并阻止全部音频写入。

## V7：时光胶囊与加密展览分享（本地完成）

### 目标

允许用户把记忆封存到未来，并把已确认展览导出为单文件、口令加密、可离线阅读的分享包。

### 产品边界

- 胶囊只来源于状态为 `published`、无需复核的展览；来源展览只是创建依据，胶囊保存独立安全快照。
- 外壳、内容快照和 display WebP 关联分表保存。未到期内容接口返回 `423`、稳定错误码和公开外壳，不读取内容快照，也不把正文、图片、成员列表或内部 ID 下发到客户端。
- 本地开启日期是仪式性门槛，不宣传为不可破解的密码学时间锁。
- 分享材料默认全部不选择，只允许来源展览内用户显式勾选的 display WebP 和已确认文字稿。素材读取完成后才出现口令步骤，此后不再发送网络请求。
- 浏览器使用 PBKDF2-SHA-256（310,000 次）+ AES-256-GCM，并为每个文件生成随机 16-byte salt 和 12-byte IV；公开外壳作为 AAD 的一部分参与完整性校验。
- 口令只存在于当前页面内存，不发送、不持久化，也不写入文件；下载结果是自包含、断网可读的 HTML。
- 分享包物理排除原图、EXIF/GPS、草稿文字稿、Agent 日志、内部 ID、未选内容和原始音频；首版只携带已确认声音文字稿。
- 当前不做公网私密链接、账号、撤回、远程失效或口令找回机制。

### 阶段完成条件

- schema 8→9 迁移只新增空胶囊结构，不改变旧馆藏、图片、展览、回访、线索和语音计数。
- 胶囊服务、数据库、API、前端、离线素材、备份恢复与真实 HTTP 流程均进入自动门禁。
- 真实浏览器覆盖桌面与移动端的空状态、未到期、到期、素材选择、错/对口令、断网阅读、焦点恢复和 Demo 状态。
- 完整 `.time-isle` 往返保留胶囊；脱敏归档物理移除胶囊标题、日期、来源、快照、图片关联和内部 ID。

## V7.1：隐私安全 PWA 外壳（已发布）

V7.1 在不改变私人记忆策展主体方向、数据结构和四项主导航的前提下，为支持环境提供可安装 PWA 外壳。安装入口只在浏览器确认可安装时渐进出现；iOS/iPadOS 提供手动添加到主屏幕说明，不支持或不满足安全来源条件时保持静默。

Service Worker 只预缓存离线边界页、对应样式和公开品牌图标，不缓存首页、API、图片、声音、归档或任何私人馆藏内容。断网时页面不会用空数据伪装馆藏，也不是完整离线馆藏模式，只展示隐私边界与重新连接入口；所有图片变体继续使用私有、禁止缓存策略。

## V7.2：记忆年轮、馆藏体检与只读验真（本地完成）

### 目标

让用户能回看一件记忆如何被修改、把旧版安全恢复为新的当前版本，并在迁移前主动核对本机馆藏和 `.time-isle` 备份；功能仍收纳在“讲解与回顾”和“数据与项目”内，不增加第五项导航。

### 产品与安全边界

- 每个有效版本保存规范快照与 SHA-256，后一版的 `parentSha256` 必须等于前一版的 `snapshotSha256`。首版类型只允许 `baseline / created / imported`，后续只允许 `edited / restored`；它用于发现断链和内容篡改，不冒充外部签名或不可篡改审计系统。
- 内容没有变化的保存保持 no-op：不增加修订、不更新时间，也不改变 ETag。编辑和旧版恢复使用 `If-Match` 版本条件；缺少条件返回 `428`，过期条件返回 `412`，避免旧页面覆盖新修改。
- 恢复旧版不是回滚或删除历史，而是把所选快照复制为新的 `restored` head，并保留恢复来源和原有父链。
- “馆藏体检”只读核对 SQLite 快检、外键、迁移账本、FTS 成员、图片/声音记录与文件完整性以及待复核内容；不会自动删除、修复或重写馆藏，结果只保留在当前进程内。
- “选择 `.time-isle` 验真”复用正式恢复前的严格解包与全量校验，只返回可恢复性、schema、条目和计数摘要，不写业务数据库或最终媒体目录；未来 schema 在写入前拒绝。
- Schema 10 的完整归档必须携带整条修订链，并对修订 section 设置精确 `20 MiB` UTF-8 JSON 预算和 10,000 条硬上限。Schema 9 及更早归档仍可恢复，导入的展品从一条 `imported` 基线开始新的年轮。
- 脱敏导出中的年轮只保留展品数、修订数和固定说明，物理排除旧正文、结构化字段、编辑备注、哈希、精确时间和内部 ID；未知字段与越级 section 一律拒绝。
- 旧 JSON 兼容导入使用独立 `64 MiB` 请求预算，避免被普通 `2 MiB` JSON 上限截断；`.time-isle` 仍走流式、分项和总展开上限，不把整包读入内存。
- 启动维护只清理超过一小时、名称符合契约的 `.exports / .restore / .inspect` 崩溃遗留 staging；新任务、无关目录和符号链接边界不会被跟随或误删。

### 阶段完成条件

- Schema 9→10 迁移只新增 `memory_revisions` 及索引，既有展品、图片、声音、展览、胶囊与检索数据保持不变。
- 年轮数据库/API/界面、并发冲突、no-op、旧版恢复、归档往返、严格脱敏、体检取消、只读验真、未来 schema 拒绝和 staging 崩溃清扫均进入独立门禁。
- 完整检查、本地真实浏览器桌面与窄屏验收、隔离迁移副本核对和 186 条真实 HTTP 断言均已通过；V7.2 已具备提交条件，但尚未执行提交、推送与线上部署。

## 每阶段循环

1. 写明目标、非目标、数据边界和页面入口。
2. 先完成纯函数与数据库门禁，再接 API 和界面。
3. 跑独立检查与完整 `npm run check`。
4. 用真实浏览器检查窄屏、断点、焦点、空状态、错误状态和 Demo 状态。
5. 删除临时数据、调试入口、测试钩子和未使用代码，再进入下一阶段。

上述循环已经用于 V5–V7.2 的完整推进。当前文件记录本地已完成的 `V7.2.0 / schema 10` 基线；提交、双远端推送与部署后验收尚未执行，因此公开发布基线仍是 `V7.1.0 / schema 9`。

# 当前候选交付负向用例防误用检查

状态：`negative-misuse-check-only`

本检查用于把当前候选交付中最容易被误读的材料逐项列成负向用例，防止模板、dry-run、会议记录、检查命令、转换预检、准备信封或决策包被误解释为 reviewer evidence、receipt acceptance、维护者授权、live submission、release approval、Phase 29 exit、Phase 30 entry、runtime execution 或 third-party execution。它只是 misuse-guard，不是 reviewer 输出，不是授权，不是审批，也不会创建任何 live submission。

## 三、必须拒绝的误用结论

以下结论必须被视为无效：

- “dry-run 通过，所以 reviewer evidence 已完成。”
- “会议记录存在，所以 receipt acceptance 已完成。”
- “`npm.cmd run check` 通过，所以 releaseReady 可以改成 true。”
- “conversion preflight 通过，所以可以创建 `data/phase30-human-evidence-submission.json`。”
- “维护者转换授权前置模板存在，所以维护者已经授权。”
- “最终锁定索引存在，所以 reviewer evidence 或 release approval 已完成。”
- “Reviewer 已收到材料，所以 reviewer evidence 或 receipt acceptance 已完成。”
- “材料签收回执存在，所以 Reviewer 输出接收门禁已经通过。”
- “reviewer 已开始审查，所以 reviewer evidence 已产生。”
- “澄清问题已解决，所以 reviewer evidence 已产生。”
- “Reviewer 已补交，所以 Reviewer 输出接收门禁已经通过。”
- “接收尝试记录存在，所以 Reviewer 输出接收门禁已经通过。”
- “字段缺口定位矩阵里标为 present 或 non-blocking-note，所以 evidence acceptance 或 release approval 已完成。”
- “字段修复责任分派单里标为 assigned 或 fix-submitted，所以字段已经修复或 Reviewer 输出接收门禁已经通过。”
- “字段修复回执与重试准备记录里标为 received 或 ready-for-receipt-attempt，所以字段已经修复或 Reviewer 输出接收门禁已经通过。”

- “对账失败回流处置单里标为 routed、returned-to-owner 或 ready-for-next-retry-batch，所以字段已经修复或 Reviewer 输出接收门禁已经通过。”

- “Reviewer 输出再对账执行记录里标为 executed，所以 Reviewer 输出接收门禁已经通过。”
- “`当前候选交付Reviewer输出再对账执行记录.md` 存在，所以 reviewer evidence 或 receipt acceptance 已完成。”
- “`当前候选交付Reviewer输出再对账结果接收记录.md` 里标为 matched 或 received，所以 Reviewer 输出接收门禁已经通过。”
- “`当前候选交付Reviewer输出再对账结果复核记录.md` 里标为 review-confirmed 或 ready-for-next-disposition，所以 Reviewer 输出接收门禁已经通过。”
- “发布决策准备信封存在，所以 release approval 已完成。”
- “发布审批决策包、Phase 29 exit 决策包、Phase 30 entry 决策包任一存在，所以对应 gate 已通过。”
- “运行时 Go/No-Go 决策包存在，所以 runtime execution 可以启用。”
- “第三方执行审批边界存在，所以 third-party execution 可以启用。”

## 四、真实转换仍需满足

创建 `data/phase30-human-evidence-submission.json` 前，仍必须同时满足：

- 10 个 evidence slots 都有真实 reviewer 输出。
- 每个 slot 都有 `reviewer`、`reviewedAt`、`evidenceRef`、`disposition`、`decisionReason` 和 `residualRisk`。
- 如 disposition 为 `accepted-with-risk`、`deferred-with-owner` 或 `blocked`，必须有 `followUpOwner`、`recoveryCondition` 和 `targetReviewDate`。
- 无 `REPLACE_WITH_...`、`YYYY-MM-DD`、`pending`、`missing`、`unassigned` 或空字段。
- reviewer 输出已通过 `当前候选交付Reviewer输出接收门禁.md`。
- 已按 `当前候选交付Reviewer输出离线汇总模板索引.md` 完成离线汇总分流。
- 冲突已通过 `阶段30人工证据冲突审查包.md` 解决。
- 风险后续已通过 `阶段30风险处置后续台账.md` 收口。
- `phase30:live-submission-conversion-preflight` 只作为预检通过，不创建文件。
- 维护者在线下明确给出 `maintainerConversionAuthorization=true` 和 `authorizationReason`。

## 六、禁止事项

不得：

- 不得创建 `data/phase30-human-evidence-submission.json`。
- 把模板、dry-run、会议记录、会后汇总草稿、转换预检、准备信封或决策包当作真实 reviewer evidence。
- 把 receipt-gate 模板当作 receipt acceptance。
- 把 authorization-template 当作维护者授权。
- 把 final package lock index 当作 reviewer evidence、receipt acceptance、维护者授权或 release approval。
- 把 reviewer handoff execution checklist 当作 reviewer evidence、receipt acceptance 或 evidence acceptance。
- 把 reviewer handoff receipt template 当作 Reviewer 输出接收门禁通过或 evidence acceptance。
- 把 reviewer review start confirmation template 当作 reviewer evidence 或 Reviewer 输出接收门禁通过。
- 把 reviewer in-review blocker clarification ledger 当作 reviewer evidence 或 Reviewer 输出接收门禁通过。
- 把 reviewer output return resubmission closure record 当作 reviewer evidence 或 Reviewer 输出接收门禁通过。
- 把 reviewer output receipt attempt log 当作 reviewer evidence 或 Reviewer 输出接收门禁通过。
- 把 reviewer output field gap matrix 当作 reviewer output、evidence acceptance 或 release approval。
- 把 reviewer output field fix assignment 当作 fixed field proof、receipt acceptance 或 release approval。
- 把 reviewer output field fix receipt retry readiness record 当作 fixed field proof、receipt acceptance 或 release approval。
- 把 reviewer output retry batch register 当作 reviewer evidence、fixed field proof、receipt acceptance 或 release approval。
- 把 reviewer output retry batch attempt reconciliation 当作 reviewer evidence、fixed field proof、receipt acceptance 或 release approval。
- 把 readiness-envelope 或 decision-packet 当作 release approval。
- 不得设置 `releaseReady=true`。
- 不得设置 `phase29ExitReady=true`。
- 不得设置 `phase30EntryReady=true`。
- 不得启用 `runtimeExecution`。
- 不得启用 `thirdPartyExecution`。

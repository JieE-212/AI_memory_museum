# 产品说明：V8.0 不确定时间线与来源校准版

## 一句话定位

时屿（TIME ISLE）是一款本地优先的 AI 私人记忆策展工具：它保存原文、照片、声音与用户确认过的证据，把散落片段组织成可以检索、回访和讲述的展览；V8 进一步允许用户把互相冲突或尚不完整的日期放到同一张来源台上核对，并亲自选择一个日期、一个范围、保留多种记录或继续不确定，而不是让系统改写原文、覆盖展品日期或裁决唯一真相。

## 版本状态

本文对应当前本地代码基线 V8.0.0（schema 12）。V8 尚未推送、尚未部署；截至 2026-07-18，线上公开 Demo 的 `/api/version` 与 `/api/health` 仍是 V7.1.0 / schema 9，因此线上不能作为不确定时间线、来源校准台或 schema 12 归档的验收依据。V8 专项检查覆盖时间校准后端/UI、collection import、media backup/restore、archive inspection、database/collection health 与 frontend；精确断言数和 HTTP smoke 总数以发布前最终完整运行输出为准，不沿用 V7.3 的 202 条旧基线。

## 从影像记忆走到 V8

文字记录“当时发生了什么”，照片还保留了表情、旧物、字迹和画面之外的情绪。V4 不把图片做成一个装饰性附件，而是把它升级为一等记忆对象：有独立的文件生命周期、隐私策略、结构化关联、区域证据、对照工具和可恢复备份。

项目的差异点不在于宣称“AI 看懂了照片”，而在于建立一条更可信的链路：

```text
原始照片 → 安全展示版本 → 用户圈选或机器候选 → 可编辑草稿 → 用户确认 → 可回看的图片证据
```

这条链路既适合真实私人数据，也能在面试中体现产品判断、安全工程和可解释 AI 的结合。V5–V6.1 在它之上加入主题展览、记忆回访、语义线索、人物档案和语音记忆；V7 解决下一步问题：一段已经核对过的记忆，怎样被留给未来，或安全地交给不登录时屿的人阅读。

```text
已确认展览 → 显式选择安全素材 → 封存本地时间胶囊
                                ↘ 浏览器端口令加密 → 自包含离线 HTML
```

时间胶囊强调仪式感，加密分享负责保密。产品不会把一个本地日期夸大成密码学时间锁，也不会把口令交给服务端代管。

V7.2 处理的是记忆长期保存后的维护问题：一次误改怎样找回、两个页面怎样避免互相覆盖、备份在真正恢复前怎样先证明结构完整。它没有增加新的顶层任务，而是在已有展品详情、“讲解与回顾”和“数据与项目”中补上可信维护闭环：

```text
有效编辑 → 规范快照 → SHA-256 父链追加 → 预览旧版 → 恢复为新的 head
旧页面提交 → If-Match 不匹配 → 412 拒绝写入 → 刷新后重新核对
本机馆藏 → 只读体检 ─┬→ 数据库、修订链、图片与声音结果
备份文件 → 隔离验真 ─┘→ 只返回可恢复摘要，不写业务数据
```

这里的“可信”指来源、版本和完整性边界可以复核，不代表系统能证明记忆内容为真，也不把本机哈希链包装成不可篡改的第三方审计账本。

V7.3 继续处理两类容易被忽略的用户决定：系统不应根据一次浏览或跳过擅自判断用户还想不想见到某段记忆；分享时，“已经带入候选”也不等于“已经同意交给别人”。

```text
明确回访选择 → welcome / later / pause → 只调整主动回访 → 随时恢复自然回访
安全候选材料 → 三层隐私编辑台 → 默认全部不选 → 最终收据 → 浏览器端加密
```

V8 处理另一类长期记忆系统必然遇到的问题：同一往事可能在当前展品日期、旧修订、原文日期摘录和照片 EXIF 中留下不同时间。系统先把可复核来源并列展示，再保存用户本次选择及其来源集合摘要；来源后来变化时，旧判断不会被静默删除，而是进入 `needsReview` 待复核。

```text
当前日期 / 修订 / 原文日期锚点 / EXIF 日期
                    ↓ 人工核对来源
一个日期 / 一个范围 / 保留多种记录 / 仍不确定
                    ↓
独立时间判断（不回写原文，不改展品日期，不裁决真相）
```

## 目标用户

- 想长期保存日记、聊天片段、旧照片和照片背后故事的人。
- 记录很多，但很难再次找到某个人、地点、物件、字迹或情绪的人。
- 希望用 AI 辅助整理私人内容，同时在意原文、原图、依据和数据边界的人。
- 希望看到完整交互闭环、数据建模和失败补偿，而不只是模型调用的面试官。

## 核心用户任务

1. 写下一段未经整理的原始记忆。
2. 可选添加最多 6 张 JPEG、PNG 或 WebP 照片，并决定“保留原图”还是“仅保留安全展示图”。
3. 为照片补充说明、拍摄时间、照片背面的故事，设置封面并调整顺序。
4. 查看 AI 给出的标题、展厅、人物、标签、情绪和说明，修改并确认后保存展品。
5. 在展品详情中浏览照片胶卷，圈选人物、地点、物件、文字、日期或其他区域，亲自确认后保存为图片线索。
6. 主动寻找“可能相似”的照片，或对选定区域进行本机文字摘录；核对结果，而不是让系统自动合并或改写记忆。
7. 沿记忆航线进入时光拼图，对照两个文字版本及其图片证据；需要时用两组对应点手动叠影。
8. 在已人工确认的时光拼图中展开来源校准台，对照四类受控时间来源，并明确选择一个日期、一个范围、保留多种记录或仍不确定；来源改变后再决定是否更新旧判断。
9. 按关键词、人物、地点、标签或情绪重新找到展品，再借助图片线索理解馆藏并核对讲解引用。
10. 从自己选择的展品生成主题展览，核对章节、引用与媒体后再确认发布。
11. 从已确认展览创建时间胶囊；到期前只看外壳，到期后再读取封存快照。
12. 在三层隐私编辑台核对公开外壳、密文内受众/用途与逐项内容、最终精确收据，再设置口令并下载可断网打开的单文件。
13. 为单件展品明确选择欢迎主动出现、指定日期以后或暂停主动出现；需要时恢复自然回访，不影响馆藏与搜索。
14. 用 `.time-isle` 完整归档迁移文字、照片、声音、展览、胶囊、回访意愿、时间判断、关联和证据，或导出物理排除私人内容的脱敏归档。
15. 在“记忆年轮”中预览展品正文的连续版本；需要纠正误改时，经二次确认把旧快照复制成新的当前版本，而不是抹掉后来历史。
16. 在“数据与项目”中运行只读馆藏体检，或先验真一份 `.time-isle` 是否可安全恢复，再决定是否执行真正恢复。

## 极简与渐进披露

从 V4 影像记忆扩展到 V8，页面仍严格保持“展品库、记录记忆、讲解与回顾、数据与项目”四个主视图。

- 添加照片位于原有记录表单内，默认折叠；只写文字的用户不会被媒体流程打断。
- 展品卡片只显示一张小封面和照片数量，不把首页变成复杂相册。
- 完整胶卷、照片背面和图片线索只在展品详情中出现。
- “照片线索”“影像线索实验台”和“手动叠影”都是可选折叠层，按需展开。
- “不确定时间线”只在讲解与回顾中显示默认折叠摘要；来源校准台只嵌入已确认事件的时光拼图，用户展开后才读取候选来源，不增加第五项导航或常驻日期表单。
- 主题展览、回访、人物档案与“胶囊与分享”都收在“讲解与回顾”已有上下文中，不增加第五项导航。
- 单件“以后怎样再见”位于展品详情，长期意愿管理位于回顾页折叠层；两者只有用户展开后才读取，不把偏好做成常驻打扰。
- 全馆最近版本位于“讲解与回顾”的折叠式“记忆年轮”，只有展开后才读取；单件展品的完整版本链继续收在详情底部，查看旧快照和恢复确认按需出现。
- 乐观并发保护不增加常驻控件；只有编辑页已经过期时才阻止保存并提示刷新，正常单页面使用不受打扰。
- 馆藏体检与备份验真位于“数据与项目”的数据边界卡片内，默认折叠，不在启动时自动扫描私人文件；体检由用户主动开始、可以停止，备份只在用户选择文件后验真。
- 胶囊工作室默认关闭；安全材料读取完成后先进入独立三层隐私编辑台，逐项确认完成才出现口令步骤。
- 桌面端导航留在页首；窄屏改为底部固定的四项水平导航，文字和点击区域居中，并为系统安全区预留空间。
- 相似候选不会自动执行动作，OCR 不会直接写入正文，EXIF 也不会弹出强制填写。
- 指针操作都有键盘后备：图片区域和手动对齐点均可输入 0–1 规范化坐标。

复杂能力被放在用户已经产生意图的地方，默认界面仍保持清爽、留白和低认知负担。

## 核心能力

### 1. 图片作为一等记忆对象

- 支持 JPEG、PNG、WebP，拒绝伪造 MIME、SVG、GIF、截断容器和像素炸弹。
- 原图流式进入本机暂存区；浏览器另外生成最长边 1600 px 的展示 WebP 和 480 px 缩略 WebP。
- “保留原图”会保存原文件、展示图和缩略图；“仅保留安全展示图”会在完成处理时删除暂存原图，只留下不含原始元数据的派生图。
- SHA-256 相同且既有全部变体重新通过磁盘完整性校验的内容才复用同一媒体资产，但不会因此合并两段展品或认定它们讲述同一事件。
- 每张图片可记录封面角色、顺序、说明、无障碍文本、拍摄时间和照片背面故事。
- 解除最后一个展品关联后，系统隔离并回收无人引用的媒体文件；刚完成上传但尚未关联的图片有 24 小时宽限，避免并发清理。清空馆藏先隔离媒体，再提交数据库事务；失败会恢复文件，物理删除未完成时会明确提示并在后续启动协调。

### 2. 照片区域成为可核对证据

用户可以在规范展示图上拖拽圈选，也可以用键盘填写 `x / y / width / height`。保存的区域使用 `canonical-preview-v1` 坐标空间，并锚定来源图片 SHA-256。

区域类型包括人物、地点、物件、文字、日期和其他；说明、备注与敏感标记都由用户填写。系统分别表达两层状态：

- `source_verified`：区域仍指向同一份图片来源。
- `user_confirmed`：区域含义由用户亲自确认。

来源完整不代表语义真实。区域证据可以修改和删除，并会进入时光拼图的两侧图片依据。

### 3. 克制的影像智能

| 能力 | 系统做什么 | 用户确认点 | 系统明确不做什么 |
| --- | --- | --- | --- |
| EXIF | 严格解析 JPEG APP1 中的拍摄时间、时区、GPS 和方向，全部先记为建议 | 用户点击后才采用拍摄时间 | 不自动写记忆日期或地点，不联网反查 GPS |
| 相似候选 | 对安全展示图生成确定性的 dHash 检索指纹，返回“可能相似”列表 | 用户自行打开候选展品核对 | 不把感知哈希当作相同证明，不自动合并、删除或修改照片 |
| 文字摘录 | 浏览器支持 `TextDetector` 时，只识别用户圈选的本地图片区域；否则提供手动摘录 | 识别文本先进入可编辑草稿，用户勾选确认后才保存为 `text` 区域证据 | 不调用第三方 OCR，不伪装识别成功，不自动写展品正文 |

GPS 被视为敏感数据。“仅保留安全展示图”不会持久化敏感 GPS 线索；选择“保留原图”时，原文件本身仍可能含元数据，相关位置线索在本地标记为敏感并对普通接口隐藏具体值。完整备份可能包含这些私人数据，必须像原相册一样妥善保管。

### 4. 时光对照，而不是自动认定

时光拼图可以把两个记忆版本的照片并排显示，保留各自说明、替代文字和原图入口。可选的“手动叠影”允许用户：

- 在左图与右图各选择两个对应点，组成两组对应关系。
- 用图片点选或 0–1 数值坐标完成相同操作。
- 调整右图透明度、撤销或重置标点。
- 在浏览器 Canvas 中计算平移、旋转和等比缩放的二维相似变换。

它只帮助目视比较，不是自动配准，也不会判断两张照片是否来自同一地点、人物或事件。只有用户确认后，两个文字版本才会组成同一往事的版本组；原文和原图始终分别保留。

### 5. 可携带的完整记忆

`.time-isle` 是当前版本的主要完整备份格式，采用严格 POSIX ustar + gzip 封装，包含：

- 馆藏文字和记忆考古关系。
- 媒体资产，以及按隐私模式保留的 `original`（可选）、`display` 和 `thumb` 文件。
- 展品与图片的顺序、封面和说明关联。
- EXIF 建议、检索指纹和用户确认的图片区域证据。
- 主题展览、回访状态、线索与人物档案、语音记录及已确认文字稿。
- 时间胶囊的外壳、安全快照和 display WebP 关联。
- 展品正文与结构化字段的完整修订链，以及恢复来源关系。
- 单件展品与事件组的时间校准、用户选择的来源键、来源集合摘要和待复核语义。

Schema 12 完整 `.time-isle` 必须声明时间校准功能 section，并将其写入独立的 `timeline/calibrations.json`；即使当前没有校准记录也必须显式包含空 section，避免“旧版本不支持”和“支持但零记录”被混为一谈。

归档中的清单会记录各文件的大小、MIME 和 SHA-256。恢复前，系统先以两遍流式解析核对 gzip / tar 边界，再在隔离暂存区验证路径、清单、哈希、图片结构、功能 section 和交叉引用；任何损坏或高于当前能力的 schema 都会整批拒绝，不产生业务写入。每项完整时间校准同时保存“判断创建时摘要”和“导出时当前来源摘要”；恢复会从归档证据重建当前候选，只有重建摘要与导出时摘要一致才继续。两份摘要原本就不相等的合法待复核判断会连同最小来源快照保留，而归档缺少来源闭包则整包拒绝。只有兼容 JSON 无法重建同一来源集合时，才保留判断并返回 `requiresTimeIsle`。验证通过后，展品、修订、媒体、关联、图片线索与记忆考古数据在同一数据库事务中恢复；如果事务失败，已经移动的媒体目录会被补偿清理。命中同哈希本地资产时也必须重新核对隐私策略、全部变体和磁盘哈希，不能静默复用损坏或不同规范的预览。

脱敏 `.time-isle` 只保留允许公开的清单与脱敏 section；图片、声音二进制和私人媒体元数据在归档层物理缺席，胶囊标题、日期、来源、快照、图片关联与内部 ID 也被固定脱敏说明替代，修订 section 只留下计数与固定说明，而不是把旧正文藏在界面之后。时间校准的脱敏摘要严格只有 `mode / calibrationCount / uncertainCount / alternativesCount / note` 五个字段，日期范围、来源键、来源集合哈希、备注、目标 ID 与精确时间均物理缺席。脱敏主馆藏仍含可读摘要，不能自动视为匿名。旧版 JSON 导入导出继续作为兼容工具，但缺少照片二进制时无法重建 EXIF 来源，不能替代完整多模态备份。

### 6. 时间胶囊：外壳可见，内容到期

时间胶囊只允许从状态为 `published`、无需复核的主题展览创建。创建时生成不可被来源展览后续编辑静默改写的安全快照，并把三类数据分开保存：

- 外壳：标题、寄语、开启日期、时区和可用状态。
- 内容：匿名章节、展项、已核验原文引用和用户选择的已确认文字稿。
- 媒体关联：只指向用户明确选择、已经就绪的 display WebP。

开启日前，胶囊列表、详情和内容接口都只使用外壳。`GET /api/capsules/:id/content` 返回 `423`、稳定错误码和公开外壳，不读取内容快照，也不下发正文、图片、展品成员或数据库内部 ID。若来源图片后来被删除，胶囊会进入需要复核状态，而不是继续展示失去来源保证的材料。

开启日期使用用户选择的 IANA 时区解释，解决“哪一天算到期”的产品语义；它仍只是本地仪式门槛。能够修改本机时间或直接读取私人数据库的人不受它提供的保密承诺约束。

### 7. 浏览器端加密的离线展览

离线分享与胶囊共用“只从已确认展览开始”的安全来源，但两者职责不同：胶囊负责封存和到期体验，离线文件负责把内容加密后交给别人。

1. 服务端只列出来源展览中的 display WebP 与已确认文字稿；带入浏览器只表示成为候选，不表示同意分享。
2. 第一层核对无需口令即可看到的公开标题、说明和文件名。它们使用通用默认值，不复制来源展览/胶囊标题或导出时间，并固定为立即可打开。
3. 第二层填写密文内的受众和用途，编辑仅属于本次分享的叙事副本，再逐章、逐件、逐条选择引用、confirmed 文字稿与 display WebP。所有选择默认关闭，修改不会回写来源展览。
4. 第三层显示公开内容、解密后内容数量、固定排除项和“下载后无法撤回；知道口令的人仍可以复制、转发或截图”的边界；用户最终勾选确认后才出现口令步骤。
5. 从口令步骤开始不再发送网络请求。口令只在当前页面内存中参与 PBKDF2-SHA-256（310,000 次）和 AES-256-GCM 加密，浏览器随后下载自包含 `.html`。
6. 收件人输入正确口令即可解密；阅读页显示受众、用途、精确计数与不可撤回边界，并通过 Blob URL 在本机还原 display WebP，断网也能使用。

每个文件使用随机 16-byte salt、12-byte IV，公开外壳作为 AAD 的一部分参与完整性校验。V2 密文严格校验精确字段、连续 `section-N / item-N / media-N`、媒体所有权、Base64 字节数和收据计数；至少需要 1 章、1 件展品和 1 条引用或文字稿。V1 历史文件继续可解密阅读。口令不上传、不持久化、不写入文件，也无法找回。

投影结果物理排除原图、EXIF/GPS、草稿文字稿、Agent 日志、数据库内部 ID、URL、SHA、未选原文与原始音频；图片只允许安全 display WebP。编辑会撤销确认，加密过程中重新确认会使旧下载任务失效；取消、关闭、返回书架和下载后都会清空草稿、明文媒体、口令与 Blob URL。

### 8. 可安装但不缓存私人馆藏的 PWA

V7.1 为时屿补齐可安装外壳，但不把 PWA 等同于“离线馆藏”。安装能力沿用原有四项主导航，不新增第五个入口；“安装到此设备”位于“数据与项目”的项目卡中，并按浏览器能力渐进披露。

- Chromium 浏览器只有在收到 `beforeinstallprompt` 后才显示安装按钮；iPhone 与 iPad 不伪装成一键安装，而是给出 Safari“添加到主屏幕”的手动步骤。
- Service Worker 只在 HTTPS 或浏览器认可的本地安全上下文注册；非安全来源、不支持的浏览器和不满足安装条件的环境会静默隐藏入口，不阻塞记录、检索和策展主流程。
- 缓存采用精确白名单，只包含 `/offline.html`、`/pwa.css` 和公开品牌 SVG。首页、主应用脚本、API、图片、声音、归档与任何用户内容都不进入 Cache Storage，也不会在运行时动态写入缓存。
- 原图、展示图和缩略图等所有图片变体统一返回 `private, no-store`。断网导航只展示不含馆藏、照片、声音或导出内容的隐私边界页，不用空列表伪装成“馆藏为空”，恢复连接后再返回真实应用。

本地模式仍是完整媒体保存、胶囊封存和归档恢复的主要体验环境；公开 Demo 继续使用临时数据、强制 Mock 与禁写边界。PWA 安装不会改变两种运行模式的数据能力，也不会让公开 Demo 变成私人内容或长期数据的存储空间。加密分享文件可以在用户下载后独立断网解密阅读，这是自包含密文文件的能力，不代表时屿应用缓存了私人馆藏。

### 9. 可校验、可恢复的记忆年轮

Schema 10 为每件展品增加连续修订链。新建或导入展品会生成第一个版本；旧馆藏第一次发生有效编辑时，系统在同一事务内先补记编辑前基线，再追加编辑后的版本。只改变时间戳或提交相同语义内容不会制造伪修订。

- 快照覆盖原文、展签、标题、展厅、人物、标签、情绪、日期、地点、重要度、收藏状态、附件说明等展品正文与结构化字段，并按固定键序列规范化后计算 SHA-256。
- 每条修订保存连续序号、当前快照哈希和前一版快照哈希；恢复记录还必须引用同一件展品链中的旧版本。馆藏体检与完整备份恢复都会重新核对序号、父链、快照哈希和当前 head。
- 列表只读取版本摘要；用户主动点击“查看”后才读取那一版快照。恢复前再次确认，执行时把旧快照复制成新的 `restored` head，当前版本和全部中间历史仍然保留。
- 编辑与恢复必须携带当前 `ETag / If-Match` 或等价版本条件。缺少条件返回 `428`；旧页面与当前版本不匹配时返回 `412` 和新的 ETag，但不在冲突响应中下发较新的私人正文。
- 正文写入与修订追加位于同一 SQLite 事务。完整修订账本按可导出的 JSON 合同设置 20 MiB 总预算；新增、编辑、恢复或整批导入一旦越界，会整项回滚而不留下半条历史。

年轮不保存图片或声音文件、媒体关联、图片区域证据、展览、胶囊和 Agent 运行日志，恢复旧版不会把这些独立对象一起倒退。SHA-256 父链用于发现链内缺口或不一致，不提供外部签名、可信时间戳、作者身份或记忆真实性证明；拥有本机数据库控制权的人也不受它提供的不可篡改承诺约束。当前并发策略负责拒绝静默覆盖，不提供多人协同合并。

完整 `.time-isle` 与完整 JSON 会携带年轮并在恢复时安全重映射冲突 ID；脱敏导出只保留展品数、修订数和固定隐私说明，旧版正文、结构化字段、编辑备注、哈希、精确时间及修订内部 ID 物理缺席。

### 10. 只读馆藏体检、备份验真与流式归档

馆藏体检只在本地由用户主动启动，并串行核对三类信息：

- 数据库：SQLite `quick_check`、外键、schema 与迁移账本、FTS 文档成员关系，以及修订链和当前 head。
- 私人媒体：图片记录、隐私模式、必需变体、内容锚点和磁盘文件完整性；声音记录与对应音频完整性。
- 待复核事项：来源失效的引用、需要复核的展览或胶囊、开放问题和草稿文字稿。

体检异步显示阶段与进度，可以停止；结果只保留在当前服务运行内，问题最多展示前 200 项。它只报告，不自动删除、修复或改写内容，“本次未发现结构性异常”也不等于证明每段记忆的语义真实。

“选择 `.time-isle` 验真”会复用真正恢复前的完整解包、manifest、哈希、媒体格式、section 与交叉引用校验，但只在随机隔离暂存区工作；成功后返回模式、schema、条目体积和主要对象计数，失败或完成都会删除本次暂存，不写入业务数据库或正式媒体目录。验真通过代表这份文件满足当前版本的恢复合同，不替代用户保管多份备份，也不保证未来所有软件都能读取。

V7.2 将 `.time-isle` 导出改为文件与 HTTP 流式路径。服务端以私有临时文件承接流式 gzip；支持 File System Access API 的浏览器把响应直接写入用户选择的文件，不支持时交给浏览器原生下载，不在前端 JavaScript 中聚合整份归档。恢复端先完整扫描 gzip / tar 结构和硬上限，再把通过边界检查的条目写入隔离区；请求中止或失败会清理本次输出与暂存。

安全约束保持显式：

- 脱敏归档的每个功能 section 都采用精确字段集合和固定隐私说明；修订只留下计数摘要。主馆藏摘要仍可能保留标题、短展签、标签和情绪，因此“脱敏”不等于匿名，公开前仍需人工核对。
- 旧 JSON 兼容导入拥有独立的 64 MiB HTTP 请求上限，以容纳受 20 MiB 预算约束的修订合同；它仍受对象数量、字段、引用和 schema 校验，不是无限导入。
- 本机 SQLite、JSON 导入和 `.time-isle` 验真/恢复都会拒绝高于当前程序能力的 future schema，提示升级应用，不会猜测性忽略未知数据。
- 启动维护只清理 `.exports`、`.restore`、`.inspect` 下超过一小时且名称符合程序规则的崩溃残留；真实目录与安全路径会先核对，较新的任务和无关文件不会被当作垃圾删除。
- 流式处理降低内存聚合风险，但仍需要足够的本机临时磁盘空间，也不把本机事务夸大成跨设备分布式事务。

### 11. 明确回访意愿，而不是行为推断

每件展品都可以在详情中设置“以后怎样再见”：

- `neutral`（自然回访）不保存长期记录，沿用往年今日、很久没见和随机漫游的原有规则。
- `welcome`（欢迎主动出现）只在当前回访类型的合格候选中优先，并在依据中标记来源为 `user-confirmed`。
- `later`（指定日期以后）在用户选择的本地日期到达前过滤主动回访；日期与 IANA 时区一起保存，跨时区行为可测试。
- `pause`（暂停主动出现）过滤所有主动回访，直到用户亲自恢复自然回访。

这项意愿不保存原因，也不会从浏览、跳过、搜索、人物关系或情绪字段自动产生。它不隐藏或删除展品，不影响搜索、详情和归档读取。单件设置与全局管理都默认折叠、按需请求；写入要求 `confirm: true`，公开 Demo 返回 403。

Schema 11 用带 CHECK、外键级联和 500 条硬上限的独立表保存 `welcome / later / pause`。完整 JSON 与 `.time-isle` 必须显式包含回访意愿，恢复时复用展品 ID 映射并在同一事务完成；脱敏归档只保留总数和固定说明，展品 ID、选择、延期日期、时区和精确时间物理缺席。分享草稿不写数据库，也不进入归档。

### 12. 不确定时间线与来源校准台

Schema 12 为单件展品或人工确认的事件组保存一份独立时间判断。面向用户的校准台只提供四种清楚选择：

- 一个日期：为本次时间线展示选择精确日；该日必须落在每个已勾选来源的区间内。
- 一个范围：保存包含起止日的时间区间；该范围必须与每个已勾选来源区间至少有一天相交。
- 保留多种记录：承认多个来源各自成立，不强行合并为一个日期；所选来源中必须至少存在一对互不相交的日期区间，避免把本可兼容的记录伪装成冲突。
- 仍不确定：明确保存“目前无法判断”，而不是让空值看起来像系统遗漏。

校准台只接收四类受控来源，并把候选值、来源类型和可复核锚点并列呈现：

- 展品当前日期：反映现在可见的结构化日期，但不因此拥有更高真实性。
- 记忆年轮中的历史修订日期：以规范快照 SHA-256 和修订序号锚定。
- 原文日期 claim：必须仍能由原文摘录与字符区间复核，失效、拒绝或撤回的 claim 不进入候选。
- 关联照片的非敏感 EXIF `captured_at`：只接受已建议或已确认的有效日期；GPS 从时间校准链路严格排除。

每个候选生成内容稳定的 `sourceKey`，整组候选生成 `sourceSetSha256`。保存时记录用户勾选的来源键、选择类型、可选范围与备注，并保存一份只包含来源键、来源类型、精度与起止日的最小不可变来源快照；快照不携带标题、摘录、媒体地址、GPS 或整段原文，使来源后来失效时旧判断仍可解释。系统不会把校准结果写回原文、展品日期、修订或 EXIF。来源集合后来发生变化时，旧判断与这份快照完整保留但派生为 `needsReview`，时间线、讲解与回顾只提示待复核，不会自动采用新来源或删除旧选择。事件级判断优先用于该事件的时间线展示，成员级判断仍保留且可单独核对。

读取校准台与不确定时间线保持只读；公共时间线摘要严格只返回重新打开目标所需的最小目标边界、判断摘要、待复核状态和事件成员 ID，不下发原文、媒体、来源快照、备注或来源哈希。保存、更新和恢复未校准状态都需要显式确认。所有写入使用 `ETag / If-Match`，缺少条件返回 `428`，过期页面返回 `412`；公开 Demo 可以查看预置的“保留多种记录”示例，但任何校准 `PUT / DELETE` 都返回 `403`。

## 产品原则

- 原文与原图优先：AI 草稿不覆盖原文，派生视图不冒充原图。
- 人工确认：AI 整理、EXIF、相似候选、OCR 和事件关系都有明确确认边界。
- 明确意愿优先：长期回访只听从用户确认，不用浏览行为推断心理；恢复自然回访会真正删除长期偏好。
- 证据可回看：文字证据保留原文字符位置，图片证据保留来源哈希与规范化区域。
- 本地优先：SQLite 与内容寻址媒体目录默认位于本机，不要求外部数据库或对象存储。
- 版本并存：同一往事的多次文字与图片记录分别保留，系统不制造唯一真相。
- 来源并列：日期冲突先展示受控来源和可复核锚点，再保存人的判断；`needsReview` 只表示来源集合发生变化，不表示旧记忆错误。
- 时间判断独立：一个日期、一个范围、多种记录与仍不确定都不会回写原文或展品日期，也不被包装成事实裁决。
- 历史追加：恢复旧版生成新的 head，不删除当前版本；过期编辑先报冲突，再由用户重新核对。
- 缺失不是矛盾：只有两侧都有可验证信息时才展示描述差异。
- 日期与保密分工：开启日期提供仪式门槛，浏览器端加密才承担离线分享的保密职责。
- 最小分享：带入编辑台不等于同意分享；章节、展品、引用、确认文字稿和安全展示图默认均不选，未选择等于物理缺席。
- 分享副本独立：公开外壳与加密叙事只属于本次文件，不复制不必要来源元数据，也不回写来源展览。
- 失败安全：上传、归档和恢复在失败时清理暂存或补偿已移动文件。
- 先验真后写入：馆藏体检保持只读，备份通过当前恢复合同后才允许进入正式事务。
- Demo 隔离：公开展示不接收私人图片，并阻止破坏示例稳定性的写操作。

## 当前范围

已经实现：

- 单条记忆整理、三阶段轨迹、编辑确认和结构化 SQLite 馆藏。
- 最多 6 张照片的本地安全上传、派生、去重、关联、展示、编辑和回收。
- 原图保留与仅安全展示图两种隐私策略。
- EXIF 待确认线索、感知相似候选和隐私优先的照片文字摘录。
- 锚定来源哈希的用户确认图片区域证据。
- 混合检索、引用式讲解、时间线、主题和摘要。
- 可解释记忆航线、带原文锚点和图片证据的时光拼图。
- 两图并排与手动两点相似变换叠影。
- 人工确认的事件版本组、单问题补充和“保留不确定”。
- 基于当前日期、历史修订、有效原文日期 claim 与非敏感 EXIF 日期的来源校准台，以及一个日期、一个范围、保留多种记录、仍不确定四类人工判断。
- 默认折叠的不确定时间线摘要、来源集合变化后的 `needsReview` 待复核，以及事件成员最小边界下的拼图重开入口。
- 用户确认后才保存的主题展览，以及一次只呈现一件记忆的回访。
- 由用户明确确认的欢迎、延期、暂停与自然回访恢复，以及按需读取的长期意愿管理。
- 可解释语义线索检索、人物/地点/主题档案与人工确认的别名关系。
- 浏览器录音或音频上传、独立音频存储、回放与人工确认文字稿。
- 只来源于已确认展览的时间胶囊；未到期只返回外壳，到期后读取安全快照。
- 三层隐私编辑、V2 精确收据、PBKDF2-SHA-256 + AES-256-GCM 加密、自包含且断网可读的 HTML 分享文件，并兼容 V1 阅读。
- 不缓存私人馆藏的可安装 PWA 外壳，以及断网时不读取用户内容的隐私边界页。
- SHA-256 快照父链、旧版预览与追加式恢复，以及 `If-Match` 乐观并发保护。
- 本地只读馆藏体检与不写入业务数据的 `.time-isle` 备份验真。
- schema 12 `.time-isle` 完整/严格脱敏流式归档、回访意愿与时间校准 ID 映射/事务恢复、崩溃暂存清扫及旧 JSON 兼容工具。
- 本地模式和受保护的公开 Demo 模式。

不在当前范围：

- 多用户账号、细粒度权限和跨设备端到端加密身份体系。
- 云端对象存储、云数据库和跨设备实时同步。
- 第三方云 OCR、自动图片理解或自动生成照片事实。
- GPS 反向地理编码或自动填写地点。
- 基于相似度自动合并、删除照片或合并记忆事件。
- 视频和动态图片存储；离线分享首版也不携带原始音频二进制。
- 公网私密链接、账号托管、分享撤回、远程失效和口令找回。
- 外部签名的不可篡改审计账本、可信时间戳、多人实时编辑或自动冲突合并。
- 馆藏异常的自动修复、自动删除或无人确认的数据迁移。
- 自动替用户发布、选择分享内容或训练私人记忆。
- 根据来源数量、EXIF 或模型置信度自动决定唯一日期，或用校准结果覆盖原文、展品日期和旧修订。

## 面试展示建议

本地 V8 推荐用六至八分钟展示：

1. 用虚构文字和一张无隐私风险的照片创建展品，简述原图/安全图、人工确认与四项主导航。
2. 编辑同一展品，再从详情展开“记忆年轮”，按需查看旧快照；恢复早期版本后展示它成为新的 head，原有版本没有消失。
3. 用两个页面演示过期 `If-Match` 被 `412` 拒绝，说明它避免静默覆盖，但不是多人实时协同或自动合并。
4. 在“数据与项目”主动运行馆藏体检，展示数据库、图片、声音和待核对事项；强调只读、可停止且不自动修复。
5. 选择一份测试 `.time-isle` 先做备份验真，再分别导出完整与脱敏归档；说明流式路径、20 MiB 修订预算、64 MiB JSON 兼容导入及 future schema 拒绝。
6. 圈选一处图片证据并进入时光拼图，说明来源哈希、用户确认和手动叠影各自只解决什么问题。

## 已知限制

- `TextDetector` 不是所有浏览器的标准可用能力；不可用或启动失败时，产品只提供手动摘录，不会降级到远程 OCR。
- EXIF 当前只严格读取 JPEG APP1/Exif 的有限字段；无时区的拍摄时间保持 floating local time，不会伪造 UTC。
- 感知相似度适合召回人工候选，不是身份、地点或同一事件证明；完全相同 SHA-256 只用于资产复用。
- 手动叠影依赖用户给出两组可靠对应点，只计算平移、旋转和等比缩放，不处理透视、非刚性变形或自动特征匹配。
- 图片区域的 `source_verified` 只证明来源锚点仍匹配，不证明标签内容为真。
- Mock 模式用于可用性演示，语言质量不等同于真实模型；真实模型讲解仍没有逐句忠实度评测。
- 语义检索是本地词汇扩展和字段加权，不是向量数据库。
- `.time-isle` 完整备份不包含 Agent 运行日志；它包含私人原文和媒体时必须由用户自行安全保存。
- 记忆年轮只覆盖展品正文与结构化字段，不是图片、声音、展览和胶囊的全对象版本系统；哈希父链也不是外部签名的不可篡改账本或事实证明。
- 乐观并发只拒绝过期写入，不会自动合并两边内容；修订账本总预算为 20 MiB，达到边界后继续产生修订会被拒绝，系统不会静默删掉旧历史腾空间。
- 馆藏体检只发现当前规则能够识别的结构和文件问题，不验证记忆语义，也不提供自动修复；备份验真会使用隔离暂存和磁盘空间，但不会恢复业务数据。
- JSON 兼容导入最多接收 64 MiB 请求；future schema 必须先升级应用，不能向旧版本降级恢复。
- 流式 `.time-isle` 避免在浏览器 JavaScript 中聚合整包，但服务端仍会创建受控临时文件；异常退出后的归档残留最快在超过一小时并再次启动维护时清扫。
- 本地时间胶囊不是可信第三方时间锁；拥有数据库或设备控制权的人可以绕过日期门槛。
- PBKDF2 会提高口令猜测成本，但不能补救弱口令。离线文件没有找回或撤回能力，口令应通过另一条安全渠道传递。
- 下载后的离线文件无法由发送者远程撤回；知道口令的人仍可复制、转发或截图。
- 自包含阅读页依赖浏览器 Web Crypto 与 Blob URL；只分享被逐项选择的 display WebP、已确认引用和文字稿，不分享原图、EXIF/GPS 或原始音频。
- 回访意愿不是心理画像或重要度模型；welcome 只做候选内优先，later/pause 只约束主动回访，不影响用户主动搜索和查看。
- 时间校准只保存用户本次展示判断与来源摘要，不证明某个日期真实；来源集合变化只触发待复核。若 JSON 迁移缺少照片媒体来源，旧判断会保留并提示需要 `.time-isle` 完成无损迁移，而不会伪造 EXIF 来源。
- Vercel `/tmp` 数据可能随实例重建且可能存在多实例，不适合长期馆藏。
- 当前没有账号、云同步或远程媒体备份；“本地优先”不等于自动具备跨设备容灾。

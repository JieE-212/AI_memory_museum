# 产品说明：公开 V14 与本地 V17 候选

## 一句话定位

时屿（TIME ISLE）是一款本地优先的 AI 私人记忆策展工具：它保存原文、照片、声音与用户确认过的证据，把散落片段组织成可以检索、回访和讲述的展览；本地 V17 候选版又补上一次性真实隔离恢复、多视角记忆对照和设备内中文语义找回，同时继续把恢复、关系判断和结果解释留给用户。

## 版本状态

已公开版本仍是 2026-07-20 发布的 V14.0.0（schema 19），功能提交为 `2dcce402b13f1d43c54c6a196b8e2273c9483eb3`；GitHub / Gitee `main` 已对齐，其中 GitHub `main` 触发 Vercel 生产发布。生产 `/api/version` 与 `/api/health` 已核验为 `14.0.0 / schema 19 / interview-demo / ephemeral-sqlite / mock-fallback`；线上是受保护的共享临时 Demo，核心纯文本体验受事务硬上限约束，V11–V14 新模块与媒体、恢复等高风险写操作保持只读或禁用。

当前本地工作区是 V17.0.0 候选版，schema 仍为 19。V15–V17 已完成实现和本地验收，但尚未提交、推送或部署；build/check、272 条真实 HTTP smoke、21/21 项 Playwright 三档门禁，以及桌面、390×844、320×700 真实浏览器验收均已通过。V18 仅完成 Windows x64 SQLite3MC canary；由于缺少 Linux `node:24-bookworm-slim` 与 CloudBase 同等证据，总体门禁为 **NO-GO**，正式数据层没有实现真正加密。V10.0.0（schema 14）继续只作为历史发布基线保留。

## 从影像记忆走到 V14

文字记录“当时发生了什么”，照片还保留了表情、旧物、字迹和画面之外的情绪。V4 不把图片做成一个装饰性附件，而是把它升级为一等记忆对象：有独立的文件生命周期、隐私策略、结构化关联、区域证据、对照工具和可恢复备份。

项目的差异点不在于宣称“AI 看懂了照片”，而在于建立一条更可信的链路：

```text
原始照片 → 安全展示版本 → 用户圈选或机器候选 → 可编辑草稿 → 用户确认 → 可回看的图片证据
```

这条链路既适合真实私人数据，也能在面试中体现产品判断、安全工程和可解释 AI 的结合。V5–V6.1 在它之上加入主题展览、记忆回访、字段与线索检索、人物档案和语音记忆；V7 解决下一步问题：一段已经核对过的记忆，怎样被留给未来，或安全地交给不登录时屿的人阅读。

```text
已确认展览 → 显式选择安全素材 → 封存本地时间胶囊
                                ↘ 浏览器端口令加密 → 自包含离线 HTML
```

时间胶囊强调仪式感，加密分享负责保密。产品不会把一个本地日期夸大成密码学时间锁，也不会把口令交给服务端代管。

V7.2 处理的是记忆长期保存后的维护问题：一次误改怎样找回、两个页面怎样避免互相覆盖、备份在真正恢复前怎样先证明结构完整。它没有增加新的顶层任务，而是在已有展品详情、“讲解与回顾”和“数据与项目”中补上可信维护闭环：

```text
有效编辑 → 规范快照 → SHA-256 父链追加 → 预览旧版 → 恢复为新的 head
旧页面提交 → If-Match 不匹配 → 412 拒绝写入 → 刷新后重新核对
本机馆藏 → 只读体检 ─┬→ 数据库、修订链、图片与声音结果
备份文件 → 隔离验真 ─┘→ 只返回可恢复摘要，不写业务数据
```

这里的“可信”指来源、版本和完整性边界可以复核，不代表系统能证明记忆内容为真，也不把本机哈希链包装成不可篡改的第三方审计账本。

V7.3 继续处理两类容易被忽略的用户决定：系统不应根据一次浏览或跳过擅自判断用户还想不想见到某段记忆；分享时，“已经带入候选”也不等于“已经同意交给别人”。

```text
明确回访选择 → welcome / later / pause → 只调整主动回访 → 随时恢复自然回访
安全候选材料 → 三层隐私编辑台 → 默认全部不选 → 最终收据 → 浏览器端加密
```

V8 处理另一类长期记忆系统必然遇到的问题：同一往事可能在当前展品日期、旧修订、原文日期摘录和照片 EXIF 中留下不同时间。系统先把可复核来源并列展示，再保存用户本次选择及其来源集合摘要；来源后来变化时，旧判断不会被静默删除，而是进入 `needsReview` 待复核。

```text
当前日期 / 修订 / 原文日期锚点 / EXIF 日期
                    ↓ 人工核对来源
一个日期 / 一个范围 / 保留多种记录 / 仍不确定
                    ↓
独立时间判断（不回写原文，不改展品日期，不裁决真相）
```

V9 继续处理“书面来源仍然解释不了冲突时怎么办”。系统只为符合条件的已确认事件生成一个问题；用户录音或选择本地音频后亲手划定片段、填写文字稿，并明确回答代表单日、范围还是仍不确定。机器不替用户转写、认人、判断情绪或决定日期。

```text
未解决且双侧有依据的日期冲突 → 一个事件级问题
→ 录音或本地音频 → 人工选段 → 手填文字稿 → 明确 day / range / uncertain
→ 已确认 day/range 成为独立口述来源；uncertain 只保留证据
```

V10 继续处理“怎样让助手组织馆藏，但不把决定权交给它”。首版执行器只在本机运行确定性证据规则；四项工具全是只读查询，固定预算内只生成带来源摘要的 `proposal / preview`。保存草稿、确认候选关系和发布是三项独立人工决定，分享仍由原有三层隐私编辑台逐项选择。

```text
2–6 件来源 / 查询 → 四项只读工具 → 固定预算回执 → 未保存策展提案
                                              ↓
             保存草稿 / 确认关系 / 发布：逐项确认；分享：另行最小披露
```

V11 先补齐后续协作需要的来源底座：TXT/Markdown 只在浏览器临时读取，用户逐段选择后由服务端重算文件、文本和 UTF-16 片段锚点；片段与新展品同事务确认，整份源文件不落库。来源护照把文字、文档、图片区域和声音片段作为独立证据，关系与主张状态都由用户确认；来源变化只进入 `needsReview`，不产生真假分数。

V12 处理“怎样邀请他人补充一段回忆，又不把协作做成账号社交”。邀请与回信都在浏览器本地加密，并由请求摘要绑定；馆主解密预览后再决定是否把它作为独立来源入馆。加密只保护馆外文件，回信身份仍是自述且未验证，文件也没有数字签名。

V13 处理“怎样换一种角度看馆藏，但不再引入一个聊天机器人”。时间、共同出现、证据与线索四种镜片只重排已保存字段和确认来源，零外部模型、零工具调用、零持久化；只有用户明确选择的 2–6 件展品才可交接给策展助手，更多来源不会被静默截断。

V14 处理“怎样暂时停止新的写入，并诚实检查备份结构”。锁馆等待在途写操作结束后，在读取正文前以 423 阻止新的 mutation；它不会加密 SQLite、图片、声音或磁盘。结构演练只核对完整归档的 manifest、哈希和引用，不执行恢复，因此既不显示“恢复成功”，也不证明灾难恢复能力。

V15 把“结构能通过”推进到“真的恢复一次”。完整 `.time-isle` 会被写入全新的一次性 SQLite、图片目录和声音目录，随后由独立检查重新核对数据关系和媒体完整性，再关闭数据库并销毁整个副本。当前馆藏从不参与这次恢复；它证明的是当前机器、当前版本可恢复，不是异机灾备承诺。

V16 回答“同一段往事怎样从多个视角一起看”。单件展品旁可以并列当前记录、同一馆主的编辑年轮、已确认入馆的亲友回信、来源护照关系与时间判断。只有人工确认的支持、补充或不同记录关系才形成对照；未绑定、已撤回、来源变化和身份未核验都不会被包装成一致或冲突。

V17 回答“忘了原句，只记得一个画面时怎样找回”。用户主动启用后，浏览器加载约 47 MB 的同源中文模型，在 Worker 内生成真实 512 维 embedding，并只在当前页面内存中建立最多 500 件展品的临时索引。查询、短文本快照和向量不发往第三方；结果只代表文字相似度，最接近也可能不相关。

## 目标用户

- 想长期保存日记、聊天片段、旧照片和照片背后故事的人。
- 记录很多，但很难再次找到某个人、地点、物件、字迹或情绪的人。
- 希望用 AI 辅助整理私人内容，同时在意原文、原图、依据和数据边界的人。
- 希望看到完整交互闭环、数据建模和失败补偿，而不只是模型调用的面试官。

## 核心用户任务

1. 写下一段未经整理的原始记忆。
2. 可选添加最多 6 张 JPEG、PNG 或 WebP 照片，并决定“保留原图”还是“仅保留安全展示图”。
3. 为照片补充说明、拍摄时间、照片背面的故事，设置封面并调整顺序。
4. 查看 AI 给出的标题、展厅、人物、标签、情绪和说明，修改并确认后保存展品。
5. 在展品详情中浏览照片胶卷，圈选人物、地点、物件、文字、日期或其他区域，亲自确认后保存为图片线索。
6. 主动寻找“可能相似”的照片，或对选定区域进行本机文字摘录；核对结果，而不是让系统自动合并或改写记忆。
7. 沿记忆航线进入时光拼图，对照两个文字版本及其图片证据；需要时用两组对应点手动叠影。
8. 在已人工确认的时光拼图中展开来源校准台，对照四类受控时间来源，并明确选择一个日期、一个范围、保留多种记录或仍不确定；来源改变后再决定是否更新旧判断。
9. 当书面日期仍冲突时展开一问一答口述史，录音或选择本地音频、人工划定片段、手填文字稿并明确时间含义；需要时重新回答或撤回，不覆盖旧证据。
10. 按关键词、人物、地点、标签或情绪重新找到展品，再借助图片线索理解馆藏并核对讲解引用。
11. 从自己选择的展品生成主题展览，核对章节、引用与媒体后再确认发布。
12. 从已确认展览创建时间胶囊；到期前只看外壳，到期后再读取封存快照。
13. 在三层隐私编辑台核对公开外壳、密文内受众/用途与逐项内容、最终精确收据，再设置口令并下载可断网打开的单文件。
14. 为单件展品明确选择欢迎主动出现、指定日期以后或暂停主动出现；需要时恢复自然回访，不影响馆藏与搜索。
15. 用 `.time-isle` 完整归档迁移文字、照片、声音、口述史、展览、胶囊、回访意愿、时间判断、关联和证据，或导出物理排除私人内容的脱敏归档。
16. 在“记忆年轮”中预览展品正文的连续版本；需要纠正误改时，经二次确认把旧快照复制成新的当前版本，而不是抹掉后来历史。
17. 在“数据与项目”中运行只读馆藏体检，或先验真一份 `.time-isle` 是否可安全恢复，再决定是否执行真正恢复。
18. 在“讲解与回顾”提出一个策展主题，核对只读来源、提案章节和评测回执；再分别决定是否保存草稿、确认候选关系和发布，分享内容仍到独立隐私编辑台逐项选择。
19. 从 TXT/Markdown 逐段选择原文，核对文件哈希、字符/行列锚点和入馆草稿，再明确确认来源与新展品；随后用来源护照记录支持、补充或不同记录，不让系统生成真假分数。
20. 围绕一件记忆生成一封共忆邀请，让对方离线填写回忆并导出加密回信；馆主核对“身份未验证、文件未签名”后再明确保存为独立来源。
21. 明确选择 2–20 件展品，用时间、共同出现、证据或线索镜片查看可解释的未保存投影；若要交给策展助手，再主动收窄到 2–6 件。
22. 在“数据与项目”中设置口令并锁馆，暂时阻止新的写入；仍可导出、验真，并用完整 `.time-isle` 运行只读结构演练，且不会把结构通过说成恢复成功。
23. 用完整 `.time-isle` 运行一次性真实隔离恢复，按“验真、恢复副本、独立体检、销毁副本”查看安全回执；当前馆藏不被替换。
24. 从单件展品打开多视角对照，并列当前记录、编辑年轮、已确认亲友回信和人工来源关系；对未绑定、待复核或身份未核验状态继续保留不确定。
25. 在“讲解与回顾”主动准备设备语义，用自然描述按文字含义找回展品；完成后释放本页 Worker 内存，不把相似度当成事实判断。

## 极简与渐进披露

从 V4 影像记忆扩展到 V14，页面仍严格保持“展品库、记录记忆、讲解与回顾、数据与项目”四个主视图。

- 添加照片位于原有记录表单内，默认折叠；只写文字的用户不会被媒体流程打断。
- 展品卡片只显示一张小封面和照片数量，不把首页变成复杂相册。
- 完整胶卷、照片背面和图片线索只在展品详情中出现。
- “照片线索”“影像线索实验台”和“手动叠影”都是可选折叠层，按需展开。
- “不确定时间线”只在讲解与回顾中显示默认折叠摘要；来源校准台只嵌入已确认事件的时光拼图，用户展开后才读取候选来源，不增加第五项导航或常驻日期表单。
- “一问一答口述史”继续嵌入时光拼图并默认折叠；只有符合条件的日期冲突才显示问题，录音、文件选择、选段、文字稿和时间含义按步骤渐进出现，窄屏保持单列和 44px 触控目标。
- “受限策展助手”收在“讲解与回顾”的既有入口，以对话层按“来源 → 只读执行 → 未保存提案 → 逐项决定 → 技术回执”渐进呈现；提案、引用、决定和最近运行默认折叠，不占据首页，也不增加导航。
- “共忆信笺”沿用展品/回顾上下文，邀请生成、回信解密预览和明确入馆分步呈现；口令在请求结束、关闭或离页时清理，文件不会自动发送或保存。
- “设备内镜片”收在“讲解与回顾”并默认折叠；四个镜片共用一块简洁结果区，策展交接只在符合 2–6 件显式选择时出现。
- “锁馆与恢复演练”收在“数据与项目”并默认折叠；V15 的真实隔离恢复继续复用这个上下文，以“验真 → 恢复副本 → 销毁副本”解释过程，未展开时不展示口令表单和备份输入。
- “多视角记忆对照”位于单件展品详情和“讲解与回顾”；先显示一张通俗摘要，需要时再展开逐字差异、来源身份边界与时间判断。
- “按意思找回”位于“讲解与回顾”并默认折叠；只有用户点击准备后才加载约 47 MB 模型，界面会先说明下载量、会话边界、停止与释放方式。
- 主题展览、回访、人物档案与“胶囊与分享”都收在“讲解与回顾”已有上下文中，不增加第五项导航。
- 单件“以后怎样再见”位于展品详情，长期意愿管理位于回顾页折叠层；两者只有用户展开后才读取，不把偏好做成常驻打扰。
- 全馆最近版本位于“讲解与回顾”的折叠式“记忆年轮”，只有展开后才读取；单件展品的完整版本链继续收在详情底部，查看旧快照和恢复确认按需出现。
- 乐观并发保护不增加常驻控件；只有编辑页已经过期时才阻止保存并提示刷新，正常单页面使用不受打扰。
- 馆藏体检与备份验真位于“数据与项目”的数据边界卡片内，默认折叠，不在启动时自动扫描私人文件；体检由用户主动开始、可以停止，备份只在用户选择文件后验真。
- 胶囊工作室默认关闭；安全材料读取完成后先进入独立三层隐私编辑台，逐项确认完成才出现口令步骤。
- 桌面端导航留在页首；窄屏改为底部固定的四项水平导航，文字和点击区域居中，并为系统安全区预留空间。
- 相似候选不会自动执行动作，OCR 不会直接写入正文，EXIF 也不会弹出强制填写。
- 指针操作都有键盘后备：图片区域和手动对齐点均可输入 0–1 规范化坐标。

复杂能力被放在用户已经产生意图的地方，默认界面仍保持清爽、留白和低认知负担。

浏览器验收使用 DOM 检查与真实点击 / 输入链路，而不是只看截图。2026-07-20，V14 已在 1265×720、390×844 和 320×700 三档完成 15/15 项 Playwright 门禁与真实目视，覆盖四项导航、固定移动底栏、默认折叠、无横向溢出、四种镜片、锁馆 423、结构演练零正文读取和移动收件箱贴边；页面控制台错误为 0。上述交互属于本地完整可写环境验收；生产只实际核验策展 sample，以及锁馆和结构演练两项正文前拒绝，其余保护边界由本地整库回归覆盖。

## 核心能力

### 1. 图片作为一等记忆对象

- 支持 JPEG、PNG、WebP，拒绝伪造 MIME、SVG、GIF、截断容器和像素炸弹。
- 原图流式进入本机暂存区；浏览器另外生成最长边 1600 px 的展示 WebP 和 480 px 缩略 WebP。
- “保留原图”会保存原文件、展示图和缩略图；“仅保留安全展示图”会在完成处理时删除暂存原图，只留下不含原始元数据的派生图。
- SHA-256 相同且既有全部变体重新通过磁盘完整性校验的内容才复用同一媒体资产，但不会因此合并两段展品或认定它们讲述同一事件。
- 每张图片可记录封面角色、顺序、说明、无障碍文本、拍摄时间和照片背面故事。
- 解除最后一个展品关联后，系统隔离并回收无人引用的媒体文件；刚完成上传但尚未关联的图片有 24 小时宽限，避免并发清理。清空馆藏先隔离媒体，再提交数据库事务；失败会恢复文件，物理删除未完成时会明确提示并在后续启动协调。

### 2. 照片区域成为可核对证据

用户可以在规范展示图上拖拽圈选，也可以用键盘填写 `x / y / width / height`。保存的区域使用 `canonical-preview-v1` 坐标空间，并锚定来源图片 SHA-256。

区域类型包括人物、地点、物件、文字、日期和其他；说明、备注与敏感标记都由用户填写。系统分别表达两层状态：

- `source_verified`：区域仍指向同一份图片来源。
- `user_confirmed`：区域含义由用户亲自确认。

来源完整不代表语义真实。区域证据可以修改和删除，并会进入时光拼图的两侧图片依据。

### 3. 克制的影像智能

| 能力 | 系统做什么 | 用户确认点 | 系统明确不做什么 |
| --- | --- | --- | --- |
| EXIF | 严格解析 JPEG APP1 中的拍摄时间、时区、GPS 和方向，全部先记为建议 | 用户点击后才采用拍摄时间 | 不自动写记忆日期或地点，不联网反查 GPS |
| 相似候选 | 对安全展示图生成确定性的 dHash 检索指纹，返回“可能相似”列表 | 用户自行打开候选展品核对 | 不把感知哈希当作相同证明，不自动合并、删除或修改照片 |
| 文字摘录 | 浏览器支持 `TextDetector` 时，只识别用户圈选的本地图片区域；否则提供手动摘录 | 识别文本先进入可编辑草稿，用户勾选确认后才保存为 `text` 区域证据 | 不调用第三方 OCR，不伪装识别成功，不自动写展品正文 |

GPS 被视为敏感数据。“仅保留安全展示图”不会持久化敏感 GPS 线索；选择“保留原图”时，原文件本身仍可能含元数据，相关位置线索在本地标记为敏感并对普通接口隐藏具体值。完整备份可能包含这些私人数据，必须像原相册一样妥善保管。

### 4. 时光对照，而不是自动认定

时光拼图可以把两个记忆版本的照片并排显示，保留各自说明、替代文字和原图入口。可选的“手动叠影”允许用户：

- 在左图与右图各选择两个对应点，组成两组对应关系。
- 用图片点选或 0–1 数值坐标完成相同操作。
- 调整右图透明度、撤销或重置标点。
- 在浏览器 Canvas 中计算平移、旋转和等比缩放的二维相似变换。

它只帮助目视比较，不是自动配准，也不会判断两张照片是否来自同一地点、人物或事件。只有用户确认后，两个文字版本才会组成同一往事的版本组；原文和原图始终分别保留。

### 5. 可携带的完整记忆

`.time-isle` 是当前版本的主要完整备份格式，采用严格 POSIX ustar + gzip 封装，包含：

- 馆藏文字和记忆考古关系。
- 媒体资产，以及按隐私模式保留的 `original`（可选）、`display` 和 `thumb` 文件。
- 展品与图片的顺序、封面和说明关联。
- EXIF 建议、检索指纹和用户确认的图片区域证据。
- 主题展览、回访状态、线索与人物档案、语音记录及已确认文字稿。
- 时间胶囊的外壳、安全快照和 display WebP 关联。
- 展品正文与结构化字段的完整修订链，以及恢复来源关系。
- 单件展品与事件组的时间校准、用户选择的来源键、来源集合摘要和待复核语义。
- 事件级口述问题、追加式回答状态、人工声音选段、手填文字稿与显式时间含义；声音字节仍由统一声音资产 section 携带。
- 受限策展运行、只读工具回执、提案和逐项决定；恢复后强制失去决定权限。
- 记忆收件箱来源回执、来源护照追加事件链，以及请求绑定的共忆回信与独立来源锚点。

Schema 19 完整 `.time-isle` 必须显式声明既有 `timeline/calibrations.json`、`oral-history/state.json`、`curator-agent/state.json`，以及 `inbox/state.json`、`provenance/state.json`、`co-memory/responses.json`；即使当前没有记录也必须包含空 section，避免“旧版本不支持”和“支持但零记录”被混为一谈。锁状态、salt、digest 与 verifier 不属于馆藏 section，在完整归档中同样排除。

归档中的清单会记录各文件的大小、MIME 和 SHA-256。恢复前，系统先以两遍流式解析核对 gzip / tar 边界，再在隔离暂存区验证路径、清单、哈希、媒体结构、功能 section 和交叉引用；任何损坏或高于当前能力的 schema 都会整批拒绝，不产生业务写入。恢复按记忆/修订/媒体、事件/声音、口述史/时间校准、收件箱/来源护照/共忆回信、依赖共忆的 provenance、展览与策展历史的顺序规划；ID 或引用冲突会回滚整个事务，不留下半套来源。非空口述史 JSON 因不携带声音二进制，仍只验真并返回 `requiresTimeIsle`。

脱敏 `.time-isle` 只保留允许公开的清单与固定计数 section；图片、声音、口述问答、共忆问答/身份标签、来源摘录/锚点、内部 ID 与哈希在归档层物理缺席。共忆只保留响应、未验证身份、加密传输与未签名四项计数；锁状态和 verifier 在两种模式都缺席。脱敏主馆藏仍含可读摘要，不能自动视为匿名。

### 6. 时间胶囊：外壳可见，内容到期

时间胶囊只允许从状态为 `published`、无需复核的主题展览创建。创建时生成不可被来源展览后续编辑静默改写的安全快照，并把三类数据分开保存：

- 外壳：标题、寄语、开启日期、时区和可用状态。
- 内容：匿名章节、展项、已核验原文引用和用户选择的已确认文字稿。
- 媒体关联：只指向用户明确选择、已经就绪的 display WebP。

开启日前，胶囊列表、详情和内容接口都只使用外壳。`GET /api/capsules/:id/content` 返回 `423`、稳定错误码和公开外壳，不读取内容快照，也不下发正文、图片、展品成员或数据库内部 ID。若来源图片后来被删除，胶囊会进入需要复核状态，而不是继续展示失去来源保证的材料。

开启日期使用用户选择的 IANA 时区解释，解决“哪一天算到期”的产品语义；它仍只是本地仪式门槛。能够修改本机时间或直接读取私人数据库的人不受它提供的保密承诺约束。

### 7. 浏览器端加密的离线展览

离线分享与胶囊共用“只从已确认展览开始”的安全来源，但两者职责不同：胶囊负责封存和到期体验，离线文件负责把内容加密后交给别人。

1. 服务端只列出来源展览中的 display WebP 与已确认文字稿；带入浏览器只表示成为候选，不表示同意分享。
2. 第一层核对无需口令即可看到的公开标题、说明和文件名。它们使用通用默认值，不复制来源展览/胶囊标题或导出时间，并固定为立即可打开。
3. 第二层填写密文内的受众和用途，编辑仅属于本次分享的叙事副本，再逐章、逐件、逐条选择引用、confirmed 文字稿与 display WebP。所有选择默认关闭，修改不会回写来源展览。
4. 第三层显示公开内容、解密后内容数量、固定排除项和“下载后无法撤回；知道口令的人仍可以复制、转发或截图”的边界；用户最终勾选确认后才出现口令步骤。
5. 从口令步骤开始不再发送网络请求。口令只在当前页面内存中参与 PBKDF2-SHA-256（310,000 次）和 AES-256-GCM 加密，浏览器随后下载自包含 `.html`。
6. 收件人输入正确口令即可解密；阅读页显示受众、用途、精确计数与不可撤回边界，并通过 Blob URL 在本机还原 display WebP，断网也能使用。

每个文件使用随机 16-byte salt、12-byte IV，公开外壳作为 AAD 的一部分参与完整性校验。V2 密文严格校验精确字段、连续 `section-N / item-N / media-N`、媒体所有权、Base64 字节数和收据计数；至少需要 1 章、1 件展品和 1 条引用或文字稿。V1 历史文件继续可解密阅读。口令不上传、不持久化、不写入文件，也无法找回。

投影结果物理排除原图、EXIF/GPS、草稿文字稿、Agent 日志、数据库内部 ID、URL、SHA、未选原文与原始音频；图片只允许安全 display WebP。编辑会撤销确认，加密过程中重新确认会使旧下载任务失效；取消、关闭、返回书架和下载后都会清空草稿、明文媒体、口令与 Blob URL。

### 8. 可安装但不缓存私人馆藏的 PWA

V7.1 为时屿补齐可安装外壳，但不把 PWA 等同于“离线馆藏”。安装能力沿用原有四项主导航，不新增第五个入口；“安装到此设备”位于“数据与项目”的项目卡中，并按浏览器能力渐进披露。

- Chromium 浏览器只有在收到 `beforeinstallprompt` 后才显示安装按钮；iPhone 与 iPad 不伪装成一键安装，而是给出 Safari“添加到主屏幕”的手动步骤。
- Service Worker 只在 HTTPS 或浏览器认可的本地安全上下文注册；非安全来源、不支持的浏览器和不满足安装条件的环境会静默隐藏入口，不阻塞记录、检索和策展主流程。
- 缓存采用精确白名单，只包含 `/offline.html`、`/pwa.css` 和公开品牌 SVG。首页、主应用脚本、API、图片、声音、归档与任何用户内容都不进入 Cache Storage，也不会在运行时动态写入缓存。
- 原图、展示图和缩略图等所有图片变体统一返回 `private, no-store`。断网导航只展示不含馆藏、照片、声音或导出内容的隐私边界页，不用空列表伪装成“馆藏为空”，恢复连接后再返回真实应用。

本地模式仍是完整媒体保存、胶囊封存和归档恢复的主要体验环境；公开 Demo 继续使用临时数据、强制 Mock 与禁写边界。PWA 安装不会改变两种运行模式的数据能力，也不会让公开 Demo 变成私人内容或长期数据的存储空间。加密分享文件可以在用户下载后独立断网解密阅读，这是自包含密文文件的能力，不代表时屿应用缓存了私人馆藏。

### 9. 可校验、可恢复的记忆年轮

Schema 10 为每件展品增加连续修订链。新建或导入展品会生成第一个版本；旧馆藏第一次发生有效编辑时，系统在同一事务内先补记编辑前基线，再追加编辑后的版本。只改变时间戳或提交相同语义内容不会制造伪修订。

- 快照覆盖原文、展签、标题、展厅、人物、标签、情绪、日期、地点、重要度、收藏状态、附件说明等展品正文与结构化字段，并按固定键序列规范化后计算 SHA-256。
- 每条修订保存连续序号、当前快照哈希和前一版快照哈希；恢复记录还必须引用同一件展品链中的旧版本。馆藏体检与完整备份恢复都会重新核对序号、父链、快照哈希和当前 head。
- 列表只读取版本摘要；用户主动点击“查看”后才读取那一版快照。恢复前再次确认，执行时把旧快照复制成新的 `restored` head，当前版本和全部中间历史仍然保留。
- 编辑与恢复必须携带当前 `ETag / If-Match` 或等价版本条件。缺少条件返回 `428`；旧页面与当前版本不匹配时返回 `412` 和新的 ETag，但不在冲突响应中下发较新的私人正文。
- 正文写入与修订追加位于同一 SQLite 事务。完整修订账本按可导出的 JSON 合同设置 20 MiB 总预算；新增、编辑、恢复或整批导入一旦越界，会整项回滚而不留下半条历史。

年轮不保存图片或声音文件、媒体关联、图片区域证据、展览、胶囊和 Agent 运行日志，恢复旧版不会把这些独立对象一起倒退。SHA-256 父链用于发现链内缺口或不一致，不提供外部签名、可信时间戳、作者身份或记忆真实性证明；拥有本机数据库控制权的人也不受它提供的不可篡改承诺约束。当前并发策略负责拒绝静默覆盖，不提供多人协同合并。

完整 `.time-isle` 与完整 JSON 会携带年轮并在恢复时安全重映射冲突 ID；脱敏导出只保留展品数、修订数和固定隐私说明，旧版正文、结构化字段、编辑备注、哈希、精确时间及修订内部 ID 物理缺席。

### 10. 只读馆藏体检、备份验真与流式归档

馆藏体检只在本地由用户主动启动，并串行核对三类信息：

- 数据库：SQLite `quick_check`、外键、schema 与迁移账本、FTS 文档成员关系，以及修订链和当前 head。
- 私人媒体：图片记录、隐私模式、必需变体、内容锚点和磁盘文件完整性；声音记录与对应音频完整性。
- 待复核事项：来源失效的引用、需要复核的展览或胶囊、开放问题和草稿文字稿。

体检异步显示阶段与进度，可以停止；结果只保留在当前服务运行内，问题最多展示前 200 项。它只报告，不自动删除、修复或改写内容，“本次未发现结构性异常”也不等于证明每段记忆的语义真实。

“选择 `.time-isle` 验真”会复用真正恢复前的完整解包、manifest、哈希、媒体格式、section 与交叉引用校验，但只在随机隔离暂存区工作；成功后返回模式、schema、条目体积和主要对象计数，失败或完成都会删除本次暂存，不写入业务数据库或正式媒体目录。验真通过代表这份文件满足当前版本的恢复合同，不替代用户保管多份备份，也不保证未来所有软件都能读取。

V7.2 将 `.time-isle` 导出改为文件与 HTTP 流式路径。服务端以私有临时文件承接流式 gzip；支持 File System Access API 的浏览器把响应直接写入用户选择的文件，不支持时交给浏览器原生下载，不在前端 JavaScript 中聚合整份归档。恢复端先完整扫描 gzip / tar 结构和硬上限，再把通过边界检查的条目写入隔离区；请求中止或失败会清理本次输出与暂存。

安全约束保持显式：

- 脱敏归档的每个功能 section 都采用精确字段集合和固定隐私说明；修订只留下计数摘要。主馆藏摘要仍可能保留标题、短展签、标签和情绪，因此“脱敏”不等于匿名，公开前仍需人工核对。
- 旧 JSON 兼容导入拥有独立的 64 MiB HTTP 请求上限，以容纳受 20 MiB 预算约束的修订合同；它仍受对象数量、字段、引用和 schema 校验，不是无限导入。
- 本机 SQLite、JSON 导入和 `.time-isle` 验真/恢复都会拒绝高于当前程序能力的 future schema，提示升级应用，不会猜测性忽略未知数据。
- 启动维护只清理 `.exports`、`.restore`、`.inspect` 下超过一小时且名称符合程序规则的崩溃残留；真实目录与安全路径会先核对，较新的任务和无关文件不会被当作垃圾删除。
- 流式处理降低内存聚合风险，但仍需要足够的本机临时磁盘空间，也不把本机事务夸大成跨设备分布式事务。

### 11. 明确回访意愿，而不是行为推断

每件展品都可以在详情中设置“以后怎样再见”：

- `neutral`（自然回访）不保存长期记录，沿用往年今日、很久没见和随机漫游的原有规则。
- `welcome`（欢迎主动出现）只在当前回访类型的合格候选中优先，并在依据中标记来源为 `user-confirmed`。
- `later`（指定日期以后）在用户选择的本地日期到达前过滤主动回访；日期与 IANA 时区一起保存，跨时区行为可测试。
- `pause`（暂停主动出现）过滤所有主动回访，直到用户亲自恢复自然回访。

这项意愿不保存原因，也不会从浏览、跳过、搜索、人物关系或情绪字段自动产生。它不隐藏或删除展品，不影响搜索、详情和归档读取。单件设置与全局管理都默认折叠、按需请求；写入要求 `confirm: true`，公开 Demo 返回 403。

Schema 11 用带 CHECK、外键级联和 500 条硬上限的独立表保存 `welcome / later / pause`。完整 JSON 与 `.time-isle` 必须显式包含回访意愿，恢复时复用展品 ID 映射并在同一事务完成；脱敏归档只保留总数和固定说明，展品 ID、选择、延期日期、时区和精确时间物理缺席。分享草稿不写数据库，也不进入归档。

### 12. 不确定时间线与来源校准台

Schema 12 为单件展品或人工确认的事件组保存一份独立时间判断。面向用户的校准台只提供四种清楚选择：

- 一个日期：为本次时间线展示选择精确日；该日必须落在每个已勾选来源的区间内。
- 一个范围：保存包含起止日的时间区间；该范围必须与每个已勾选来源区间至少有一天相交。
- 保留多种记录：承认多个来源各自成立，不强行合并为一个日期；所选来源中必须至少存在一对互不相交的日期区间，避免把本可兼容的记录伪装成冲突。
- 仍不确定：明确保存“目前无法判断”，而不是让空值看起来像系统遗漏。

校准台只接收四类受控来源，并把候选值、来源类型和可复核锚点并列呈现：

- 展品当前日期：反映现在可见的结构化日期，但不因此拥有更高真实性。
- 记忆年轮中的历史修订日期：以规范快照 SHA-256 和修订序号锚定。
- 原文日期 claim：必须仍能由原文摘录与字符区间复核，失效、拒绝或撤回的 claim 不进入候选。
- 关联照片的非敏感 EXIF `captured_at`：只接受已建议或已确认的有效日期；GPS 从时间校准链路严格排除。

每个候选生成内容稳定的 `sourceKey`，整组候选生成 `sourceSetSha256`。保存时记录用户勾选的来源键、选择类型、可选范围与备注，并保存一份只包含来源键、来源类型、精度与起止日的最小不可变来源快照；快照不携带标题、摘录、媒体地址、GPS 或整段原文，使来源后来失效时旧判断仍可解释。系统不会把校准结果写回原文、展品日期、修订或 EXIF。来源集合后来发生变化时，旧判断与这份快照完整保留但派生为 `needsReview`，时间线、讲解与回顾只提示待复核，不会自动采用新来源或删除旧选择。事件级判断优先用于该事件的时间线展示，成员级判断仍保留且可单独核对。

读取校准台与不确定时间线保持只读；公共时间线摘要严格只返回重新打开目标所需的最小目标边界、判断摘要、待复核状态和事件成员 ID，不下发原文、媒体、来源快照、备注或来源哈希。保存、更新和恢复未校准状态都需要显式确认。所有写入使用 `ETag / If-Match`，缺少条件返回 `428`，过期页面返回 `412`；公开 Demo 可以查看预置的“保留多种记录”示例，但任何校准 `PUT / DELETE` 都返回 `403`。

### 13. 一问一答口述史与声音时间段

V9 只对“已确认事件 + 尚未解决 + 双侧都有可验证日期依据 + 至少一对来源区间互不相交”的情况生成一个问题。问题由规则根据当前来源集合确定性产生，不调用模型自由发挥；它保存来源快照、稳定问题键与 `questionSetSha256`，让来源改变后的旧页面无法继续提交过时答案。

用户完成回答必须经过清楚的人工链路：

1. 明确选择当前日期问题。
2. 录制单段 WebM，或选择本地 WebM / M4A。
3. 用两个原生范围控件设置开始/结束毫秒，也可把当前播放位置设为边界并只试听选段。
4. 手工填写文字稿；系统不自动转写，也不识别说话人或判断情绪。
5. 明确选择时间含义：`day / range / uncertain`，并确认文字稿是自己核对后的表述。
6. 保存为独立事件级来源，而不是写回某件成员展品。

只有 `confirmed + day/range` 会生成 `oral-history` 时间候选；`confirmed + uncertain` 保留声音与文字证据但不制造结构化日期，`draft` 从不进入候选。回答状态采用 `draft / confirmed / superseded / withdrawn` 追加式历史：新确认回答替代旧回答的当前地位，旧记录默认折叠保留；撤回停止当前来源，但不抹去它曾经存在。回答变化只会让引用旧口述来源的时间校准进入 `needsReview`，不会自动调整判断或原日期。

写入同时要求 `ETag / If-Match`、当前 `questionSetSha256` 和稳定 `submissionId`；缺少条件、来源已变化或页面过期会失败，网络重试可幂等返回同一次结果。口述回答对声音资产形成正式引用，删除与 GC 必须计入这份使用量。公开 Demo 只展示问题与已有来源，麦克风、音频文件选择、PUT 与 DELETE 均关闭或返回 403，并且不伪造可播放音频。

### 14. 受限本地策展助手与逐项决定

V10 的助手不是一个拥有任意工具的后台 Agent，也不是外部大模型入口。用户可以明确选择 2–6 件展品或给出查询；执行器只允许四项本地只读工具：搜索记忆摘要、读取记忆证据、读取已确认关系、读取现有展览摘要。它不能访问网络、文件系统、任意 SQL，也没有写入、删除、发布或分享工具。

每次运行固定最多 6 个步骤、4 次工具调用、2 秒、262,144 字节结果和 6 件来源。界面展示预算、步骤回执、引用与重放评测；运行可取消，进程重启时遗留任务转为 `interrupted` 并禁止续跑。输出只有绑定 `requestSha256 / sourceSetSha256 / proposalSha256` 的未保存 `proposal / preview`。来源后来变化时，旧提案保留但进入 `needsReview`，不能继续决定。

真正的写入被拆成三个互不继承授权的按钮：

1. `save_exhibition`：只把当前提案保存为草稿。
2. `confirm_relationship`：只确认这次提案中的候选关系；没有候选时不可批准。
3. `publish_exhibition`：只发布已经由第一步保存的草稿。

每项都要求 `confirm: true`、当前 `ETag / If-Match` 和稳定 `Idempotency-Key`；批准与拒绝都留下追加式决定回执。分享不属于动作列表，发布后只交接既有三层隐私编辑台，仍默认不选择任何私人材料。

Schema 14 以四张独立表保存运行、步骤、提案和决定。完整归档可迁移这些审计记录，但恢复后强制标记为历史、待复核且禁止决定，不能把旧批准重新执行；脱敏归档只留下安全计数。公开 Demo 的 sample 由 API 合成、零持久化，全部非 GET/HEAD 请求在读取请求体前返回 403。

### 15. 共忆信笺与独立见证

V12 只做一问一答的最小协作。馆主从一件记忆发起问题，浏览器将邀请内容封装为可断网打开的 HTML；对方在本地填写回答，再导出与原请求 SHA-256 绑定的加密 JSON。两端都使用 PBKDF2-SHA-256（310,000 次）和 AES-256-GCM，口令不上传、不持久化，文件也不会自动发送。

回信的身份标签只能是自述，`verified` 固定为 `false`，`signed` 固定为 `false`。馆主解密后必须核对问题、目标记忆、回答和边界，再明确确认保存；系统不会把回信合并进原文，也不会据此推断人物、关系、日期、说话人、情绪或事实。馆外文件是密文，但确认入馆后的问答与来源锚点会成为本机 SQLite 的普通明文结构。

完整归档保存共忆问答、请求/回答绑定和来源锚点，恢复按依赖顺序重写 ID；脱敏归档只留下四项安全计数。若目标记忆冲突、请求摘要不一致、引用失效或 provenance 建立失败，整包事务回滚，不留下只有回信没有来源的半恢复。

### 16. 设备内可解释镜片

V13 提供时间、共同出现、证据和线索四种只读投影。用户必须明确选择 2–20 件已保存展品；客户端只发送 ID 和镜片参数，服务端按 ID 重新读取当前馆藏，避免把客户端上传的正文副本当成来源。结果会说明排序字段、命中理由、来源引用和固定边界。

镜片执行器是确定性的本地规则：外部模型为 0、工具调用为 0、持久化为 0。它不从标题或正文补日期，不把共同出现解释成人际关系，不把证据数量包装成真实性，也不扩展查询词来制造语义命中。用户可将明确选择的 2–6 件展品形成一次内存简报交给策展助手；7–20 件时必须重新选择，不能静默截断，更不会自动运行、保存或发布。

### 17. 锁馆与结构恢复演练

V14 的锁馆是应用层写入门禁。开始锁馆时，运行时先阻止新的 mutation 进入，再等待已开始的写请求和后台媒体维护结束；锁定后，新的写请求会在读取 Content-Type 和正文前返回 `423`，而 GET、导出、归档只读验真与结构恢复演练保持可用。锁状态使用 schema 19 单例和 revision CAS；缺失、损坏或正在切换时按 fail-closed 处理。

口令只用于派生 recovery verifier，应用不保存明文口令；公开状态也不返回盐或 digest。verifier 属于敏感认证材料，不进入日志、普通 JSON 或 `.time-isle`。但锁馆不会加密 SQLite、图片、声音或磁盘，也不阻止拥有设备和数据文件的人绕过应用读取明文，因此不能称为“整馆静态加密”。

结构演练只接受完整 `.time-isle`，在隔离暂存区复用正式恢复前的结构预检，并额外核对 manifest、逐项哈希与核心引用。它不写当前馆藏、不执行隔离恢复，结果固定说明 `actualRestorePerformed=false`、`isolatedRestorePerformed=false`、`disasterRecoveryProven=false` 和 `diskEncryptionProvided=false`；产品只显示“结构验真通过”，不显示“恢复成功”。

### 18. 一次性真实隔离恢复

V15 在结构验真之外新增真正的恢复副本。系统为每次演练创建全新的临时 SQLite、图片目录和声音目录，复用正式恢复合同写入完整 `.time-isle`；当前馆藏的数据库、媒体路径和存储能力不会注入演练运行时。

恢复后由独立读取路径核对 SQLite `quick_check`、外键、迁移账本、FTS、展品关系，以及图片和声音数量、引用与 SHA-256。随后关闭副本数据库并确认目录已不存在，才返回“通过一次性恢复”回执。取消、损坏归档、并发运行、预算越界和清理失败都有永久门禁；若旧副本未确认销毁，系统不会继续读取下一份备份或创建新副本。

这项能力仍不证明跨设备灾备、RTO/RPO、进程隔离、生产环境恢复或磁盘加密。回执只给出安全计数和阶段状态，不返回标题、正文、内部 ID、文件路径、单文件哈希或 ID 映射。

### 19. 多视角记忆对照

V16 的对照是 GET-only 派生视图，不增加数据库表，也不把已有内容复制成新的“AI 结论”。它并列四类现有来源：当前记录、同一馆主的编辑年轮、已确认入馆的共忆回信，以及来源护照和时间判断。

只有用户已经确认的 `supports / supplements / different_record` 关系会进入对照主张；未关联来源护照的回信会显示“尚未建立对照关系”，来源变化后进入 `needsReview`，撤回后不再形成当前关系。亲友身份始终是自述、未核验。系统不计算可信度，不判断谁更准确，也不把字面差异自动解释为矛盾。

### 20. 设备内中文语义找回

V17 将原有搜索准确更名为“字段与线索检索”，并新增独立的“按意思找回”。用户点击准备后，浏览器从同源加载固定版本的 Transformers.js 3.8.1、ORT WASM 和 `Xenova/bge-small-zh-v1.5` q8，在 Module Worker 内生成 512 维 embedding；服务端只提供标题、展签、原文、标签和已确认文字稿的有界只读快照。

模型和运行库总计约 47 MB，最多索引 500 件展品，并有 1 MiB 聚合快照预算。查询、向量和索引只保留在本页 Worker 内存，刷新、停止或释放后需要重新准备；模型静态资源可能仍留在普通 HTTP 缓存，因此“释放”只承诺清理本页运行内存。馆藏变化会让旧索引失效，避免用过期快照继续显示成功。

真实模型参与和同源资源哈希都进入永久门禁；测试还阻止第三方请求，并核对 localStorage、sessionStorage、IndexedDB 与 Cache Storage 不因准备或释放而新增内容。相似度只是模型生成的文本排序，不是事实、关系、情绪、真实性或概率判断，embedding 也不能称为匿名数据。

## 产品原则

- 原文与原图优先：AI 草稿不覆盖原文，派生视图不冒充原图。
- 人工确认：AI 整理、EXIF、相似候选、OCR 和事件关系都有明确确认边界。
- 明确意愿优先：长期回访只听从用户确认，不用浏览行为推断心理；恢复自然回访会真正删除长期偏好。
- 证据可回看：文字证据保留原文字符位置，图片证据保留来源哈希与规范化区域。
- 本地优先：SQLite 与内容寻址媒体目录默认位于本机，不要求外部数据库或对象存储。
- 版本并存：同一往事的多次文字与图片记录分别保留，系统不制造唯一真相。
- 来源并列：日期冲突先展示受控来源和可复核锚点，再保存人的判断；`needsReview` 只表示来源集合发生变化，不表示旧记忆错误。
- 时间判断独立：一个日期、一个范围、多种记录与仍不确定都不会回写原文或展品日期，也不被包装成事实裁决。
- 口述不代述：声音选段、文字稿和时间含义都由用户亲自确认；系统不自动转写、识别说话人、判断情绪或从声音补造日期。
- 助手不越权：本地策展执行只读、定额、可取消、可重放；提案不是保存，单项批准不代表批量授权，发布也不等于同意分享。
- 见证不冒认：共忆回信即使经过加密也仍是自述、未验证且未签名；只有用户明确确认后才成为独立来源，不合并原记忆。
- 镜片不下结论：设备内投影只重排已保存字段与已确认来源，零模型、零工具、零保存；共同出现、证据数量和线索命中都不等于事实。
- 锁馆不冒充加密：应用层门禁只阻止新的写请求；本机 SQLite 与媒体仍是明文，必须依赖设备和系统保护。
- 演练不冒充恢复：结构通过只说明当前规则能核对归档结构、哈希和引用，不代表已执行恢复或已经具备灾难恢复能力。
- 恢复不冒充容灾：V15 会真实恢复一次性副本，但只证明当前机器、当前版本通过；副本必须在返回成功前销毁。
- 对照不裁决：V16 只并列已有记录与人工确认关系，未核验身份和待复核来源保持可见，不输出可信度。
- 相似不等于事实：V17 的 embedding 只用于设备内文字排序，最接近的结果也可能不相关，向量不能被包装成匿名数据。
- 证据可撤回但历史不抹除：重新回答与撤回通过追加状态改变当前来源，旧回答默认折叠保留，不覆盖原始记忆。
- 历史追加：恢复旧版生成新的 head，不删除当前版本；过期编辑先报冲突，再由用户重新核对。
- 缺失不是矛盾：只有两侧都有可验证信息时才展示描述差异。
- 日期与保密分工：开启日期提供仪式门槛，浏览器端加密才承担离线分享的保密职责。
- 最小分享：带入编辑台不等于同意分享；章节、展品、引用、确认文字稿和安全展示图默认均不选，未选择等于物理缺席。
- 分享副本独立：公开外壳与加密叙事只属于本次文件，不复制不必要来源元数据，也不回写来源展览。
- 失败安全：上传、归档和恢复在失败时清理暂存或补偿已移动文件。
- 先验真后写入：馆藏体检保持只读，备份通过当前恢复合同后才允许进入正式事务。
- Demo 隔离：公开展示不接收私人图片，并阻止破坏示例稳定性的写操作。

- 默认折叠的不确定时间线摘要、来源集合变化后的 `needsReview` 待复核，以及事件成员最小边界下的拼图重开入口。
- 事件级一问一答口述史：单段录音或本地 WebM/M4A、人工毫秒选段、手填文字稿、显式 day/range/uncertain，以及 draft/confirmed/superseded/withdrawn 追加状态。
- `ETag / If-Match + questionSetSha256 + submissionId` 并发与幂等保护；已确认 day/range 生成独立口述时间来源，uncertain 只保存证据，回答变化使旧校准进入 `needsReview`。
- 固定四项只读工具、固定预算、可取消/中断、带来源引用的未保存策展提案，以及可回放的运行/步骤/提案审计。
- 保存草稿、确认候选关系和发布三项独立人工决定；`ETag / If-Match + Idempotency-Key + confirm` 防止过期或重复副作用，分享继续由三层隐私编辑台独立确认。
- 用户确认后才保存的主题展览，以及一次只呈现一件记忆的回访。
- 由用户明确确认的欢迎、延期、暂停与自然回访恢复，以及按需读取的长期意愿管理。
- 可解释字段与线索检索、人物/地点/主题档案与人工确认的别名关系。
- 浏览器录音或音频上传、独立音频存储、回放与人工确认文字稿。
- 只来源于已确认展览的时间胶囊；未到期只返回外壳，到期后读取安全快照。
- 三层隐私编辑、V2 精确收据、PBKDF2-SHA-256 + AES-256-GCM 加密、自包含且断网可读的 HTML 分享文件，并兼容 V1 阅读。
- 不缓存私人馆藏的可安装 PWA 外壳，以及断网时不读取用户内容的隐私边界页。
- SHA-256 快照父链、旧版预览与追加式恢复，以及 `If-Match` 乐观并发保护。
- 本地只读馆藏体检与不写入业务数据的 `.time-isle` 备份验真。
- UTF-8 TXT/Markdown 的浏览器临时读取、服务端文件/文本/片段重验、UTF-16 锚点，以及来源片段与新展品的同事务人工入馆；整份源文件不落库。
- 来源护照的不可变主张、支持/补充/不同记录三类人工关系、追加确认/撤回事件和来源变化 `needsReview`；不生成真假分数。
- 共忆信笺的离线邀请/回信加密、请求摘要绑定、自述未验证身份、明确入馆确认，以及完整/脱敏归档和冲突整包回滚。
- 时间、共同出现、证据、线索四种设备内确定性镜片；按 ID 重读 2–20 件馆藏，零模型、零工具、零保存，并只允许 2–6 件显式交接策展简报。
- schema 19 锁馆单例、revision CAS、派生 verifier、正文前 423 写门禁、在途写入/维护协调，以及锁馆期间仍可用的只读导出与验真。
- 只核对完整 `.time-isle` manifest、哈希和核心引用的结构恢复演练；不写当前馆藏、不执行隔离恢复、不提供灾难恢复或磁盘加密证明。
- 把完整 `.time-isle` 真正写入一次性 SQLite、图片和声音副本，独立体检后销毁的 V15 隔离恢复；不替换当前馆藏，也不提供异机灾备承诺。
- 当前记录、编辑年轮、已确认共忆、人工来源关系与时间判断的 V16 多视角只读对照；未绑定、已撤回和待复核状态不会被伪装成有效结论。
- 用户主动启用、同源自托管、Worker 会话内运行的 V17 中文 embedding；查询、短文本、索引和向量不发送到第三方，也不持久化为向量数据库。
- schema 19 `.time-isle` 完整/严格脱敏流式归档、共忆与 provenance 依赖恢复、口述史事件/声音 ID 映射与引用保护、回访意愿/时间校准事务恢复，以及受限策展四表 ID 重写和历史授权隔离；锁状态与 verifier 明确排除在普通归档外。
- 本地模式和受保护的公开 Demo 模式。

不在当前范围：

- 多用户账号、细粒度权限和跨设备端到端加密身份体系。
- 云端对象存储、云数据库和跨设备实时同步。
- 第三方云 OCR、自动图片理解或自动生成照片事实。
- 自动语音转写、说话人识别、情绪判断或根据音色/语气推断时间与事实。
- GPS 反向地理编码或自动填写地点。
- 基于相似度自动合并、删除照片或合并记忆事件。
- 视频和动态图片存储；离线分享首版也不携带原始音频二进制。
- 公网私密链接、账号托管、分享撤回、远程失效和口令找回。
- 外部签名的不可篡改审计账本、可信时间戳、多人实时编辑或自动冲突合并。
- 共忆回信的实名认证、数字签名、可信时间戳、自动发送、账号协作或自动合并原记忆。
- 外部语义服务、云端向量库或跨会话持久向量索引；V17 只保留当前页面 Worker 内存中的临时 embedding。
- SQLite、图片、声音或整块磁盘的静态加密；V18 总体 NO-GO，当前锁馆仍不是防止设备持有者读取数据的安全边界。
- 自动恢复、异机/跨设备容灾、RTO/RPO 或生产灾难恢复承诺；V15 只证明当前机器、当前版本的一次性副本能够恢复并销毁。
- 馆藏异常的自动修复、自动删除或无人确认的数据迁移。
- 自动替用户发布、选择分享内容或训练私人记忆。
- 后台自治策展、自动确认关系、自动发布或自动分享口述与媒体内容。
- 根据来源数量、EXIF 或模型置信度自动决定唯一日期，或用校准结果覆盖原文、展品日期和旧修订。

## 公开 Demo 保护

截至 2026-07-20，公开网址已经部署 V14.0.0 / schema 19，生产环境启用 `INTERVIEW_DEMO=true`，接口已核验为 `interview-demo / ephemeral-sqlite / mock-fallback`。GitHub 与 Gitee 的 `main` 已对齐到同一 V14 发布线；V10.0.0 / schema 14 仍作为 2026-07-19 的历史发布基线记录，不再代表当前线上版本。

V15–V17 目前只存在于本地 V17 候选工作区，尚未提交、推送或部署。公开网址不会展示一次性真实隔离恢复、多视角对照或“按意思找回”，面试时不得把本地验收结果说成线上生产结果。

V14 线上提供受保护的共享临时 Demo，不是私人记忆托管环境。核心纯文本、整理运行、时光拼图与补问只能在事务硬上限内临时体验；共忆写入、锁馆切换、结构演练上传与其他高风险 mutation 保持只读或禁用。针对锁馆和结构恢复演练分别发送虚构 `text/plain` 探针，生产接口均在读取正文前返回 `403`、`MUSEUM_LOCK_DEMO_READ_ONLY` 且 `bodyBytesRead=0`；核验前后的 stats、lock 与 runs 均不变。策展助手 sample 明确标记 `synthetic=true / demo=true`，调用前后持久化 runs 均为 0。完整可写体验只在隔离本地环境进行。

该模式会把 SQLite 和媒体目录切换到系统临时位置，冷启动注入四件示例展品和 1 场可分享的已确认示例展览。Demo 可以浏览馆藏、使用讲解、记忆航线与三层分享预览，但：

- 全部媒体写操作返回 403，包括上传、指纹写入、关联和图片区域证据增删改。
- 麦克风保持关闭，全部音频写入返回 403；时间胶囊也不在共享临时实例中持久化。
- 口述史只读展示符合条件的问题与已有来源；不会触发麦克风或文件选择，PUT / DELETE 返回 403，也不伪造可播放回答。
- 策展助手只展示 API 合成的只读 sample，不读取或写入运行表；所有 POST / DELETE 在解析请求体、ETag 和幂等键之前返回 403，也不会调用外部模型。
- 禁止删除、清空、导入和完整归档恢复。
- 预置展品禁止修改；示例年轮可以只读查看，编辑和历史恢复均返回 403。
- 单件回访意愿与长期意愿管理只展示交互，保存、清除和回访浏览状态写入均返回 403。
- 预置的不确定时间线和“保留多种记录”判断可只读查看；校准新增、修改和恢复未校准状态统一返回 403，不能改写共享示例。
- 本机馆藏体检和备份验真入口保持禁用，共享临时服务不接收用户的私人备份文件。
- Demo 在代码层强制使用本地 Mock；即使环境误配 AI Key，也不会调用外部模型。
- 可新增的共享文本、整理运行、时光拼图与补问分别受 SQLite 事务硬上限保护，并发请求达到上限后返回 429。
- 临时新增的纯文字示例可能被同一实例的其他访客看到，并会随实例重建消失。
- 已播种的确认展览允许演示三层隐私编辑与浏览器本地加密；口令仍不进入服务端，下载结果只应使用无隐私风险的示例数据。

公开 Demo 不是私人会话空间。它不部署开发者的本地馆藏，代码也不会在 Demo 模式使用真实模型密钥；访客不应提交私人内容。

## 面试展示建议

公开 V14 继续承担稳定的 60 秒主线；若展示本地候选能力，应切换到隔离的 V17 工作区，并始终使用可丢弃的虚构数据：

1. 用虚构文字和一张无隐私风险的照片创建展品，简述原图/安全图、人工确认与四项主导航。
2. 编辑同一展品，再从详情展开“记忆年轮”，按需查看旧快照；恢复早期版本后展示它成为新的 head，原有版本没有消失。
3. 用两个页面演示过期 `If-Match` 被 `412` 拒绝，说明它避免静默覆盖，但不是多人实时协同或自动合并。
4. 在“数据与项目”主动运行馆藏体检，展示数据库、图片、声音和待核对事项；强调只读、可停止且不自动修复。
5. 选择一份测试 `.time-isle` 先做备份验真，再分别导出完整与脱敏归档；说明流式路径、20 MiB 修订预算、64 MiB JSON 兼容导入及 future schema 拒绝。
6. 圈选一处图片证据并进入时光拼图，说明来源哈希、用户确认和手动叠影各自只解决什么问题。
7. 为一件展品设置“指定日期以后”或“暂停主动出现”，再恢复自然回访；强调明确意愿而不是行为推断。
8. 从已确认主题展览进入三层编辑台，逐项选择一张 display WebP 和一份 confirmed 文字稿，核对收据后生成自包含 HTML并断网解密。
9. 打开一组已确认时光拼图，展开来源校准台，对照当前日期、修订、原文日期锚点和照片 EXIF；选择“保留多种记录”，再说明来源变化为何只产生 `needsReview`，而不会改写两段原文或日期。
10. 在同一拼图展开一问一答口述史，用隔离测试数据录制/选择单段音频、手工划定时间段、填写文字稿并明确选择 `uncertain` 或 day/range；指出只有已确认 day/range 才成为时间来源，重答或撤回保留旧历史。
11. 打开受限策展助手，选择 2–6 件虚构展品并查看四项只读工具回执、固定预算与未保存提案；分别演示保存草稿、拒绝/确认关系和发布的独立决定，再说明分享仍需进入三层编辑台。
12. 为一件虚构记忆生成共忆邀请，打开离线页并导出回信，再回到馆主侧核对“身份未验证、文件未签名、不会自动合并”，最后明确确认入馆。
13. 在“讲解与回顾”依次运行时间与证据镜片，展示零模型/零工具/零保存回执；选择 2–6 件交给策展助手，再用 7 件来源说明系统要求重选而不是静默截断。
14. 在“数据与项目”锁馆，验证新写入在正文前返回 423，而导出、归档验真和结构演练仍可用；最后指出锁馆不是磁盘加密，结构演练也没有执行恢复。
15. 选择一份虚构完整 `.time-isle` 运行 V15 一次性恢复，按阶段展示恢复副本、独立体检和销毁回执；强调当前馆藏没有被替换，也不承诺异机灾备。
16. 从单件展品打开 V16 多视角对照，比较当前记录、编辑年轮与已确认回信；用未绑定或待复核示例说明系统不判断谁记得更准。
17. 在“讲解与回顾”主动准备 V17“按意思找回”，说明约 47 MB 同源模型、会话内 Worker 和 500 件上限；用自然描述找回展品后释放本页内存，并提醒最接近也可能不相关。

## 已知限制

- `TextDetector` 不是所有浏览器的标准可用能力；不可用或启动失败时，产品只提供手动摘录，不会降级到远程 OCR。
- EXIF 当前只严格读取 JPEG APP1/Exif 的有限字段；无时区的拍摄时间保持 floating local time，不会伪造 UTC。
- 感知相似度适合召回人工候选，不是身份、地点或同一事件证明；完全相同 SHA-256 只用于资产复用。
- 手动叠影依赖用户给出两组可靠对应点，只计算平移、旋转和等比缩放，不处理透视、非刚性变形或自动特征匹配。
- 图片区域的 `source_verified` 只证明来源锚点仍匹配，不证明标签内容为真。
- Mock 模式用于可用性演示，语言质量不等同于真实模型；真实模型讲解仍没有逐句忠实度评测。
- 展品库搜索是本地词汇扩展、FTS5 和字段加权，准确名称为“字段与线索检索”；“按意思找回”则由 V17 自托管的真实 embedding 在浏览器 Worker 内完成，二者不会混称。
- `.time-isle` 完整备份不包含早期文字整理的 `agent_runs / steps / events`；V10 只新增受限策展的运行/回执/提案/决定审计，并在恢复后强制降为不可执行历史。完整归档包含私人原文和媒体时仍必须由用户自行安全保存。
- 记忆年轮只覆盖展品正文与结构化字段，不是图片、声音、展览和胶囊的全对象版本系统；哈希父链也不是外部签名的不可篡改账本或事实证明。
- 乐观并发只拒绝过期写入，不会自动合并两边内容；修订账本总预算为 20 MiB，达到边界后继续产生修订会被拒绝，系统不会静默删掉旧历史腾空间。
- 馆藏体检只发现当前规则能够识别的结构和文件问题，不验证记忆语义，也不提供自动修复；备份验真会使用隔离暂存和磁盘空间，但不会恢复业务数据。
- JSON 兼容导入最多接收 64 MiB 请求；future schema 必须先升级应用，不能向旧版本降级恢复。
- 流式 `.time-isle` 避免在浏览器 JavaScript 中聚合整包，但服务端仍会创建受控临时文件；异常退出后的归档残留最快在超过一小时并再次启动维护时清扫。
- 本地时间胶囊不是可信第三方时间锁；拥有数据库或设备控制权的人可以绕过日期门槛。
- PBKDF2 会提高口令猜测成本，但不能补救弱口令。离线文件没有找回或撤回能力，口令应通过另一条安全渠道传递。
- 下载后的离线文件无法由发送者远程撤回；知道口令的人仍可复制、转发或截图。
- 自包含阅读页依赖浏览器 Web Crypto 与 Blob URL；只分享被逐项选择的 display WebP、已确认引用和文字稿，不分享原图、EXIF/GPS 或原始音频。
- 回访意愿不是心理画像或重要度模型；welcome 只做候选内优先，later/pause 只约束主动回访，不影响用户主动搜索和查看。
- 时间校准只保存用户本次展示判断与来源摘要，不证明某个日期真实；来源集合变化只触发待复核。若 JSON 迁移缺少照片媒体来源，旧判断会保留并提示需要 `.time-isle` 完成无损迁移，而不会伪造 EXIF 来源。
- 口述史没有自动转写、说话人识别或情绪分析；文字稿与时间含义完全依赖用户核对。非空口述史不能由缺少声音二进制的 JSON 单独恢复，必须使用 `.time-isle`。
- 受限策展助手是确定性的本地规则执行器，不代表通用 Agent、外部大模型质量或自主发布能力；重放评测验证保存回执和提案一致性，不证明策展主题在审美或事实层面正确。
- 共忆邀请与回信的浏览器端加密只能保护文件内容，不验证回信人身份，也没有数字签名、可信时间戳、自动撤回或口令找回；确认入馆后的问答仍是本机 SQLite 明文。
- 设备内镜片不是向量模型或语义理解系统；它只按固定规则重排明确保存的字段和已确认来源，预览不会自动持久化。
- 锁馆不是 SQLite、媒体或磁盘静态加密，也不是操作系统访问控制；拥有设备和数据文件的人仍可能绕过应用读取明文。
- 结构恢复演练没有执行当前馆藏恢复或隔离恢复，只能说明输入归档通过当前结构、哈希和引用检查，不能证明备份在灾难场景中一定可恢复。
- V15 隔离恢复只验证当前机器、当前应用版本的一次性副本；它没有证明异机灾备、RTO/RPO、进程隔离或生产恢复。销毁失败会闭锁后续演练，而不是继续处理下一份备份。
- V16 多视角对照只呈现已有记录和人工确认关系；亲友身份仍未核验，来源变化后的关系只是待复核，文字不同不等于事实冲突。
- V17 首次准备需要下载约 47 MB 同源资源并占用设备算力；会话索引刷新后消失，最接近结果也可能不相关，释放本页内存不等于清除浏览器 HTTP 缓存。
- V18 尚未进入正式实现。Windows 固定 SQLite3MC 驱动 canary 通过不能证明 Linux/CloudBase 可用，SQLite3MC 也不是 Zetetic 官方 SQLCipher；当前数据库与媒体仍未静态加密。
- Vercel `/tmp` 数据可能随实例重建且可能存在多实例，不适合长期馆藏。
- 当前没有账号、云同步或远程媒体备份；“本地优先”不等于自动具备跨设备容灾。

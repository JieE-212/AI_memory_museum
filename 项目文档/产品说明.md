# 产品说明：V7.1 时间胶囊、加密分享与隐私安全 PWA

## 一句话定位

时屿（TIME ISLE）是一款本地优先的 AI 私人记忆策展工具：它保存原文、照片、声音与用户确认过的证据，把散落片段组织成可以检索、回访和讲述的展览；V7 进一步允许用户把确认后的展览封存到未来，或在浏览器内生成口令加密、断网可读的单文件，V7.1 则提供不缓存私人馆藏的可安装外壳。

## 版本状态

本文对应已发布的 V7.1.0（schema 9）。V5–V7.1 已完成实现、完整检查、迁移副本核对、本地验收、双远端推送与 Vercel 部署后验收；线上公开 Demo 的 `/api/version` 与 `/api/health` 已确认返回 V7.1.0 / schema 9。

## 从影像记忆走到 V7.1

文字记录“当时发生了什么”，照片还保留了表情、旧物、字迹和画面之外的情绪。V4 不把图片做成一个装饰性附件，而是把它升级为一等记忆对象：有独立的文件生命周期、隐私策略、结构化关联、区域证据、对照工具和可恢复备份。

项目的差异点不在于宣称“AI 看懂了照片”，而在于建立一条更可信的链路：

```text
原始照片 → 安全展示版本 → 用户圈选或机器候选 → 可编辑草稿 → 用户确认 → 可回看的图片证据
```

这条链路既适合真实私人数据，也能在面试中体现产品判断、安全工程和可解释 AI 的结合。V5–V6.1 在它之上加入主题展览、记忆回访、语义线索、人物档案和语音记忆；V7 解决下一步问题：一段已经核对过的记忆，怎样被留给未来，或安全地交给不登录时屿的人阅读。

```text
已确认展览 → 显式选择安全素材 → 封存本地时间胶囊
                                ↘ 浏览器端口令加密 → 自包含离线 HTML
```

时间胶囊强调仪式感，加密分享负责保密。产品不会把一个本地日期夸大成密码学时间锁，也不会把口令交给服务端代管。

## 目标用户

- 想长期保存日记、聊天片段、旧照片和照片背后故事的人。
- 记录很多，但很难再次找到某个人、地点、物件、字迹或情绪的人。
- 希望用 AI 辅助整理私人内容，同时在意原文、原图、依据和数据边界的人。
- 希望看到完整交互闭环、数据建模和失败补偿，而不只是模型调用的面试官。

## 核心用户任务

1. 写下一段未经整理的原始记忆。
2. 可选添加最多 6 张 JPEG、PNG 或 WebP 照片，并决定“保留原图”还是“仅保留安全展示图”。
3. 为照片补充说明、拍摄时间、照片背面的故事，设置封面并调整顺序。
4. 查看 AI 给出的标题、展厅、人物、标签、情绪和说明，修改并确认后保存展品。
5. 在展品详情中浏览照片胶卷，圈选人物、地点、物件、文字、日期或其他区域，亲自确认后保存为图片线索。
6. 主动寻找“可能相似”的照片，或对选定区域进行本机文字摘录；核对结果，而不是让系统自动合并或改写记忆。
7. 沿记忆航线进入时光拼图，对照两个文字版本及其图片证据；需要时用两组对应点手动叠影。
8. 按关键词、人物、地点、标签或情绪重新找到展品，再借助图片线索理解馆藏并核对讲解引用。
9. 从自己选择的展品生成主题展览，核对章节、引用与媒体后再确认发布。
10. 从已确认展览创建时间胶囊；到期前只看外壳，到期后再读取封存快照。
11. 显式勾选愿意分享的安全展示图与已确认文字稿，在浏览器内设置口令并下载可断网打开的单文件。
12. 用 `.time-isle` 完整归档迁移文字、照片、声音、展览、胶囊、关联和证据，或导出物理排除私人内容的脱敏归档。

## 极简与渐进披露

从 V4 影像记忆扩展到 V7 时间胶囊，页面仍严格保持“展品库、记录记忆、讲解与回顾、数据与项目”四个主视图。

- 添加照片位于原有记录表单内，默认折叠；只写文字的用户不会被媒体流程打断。
- 展品卡片只显示一张小封面和照片数量，不把首页变成复杂相册。
- 完整胶卷、照片背面和图片线索只在展品详情中出现。
- “照片线索”“影像线索实验台”和“手动叠影”都是可选折叠层，按需展开。
- 主题展览、回访、人物档案与“胶囊与分享”都收在“讲解与回顾”已有上下文中，不增加第五项导航。
- 胶囊工作室默认关闭；先选择已确认展览和安全素材，材料读取完成后才进入独立口令步骤。
- 桌面端导航留在页首；窄屏改为底部固定的四项水平导航，文字和点击区域居中，并为系统安全区预留空间。
- 相似候选不会自动执行动作，OCR 不会直接写入正文，EXIF 也不会弹出强制填写。
- 指针操作都有键盘后备：图片区域和手动对齐点均可输入 0–1 规范化坐标。

复杂能力被放在用户已经产生意图的地方，默认界面仍保持清爽、留白和低认知负担。

## 核心能力

### 1. 图片作为一等记忆对象

- 支持 JPEG、PNG、WebP，拒绝伪造 MIME、SVG、GIF、截断容器和像素炸弹。
- 原图流式进入本机暂存区；浏览器另外生成最长边 1600 px 的展示 WebP 和 480 px 缩略 WebP。
- “保留原图”会保存原文件、展示图和缩略图；“仅保留安全展示图”会在完成处理时删除暂存原图，只留下不含原始元数据的派生图。
- SHA-256 相同且既有全部变体重新通过磁盘完整性校验的内容才复用同一媒体资产，但不会因此合并两段展品或认定它们讲述同一事件。
- 每张图片可记录封面角色、顺序、说明、无障碍文本、拍摄时间和照片背面故事。
- 解除最后一个展品关联后，系统隔离并回收无人引用的媒体文件；刚完成上传但尚未关联的图片有 24 小时宽限，避免并发清理。清空馆藏先隔离媒体，再提交数据库事务；失败会恢复文件，物理删除未完成时会明确提示并在后续启动协调。

### 2. 照片区域成为可核对证据

用户可以在规范展示图上拖拽圈选，也可以用键盘填写 `x / y / width / height`。保存的区域使用 `canonical-preview-v1` 坐标空间，并锚定来源图片 SHA-256。

区域类型包括人物、地点、物件、文字、日期和其他；说明、备注与敏感标记都由用户填写。系统分别表达两层状态：

- `source_verified`：区域仍指向同一份图片来源。
- `user_confirmed`：区域含义由用户亲自确认。

来源完整不代表语义真实。区域证据可以修改和删除，并会进入时光拼图的两侧图片依据。

### 3. 克制的影像智能

| 能力 | 系统做什么 | 用户确认点 | 系统明确不做什么 |
| --- | --- | --- | --- |
| EXIF | 严格解析 JPEG APP1 中的拍摄时间、时区、GPS 和方向，全部先记为建议 | 用户点击后才采用拍摄时间 | 不自动写记忆日期或地点，不联网反查 GPS |
| 相似候选 | 对安全展示图生成确定性的 dHash 检索指纹，返回“可能相似”列表 | 用户自行打开候选展品核对 | 不把感知哈希当作相同证明，不自动合并、删除或修改照片 |
| 文字摘录 | 浏览器支持 `TextDetector` 时，只识别用户圈选的本地图片区域；否则提供手动摘录 | 识别文本先进入可编辑草稿，用户勾选确认后才保存为 `text` 区域证据 | 不调用第三方 OCR，不伪装识别成功，不自动写展品正文 |

GPS 被视为敏感数据。“仅保留安全展示图”不会持久化敏感 GPS 线索；选择“保留原图”时，原文件本身仍可能含元数据，相关位置线索在本地标记为敏感并对普通接口隐藏具体值。完整备份可能包含这些私人数据，必须像原相册一样妥善保管。

### 4. 时光对照，而不是自动认定

时光拼图可以把两个记忆版本的照片并排显示，保留各自说明、替代文字和原图入口。可选的“手动叠影”允许用户：

- 在左图与右图各选择两个对应点，组成两组对应关系。
- 用图片点选或 0–1 数值坐标完成相同操作。
- 调整右图透明度、撤销或重置标点。
- 在浏览器 Canvas 中计算平移、旋转和等比缩放的二维相似变换。

它只帮助目视比较，不是自动配准，也不会判断两张照片是否来自同一地点、人物或事件。只有用户确认后，两个文字版本才会组成同一往事的版本组；原文和原图始终分别保留。

### 5. 可携带的完整记忆

`.time-isle` 是当前版本的主要完整备份格式，采用严格 POSIX ustar + gzip 封装，包含：

- 馆藏文字和记忆考古关系。
- 媒体资产，以及按隐私模式保留的 `original`（可选）、`display` 和 `thumb` 文件。
- 展品与图片的顺序、封面和说明关联。
- EXIF 建议、检索指纹和用户确认的图片区域证据。
- 主题展览、回访状态、线索与人物档案、语音记录及已确认文字稿。
- 时间胶囊的外壳、安全快照和 display WebP 关联。

归档中的清单会记录各文件的大小、MIME 和 SHA-256。恢复前，系统先在隔离暂存区完整解包并验证路径、清单、哈希、图片结构和交叉引用；任何损坏都会整批拒绝，不产生业务写入。验证通过后，展品、媒体、关联、图片线索与记忆考古数据在同一数据库事务中恢复；如果事务失败，已经移动的媒体目录会被补偿清理。命中同哈希本地资产时也必须重新核对隐私策略、全部变体和磁盘哈希，不能静默复用损坏或不同规范的预览。

脱敏 `.time-isle` 只保留允许公开的清单与脱敏 section；图片、声音二进制和私人媒体元数据在归档层物理缺席，胶囊标题、日期、来源、快照、图片关联与内部 ID 也被固定脱敏说明替代，而不是只在界面上隐藏。旧版 JSON 导入导出继续作为兼容工具，但不替代完整多模态备份。

### 6. 时间胶囊：外壳可见，内容到期

时间胶囊只允许从状态为 `published`、无需复核的主题展览创建。创建时生成不可被来源展览后续编辑静默改写的安全快照，并把三类数据分开保存：

- 外壳：标题、寄语、开启日期、时区和可用状态。
- 内容：匿名章节、展项、已核验原文引用和用户选择的已确认文字稿。
- 媒体关联：只指向用户明确选择、已经就绪的 display WebP。

开启日前，胶囊列表、详情和内容接口都只使用外壳。`GET /api/capsules/:id/content` 返回 `423`、稳定错误码和公开外壳，不读取内容快照，也不下发正文、图片、展品成员或数据库内部 ID。若来源图片后来被删除，胶囊会进入需要复核状态，而不是继续展示失去来源保证的材料。

开启日期使用用户选择的 IANA 时区解释，解决“哪一天算到期”的产品语义；它仍只是本地仪式门槛。能够修改本机时间或直接读取私人数据库的人不受它提供的保密承诺约束。

### 7. 浏览器端加密的离线展览

离线分享与胶囊共用“只从已确认展览开始”的安全来源，但两者职责不同：胶囊负责封存和到期体验，离线文件负责把内容加密后交给别人。

1. 服务端只列出来源展览中的 display WebP 与已确认文字稿，默认全部不勾选。
2. 用户明确选择材料后，浏览器先读取完整安全载荷；口令区直到读取完成才出现。
3. 从这一步开始不再发送网络请求。口令只在当前页面内存中参与 PBKDF2-SHA-256（310,000 次）和 AES-256-GCM 加密。
4. 浏览器下载自包含 `.html`。文件公开外壳包含标题、寄语和开启日期；正文、图片与文字稿均在密文中。
5. 收件人到期后输入正确口令即可解密，阅读页通过 Blob URL 在本机还原 display WebP，断网也能使用。

每个文件使用随机 16-byte salt、12-byte IV，公开外壳作为 AAD 的一部分参与完整性校验。口令不上传、不持久化、不写入文件，也无法找回。分享结构物理排除原图、EXIF/GPS、草稿文字稿、Agent 日志、内部 ID、未选内容和原始音频二进制；首版只允许已确认声音文字稿进入。

展览中已经通过证据核验的原文引用会随章节结构进入密文；未核验引用不能进入胶囊或离线分享。图片和声音文字稿仍遵循逐项选择，不能因为属于同一展览就被自动打包。

### 8. 可安装但不缓存私人馆藏的 PWA

V7.1 为时屿补齐可安装外壳，但不把 PWA 等同于“离线馆藏”。安装能力沿用原有四项主导航，不新增第五个入口；“安装到此设备”位于“数据与项目”的项目卡中，并按浏览器能力渐进披露。

- Chromium 浏览器只有在收到 `beforeinstallprompt` 后才显示安装按钮；iPhone 与 iPad 不伪装成一键安装，而是给出 Safari“添加到主屏幕”的手动步骤。
- Service Worker 只在 HTTPS 或浏览器认可的本地安全上下文注册；非安全来源、不支持的浏览器和不满足安装条件的环境会静默隐藏入口，不阻塞记录、检索和策展主流程。
- 缓存采用精确白名单，只包含 `/offline.html`、`/pwa.css` 和公开品牌 SVG。首页、主应用脚本、API、图片、声音、归档与任何用户内容都不进入 Cache Storage，也不会在运行时动态写入缓存。
- 原图、展示图和缩略图等所有图片变体统一返回 `private, no-store`。断网导航只展示不含馆藏、照片、声音或导出内容的隐私边界页，不用空列表伪装成“馆藏为空”，恢复连接后再返回真实应用。

本地模式仍是完整媒体保存、胶囊封存和归档恢复的主要体验环境；公开 Demo 继续使用临时数据、强制 Mock 与禁写边界。PWA 安装不会改变两种运行模式的数据能力，也不会让公开 Demo 变成私人内容或长期数据的存储空间。加密分享文件可以在用户下载后独立断网解密阅读，这是自包含密文文件的能力，不代表时屿应用缓存了私人馆藏。

## 产品原则

- 原文与原图优先：AI 草稿不覆盖原文，派生视图不冒充原图。
- 人工确认：AI 整理、EXIF、相似候选、OCR 和事件关系都有明确确认边界。
- 证据可回看：文字证据保留原文字符位置，图片证据保留来源哈希与规范化区域。
- 本地优先：SQLite 与内容寻址媒体目录默认位于本机，不要求外部数据库或对象存储。
- 版本并存：同一往事的多次文字与图片记录分别保留，系统不制造唯一真相。
- 缺失不是矛盾：只有两侧都有可验证信息时才展示描述差异。
- 日期与保密分工：开启日期提供仪式门槛，浏览器端加密才承担离线分享的保密职责。
- 最小分享：已核验引用随确认展览进入；安全展示图和已确认文字稿只有用户显式选择后才进入分享包，未选择等于物理缺席。
- 失败安全：上传、归档和恢复在失败时清理暂存或补偿已移动文件。
- Demo 隔离：公开展示不接收私人图片，并阻止破坏示例稳定性的写操作。

## 当前范围

已经实现：

- 单条记忆整理、三阶段轨迹、编辑确认和结构化 SQLite 馆藏。
- 最多 6 张照片的本地安全上传、派生、去重、关联、展示、编辑和回收。
- 原图保留与仅安全展示图两种隐私策略。
- EXIF 待确认线索、感知相似候选和隐私优先的照片文字摘录。
- 锚定来源哈希的用户确认图片区域证据。
- 混合检索、引用式讲解、时间线、主题和摘要。
- 可解释记忆航线、带原文锚点和图片证据的时光拼图。
- 两图并排与手动两点相似变换叠影。
- 人工确认的事件版本组、单问题补充和“保留不确定”。
- 用户确认后才保存的主题展览，以及一次只呈现一件记忆的回访。
- 可解释语义线索检索、人物/地点/主题档案与人工确认的别名关系。
- 浏览器录音或音频上传、独立音频存储、回放与人工确认文字稿。
- 只来源于已确认展览的时间胶囊；未到期只返回外壳，到期后读取安全快照。
- 浏览器端 PBKDF2-SHA-256 + AES-256-GCM 加密、自包含且断网可读的 HTML 分享文件。
- 不缓存私人馆藏的可安装 PWA 外壳，以及断网时不读取用户内容的隐私边界页。
- `.time-isle` 完整/脱敏归档、事务恢复与旧 JSON 兼容工具。
- 本地模式和受保护的公开 Demo 模式。

不在当前范围：

- 多用户账号、细粒度权限和跨设备端到端加密身份体系。
- 云端对象存储、云数据库和跨设备实时同步。
- 第三方云 OCR、自动图片理解或自动生成照片事实。
- GPS 反向地理编码或自动填写地点。
- 基于相似度自动合并、删除照片或合并记忆事件。
- 视频和动态图片存储；离线分享首版也不携带原始音频二进制。
- 公网私密链接、账号托管、分享撤回、远程失效和口令找回。
- 自动替用户发布、选择分享内容或训练私人记忆。

## 公开 Demo 保护

启用 `INTERVIEW_DEMO=true` 后，SQLite 和媒体目录切换到系统临时位置，冷启动注入四件示例展品。Demo 可以浏览馆藏、使用讲解和记忆航线，但：

- 全部媒体写操作返回 403，包括上传、指纹写入、关联和图片区域证据增删改。
- 麦克风保持关闭，全部音频写入返回 403；时间胶囊也不在共享临时实例中持久化。
- 禁止删除、清空、导入和完整归档恢复。
- 预置展品禁止修改。
- Demo 在代码层强制使用本地 Mock；即使环境误配 AI Key，也不会调用外部模型。
- 可新增的共享文本、整理运行、时光拼图与补问分别受 SQLite 事务硬上限保护，并发请求达到上限后返回 429。
- 临时新增的纯文字示例可能被同一实例的其他访客看到，并会随实例重建消失。
- 在具备已确认示例展览时，可以演示安全素材选择和浏览器本地加密；口令仍不进入服务端，下载结果只应使用无隐私风险的示例数据。

公开 Demo 不是私人会话空间。它不部署开发者的本地馆藏，代码也不会在 Demo 模式使用真实模型密钥；访客不应提交私人内容。

## 面试展示建议

本地 V7.1 推荐用六分钟展示：

1. 在“记录记忆”中展开照片入口，说明两种保存策略并加入一张无隐私风险的测试照片。
2. 展示封面、排序、照片说明、EXIF 拍摄时间建议，以及卡片缩略图和详情胶卷。
3. 圈选照片中的旧物或文字，强调来源哈希与“用户确认”是两层不同保证。
4. 在影像实验台展示“可能相似”候选和本机文字草稿，明确它们不会自动合并或改正文。
5. 进入时光拼图，展示两图并排和手动两点叠影，再说明只有用户能确认同一往事。
6. 导出 `.time-isle`，用损坏零写入、事务恢复和脱敏归档物理排除私人媒体与胶囊私密字段收尾。
7. 从一场已确认主题展览进入“胶囊与分享”：先展示未到期只见外壳，再只勾选一张 display WebP 和一份已确认文字稿。
8. 等素材读取完成后输入口令，下载自包含 HTML；断网打开并分别演示错误口令、正确口令，最后说明日期是仪式门槛、Web Crypto 才负责保密。

公开 Demo 应改为展示既有示例、讲解、航线和安全边界，不现场上传私人照片。

## 已知限制

- `TextDetector` 不是所有浏览器的标准可用能力；不可用或启动失败时，产品只提供手动摘录，不会降级到远程 OCR。
- EXIF 当前只严格读取 JPEG APP1/Exif 的有限字段；无时区的拍摄时间保持 floating local time，不会伪造 UTC。
- 感知相似度适合召回人工候选，不是身份、地点或同一事件证明；完全相同 SHA-256 只用于资产复用。
- 手动叠影依赖用户给出两组可靠对应点，只计算平移、旋转和等比缩放，不处理透视、非刚性变形或自动特征匹配。
- 图片区域的 `source_verified` 只证明来源锚点仍匹配，不证明标签内容为真。
- Mock 模式用于可用性演示，语言质量不等同于真实模型；真实模型讲解仍没有逐句忠实度评测。
- 语义检索是本地词汇扩展和字段加权，不是向量数据库。
- `.time-isle` 完整备份不包含 Agent 运行日志；它包含私人原文和媒体时必须由用户自行安全保存。
- 本地时间胶囊不是可信第三方时间锁；拥有数据库或设备控制权的人可以绕过日期门槛。
- PBKDF2 会提高口令猜测成本，但不能补救弱口令。离线文件没有找回或撤回能力，口令应通过另一条安全渠道传递。
- 自包含阅读页依赖浏览器 Web Crypto 与 Blob URL；首版只分享 display WebP 和已确认文字稿，不分享原图、EXIF/GPS 或原始音频。
- Vercel `/tmp` 数据可能随实例重建且可能存在多实例，不适合长期馆藏。
- 当前没有账号、云同步或远程媒体备份；“本地优先”不等于自动具备跨设备容灾。

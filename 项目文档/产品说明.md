# 产品说明

## 一句话定位

AI 记忆博物馆是一款本地优先的个人记忆整理工具：用户保留原始叙述，AI 提供可编辑的策展建议，馆藏随后可以被检索、讲解和回顾。

## 目标用户

- 想长期保存日记、聊天片段或照片故事的人。
- 记录很多，但很难再次找到某个人、地点或情绪的人。
- 希望使用 AI 整理私人内容，同时在意原文、依据和数据边界的人。
- 需要快速理解项目价值、交互闭环和工程取舍的面试官。

## 核心用户任务

1. 写下一段未经整理的记忆。
2. 查看 AI 给出的标题、展厅、人物、标签、情绪和说明。
3. 修改并确认草稿，将它保存为展品。
4. 按关键词、人物、地点、标签或情绪重新找到展品。
5. 向讲解员提问，并核对回答引用了哪些展品。
6. 从时间线和主题中回看一组记忆。
7. 导出、脱敏或恢复自己的数据。

## 产品原则

- 原文优先：AI 生成的是展品说明，不覆盖原始记忆。
- 人工确认：模型输出必须经过用户确认才进入馆藏。
- 依据可见：Agent 步骤、检索命中原因和讲解引用都可以回看。
- 本地优先：默认使用本机 SQLite，不要求外部数据库。
- 证据克制：没有足够展品时明确说明，不编造个人经历。
- Demo 隔离：公开展示只使用示例数据、临时存储和安全限制。

## 当前范围

保留：

- 单条记忆整理与编辑确认。
- 结构化 SQLite 馆藏。
- Agent 整理轨迹。
- 混合检索和引用回答。
- 时间线、主题与摘要。
- JSON 备份、脱敏和恢复。
- 本地模式与公开 Demo 模式。

不在当前范围：

- 多用户账号和权限系统。
- 云端长期数据库与跨设备实时同步。
- 第三方插件运行时。
- 发布审批、治理看板和阶段管理工具。
- 图片/音频文件存储与 OCR 服务。
- 自动替用户发布或分享私人记忆。

## 面试展示建议

推荐用三分钟完成以下路径：

1. 打开展品库，说明示例数据和 Demo 安全提示。
2. 进入“记录记忆”，放入示例并执行 AI 整理。
3. 展开 Agent 轨迹，强调草稿需要人工确认。
4. 保存后搜索“朋友 陪伴”，展示命中依据。
5. 向讲解员提问，点击引用回到具体展品。
6. 打开“数据与项目”，说明 SQLite、脱敏导出和 Demo 隔离。

## 已知限制

- Mock 模式主要用于可用性演示，语言质量不等同于真实模型。
- 语义检索是本地词汇扩展和字段加权，不是向量数据库。
- Vercel `/tmp` 数据会随实例重建，不适合真实长期存储。
- 目前没有登录，公开 Demo 中的新增内容可能被同一实例的其他访问者看到，并会在实例重建后消失。

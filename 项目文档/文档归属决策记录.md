# 文档归属决策记录

更新日期：2026-07-05

Decision: `docs-in-repo`

## 1. 决策

接受 `docs-in-repo`：项目文档应进入当前 git root 内的 `项目文档/` 目录，和代码、脚本、收口材料一起进入同一个仓库边界。

当前仍不是 git index 迁移执行：

- `gitPathShape=accepted-flattened-project-root`
- `docsOwnershipDecision=docs-in-repo`
- `gitIndexMigration=committed-flattened-project-root`
- `outsideMirrorRetained=true`
- `indexMutation=committed-path-migration`

## 2. 执行方式

本轮只把 `../项目文档` 镜像到当前 git root 内的 `项目文档/`，不删除仓库外文档目录。

这样可以避免旧索引中的 `项目文档/...` 被误解释成文档删除，也为后续路径迁移提交保留完整审查材料。

## 3. 后续边界

后续仍需要维护者明确授权，才能执行真正的 git index 迁移。授权前继续禁止：

- `git add -A`
- `git add -u`
- `git rm`
- `git commit`
- `git reset --hard`
- `git checkout --`
- `git clean`

## 4. 当前结论

文档归属决策已从 `pending` 推进为 `docs-in-repo`。索引迁移提交已完成，但不代表 release approval。

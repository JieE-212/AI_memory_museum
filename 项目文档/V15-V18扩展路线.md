# V15–V18 扩展路线：让记忆可恢复、可对照、可联想、可真正锁起

> 状态（2026-07-22）：V15、V16、V17 已完成实现、统一门禁并随 V17.0.0 发布；功能提交为 `413f78640baad7eae6324ef14bc291f05325fbf6`，热修复为 `38d3450b9f26efdc59df4860317cdf9513e77e65`，Vercel 与 CloudBase 部署 `002` 已完成生产核验。V15 完整恢复上传继续只在隔离本地环境开放，V17 手机真机复核仍待执行。V18 停在 V18.0 可行性门禁，当前总体结论为 **NO-GO**。

## 不变的产品主轴

时屿仍是“把私人记忆整理成可回看展品”的本地优先工具，不扩张为通用备忘录、网盘或社交平台。

- 顶层导航继续只有“展品库、记录记忆、讲解与回顾、数据与项目”四项。
- 新功能放回最相关的已有页面，默认折叠或按需启动；手机仍保留固定底栏和单列阅读。
- 首屏先说“它能帮你做什么”，技术边界放在可展开说明中；按钮使用动作名称，失败时告诉用户下一步怎么办。
- 用户写下的原文、照片、声音与人工确认始终优先。系统只整理证据，不替用户判断哪段记忆“更真实”。
- 每个大阶段结束都运行专项检查、完整回归和桌面/窄屏体验，并删除一次性样本、临时数据库、调试入口与不再使用的实验代码。
- 面试展示只陈述已经由实现与门禁证明的能力；模型、隔离和加密均不使用模糊词包装。

## V15：真实隔离恢复演练

> 阶段状态：**已完成并随 V17 发布**。真实恢复副本、恢复后独立复核、销毁回执、小白界面与永久自动门禁均已落地；公开 Demo 在读取正文前拒绝完整备份上传，不把本机演练描述成异机灾备。

### 一句话目标

用户选择一份完整 `.time-isle` 备份后，系统把它真正恢复到一次性馆藏副本，完成体检并销毁副本；整个过程不碰当前馆藏。

### 小阶段

1. **V15.0｜合同与隔离边界**：定义三阶段回执、资源预算、单任务并发和失败即关闭规则；公开 Demo 在必要的全局 Host / Origin 同源校验后、读取归档类型与正文前拒绝私人备份。
2. **V15.1｜真实恢复副本**：创建全新的临时 SQLite、图片目录和声音目录，复用正式恢复流程完成实际写入；当前馆藏的数据库、路径和存储能力不得进入运行时。
3. **V15.2｜副本体检与销毁**：核对 SQLite quick check、外键、迁移账本、FTS、引用、媒体数量与 SHA-256；关闭数据库并确认目录已经不存在后才返回成功。
4. **V15.3｜小白界面与永久门禁**：在“数据与项目”的锁馆面板中提供默认折叠入口，用“验真 → 恢复副本 → 销毁副本”解释过程；加入核心、API、HTTP、取消、并发、损坏包与清理失败测试。

### 诚实边界

演练只证明“这份备份能在当前机器、当前版本的一次性副本中恢复”，不证明异机灾备、RTO/RPO、进程隔离、磁盘加密或生产环境恢复能力。回执不返回标题、正文、内部 ID、文件路径、单文件哈希或映射表。

## V16：多视角记忆对照

> 阶段状态：**已完成并随 V17 发布**。对照由现有展品、年轮、已确认回信和用户确认的来源关系只读派生；未绑定、已撤回或原文变化后的关系不会被冒充成有效结论。

### 一句话目标

在一件展品旁边并排看“我现在怎样记录、我过去怎样修改、亲友后来怎样讲、来源怎样关联”，帮助用户发现差异，但不替用户裁决。

### 小阶段

1. **V16.0｜来源兼容修正**：补齐口述史时间来源在前端的显示支持，避免已经存在的合法来源被静默过滤，并加永久测试。
2. **V16.1｜只读对照合同**：从当前展品、记忆年轮、已确认共忆回信、来源护照与时间判断派生只读 DTO；GET-only、零新增表、零重复持久化。
3. **V16.2｜关系与不确定性**：只展示用户已经确认的 `支持 / 补充 / 不同记录` 关系；尚未绑定来源护照的亲友回信明确标记为“尚未建立对照关系”，不得自动称为一致或冲突。
4. **V16.3｜渐进式阅读界面**：入口放在展品详情和“讲解与回顾”，默认显示一张通俗摘要；需要时再展开逐字差异、来源身份边界和时间线，不增加导航。
5. **V16.4｜永久验收**：覆盖来源变化、删除、未核验身份、无关系、长文本、空状态、Demo、桌面与 390/320px；确认归档、恢复与 schema 不因派生视图而增加重复数据。

### 诚实边界

“对照”只呈现已有记录和人工关系，不计算可信度，不判断谁记得更准，也不把文字差异自动解释为矛盾。亲友身份仍是自述、未核验。

## V17：设备内语义回忆

> 阶段状态：**已完成并随 V17 发布**。真实中文 embedding 在用户主动点击后于浏览器 Worker 中加载；索引、查询和向量只留在当前会话内存，旧搜索已明确称为“字段与线索检索”。自动化与桌面/390/320 三档真实浏览器门禁已通过，V17 发布后的手机真机复核仍待执行。

### 一句话目标

用户可以输入“小时候下雨天等爸爸”这类自然描述，在自己的设备上按文本含义找回相关展品；查询、正文与向量不发给云端。

### 小阶段

1. **V17.0｜术语纠偏与模型门禁**：把现有 FTS5、字段匹配和固定词表能力改称“字段与线索检索”；使用完全虚构的中文评测集比较候选模型的许可、体积、速度、Recall@3 与误召回，未通过则不得称为语义检索。
2. **V17.1｜本地推理底座**：固定自托管模型、tokenizer、运行库与 SHA-256；在 Web Worker 中运行真实 embedding，优先可选 WebGPU、必须有 WASM 回退，并支持取消和释放。
3. **V17.2｜会话级索引**：新增有硬上限的 GET-only 来源快照；只索引展品标题、说明、正文、标签和已确认文字稿。查询、向量与索引仅保留在 Worker 内存，刷新后需要重新准备。
4. **V17.3｜“按意思找回”界面**：放在“讲解与回顾”并默认折叠；用户主动点击后才加载模型，提前展示下载大小、隐私边界、进度与清除按钮；失败时明确显示“设备语义不可用”，可切换到旧的字段检索但不得继续显示语义成功。
5. **V17.4｜隐私与多视口门禁**：覆盖中文释义和负例、第三方请求拦截、浏览器存储泄漏、馆藏变化失效、500 件预算、取消、重复启动、离线，以及桌面、390×844、320×700 真实浏览器视口；清除所有评测临时页面与样本文本。三档视口不冒充手机真机，发布后真机复核单列为待办。

### 诚实边界

结果是模型生成的文本相似度排序，不是事实、关系、情绪、真实性或概率判断；embedding 也可能包含原文特征，不能称为匿名数据。若没有真实模型参与，本功能不成立。

## V18：真正的加密锁馆

> 阶段状态：**停在 V18.0 门禁，NO-GO**。Windows x64 原生 canary 已通过，但尚无 Linux `node:24-bookworm-slim` 或 CloudBase 同等执行证据；不得进入 V18.1。完整证据和边界见 [V18.0 加密可行性门禁记录](./V18.0加密可行性门禁记录.md)。

### 一句话目标

启动默认锁定；没有口令时，时屿无法读取当前馆藏，数据目录里的 SQLite 内容、图片、声音与加密馆藏备份都保持认证加密。

### 小阶段

1. **V18.0｜可行性与威胁模型门禁**：验证维护中的原生 SQLite 加密驱动能否覆盖 Node 24、Windows x64、Linux Docker/CloudBase、WAL 与现有同步数据层，并准确区分 SQLite3MC 与 Zetetic 官方 SQLCipher；同时完成 scrypt、HKDF、AES-GCM 已知答案和篡改测试。门禁不通过时停止“真正加密”表述，不用前端加密或旧写门禁冒充完成。
2. **V18.1｜延迟启动的保险库运行时**：把静态壳、状态与解锁入口放在前置进程；敏感数据库与媒体密钥只存在于可终止的 vault worker。启动锁定，锁定时先拒绝新请求、排空在途操作、关闭数据库并结束 worker。
3. **V18.2｜新建加密馆藏**：随机生成 DEK，使用 scrypt 派生的 KEK 包装，HKDF 分离数据库与媒体密钥；使用经过 V18.0 跨平台门禁的 SQLite 加密方案保存数据库，AES-256-GCM 保存随机文件名的图片和声音。修改口令只重新包装 DEK。
4. **V18.3｜安全迁移与加密备份**：先备份和核对空间，在 sibling staging 中迁移，逐项验证数量、引用与媒体哈希后原子切换；中断时继续使用原馆藏。新增 `.time-isle-vault`，并把旧 `.time-isle` 明确标为“解锁后的明文兼容导出”。
5. **V18.4｜小白向导与对抗性收尾**：用“尚未启用 / 迁移中 / 已解锁 / 已锁定 / 需要修复”解释状态；首次向导先说明能保护与不能保护的内容、忘记口令无法恢复，再设置口令。公开 Demo 在读取口令正文前拒绝 setup/unlock/restore。

### 诚实边界

加密保护锁定状态和未解锁进程中的静态内容，不防已解锁内存、管理员权限、恶意软件、键盘记录、截图、swap/休眠、系统快照、文件外部大小与数量元数据、回滚、删除或勒索。旧明文文件和旧备份不会因启用 V18 自动变成“从未存在”；仍建议开启 BitLocker 等系统磁盘加密。

## 每阶段统一完成条件

每个版本必须同时满足以下条件，才能进入下一阶段：

1. 专项契约、数据库/API、前端与安全负例进入 `scripts/check-all.js` 的永久门禁。
2. `npm run build` 和包含真实 HTTP smoke 的 `npm run check` 全绿。
3. 真实浏览器完成桌面、390×844 与 320×700 的可读性、键盘、触控和主流程验收。
4. 公开 Demo 的私人上传、口令和写入继续保持“读取正文前拒绝”。
5. 删除临时数据库、媒体、模型评测样本、一次性调试脚本、废弃开关和未使用分支；保留的测试必须能长期阻止回归。
6. 更新产品说明、技术设计、面试展示手册、README 与简历描述；只有最后统一回归后再决定提交与发布。

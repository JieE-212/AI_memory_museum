## 4. 会议议程

1. 确认本会议包不是 release approval。
2. 确认 `data/phase30-human-evidence-submission.json` 尚未创建。
3. 确认 5 个 reviewer 角色是否到场。
4. 逐项审查 10 个 evidence slots 的证据来源要求。
5. 确认每个 reviewer 会后需要填写的 `evidenceRef`、`reviewer`、`reviewedAt`、`disposition`、`decisionReason` 和 `residualRisk`。
6. 对 `accepted-with-risk`、`deferred-with-owner` 或 `blocked` 项确认 `followUpOwner`、`recoveryCondition` 与 `targetReviewDate`。
7. 确认只有维护者在收齐真实证据后，才能创建 live submission。

## 6. 会后动作

- reviewer 在线下填写 `阶段30人工证据提交样例.json` 的对应槽位。
- 维护者确认所有槽位没有 `REPLACE_WITH_...` 或 `YYYY-MM-DD` 占位符后，再考虑创建 live submission。
- 创建 live submission 前继续运行 `phase30:review-session-package`、`phase30:reviewer-handoff` 和 `phase30:evidence-submission-template`。
- 创建 live submission 后运行 `phase30:evidence-submission-gate`。

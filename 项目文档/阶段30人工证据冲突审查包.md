## 3. 冲突类型

| conflictType | trigger | required human action |
| --- | --- | --- |
| `duplicate-evidence-id-conflict` | 同一 `evidenceId` 出现多个互斥 records | 指定唯一有效 record，记录 rejected record 和原因 |
| `reviewer-identity-conflict` | `reviewer` 与 owner role 或授权 reviewer 不一致 | release owner 或对应 reviewer lead 确认身份 |
| `disposition-conflict` | 同一槽位出现互斥 `disposition` | 记录最终处置、被覆盖处置和决定理由 |
| `evidence-ref-conflict` | `evidenceRef` 缺失、不可访问或指向互斥材料 | 替换为可审计证据引用 |
| `risk-followup-conflict` | `accepted-with-risk`、`deferred-with-owner` 或 `blocked` 缺少后续字段 | 补齐 `followUpOwner`、`recoveryCondition`、`targetReviewDate` |
| `release-runtime-boundary-conflict` | release approval、runtime go/no-go、Phase 29 exit 或 Phase 30 entry 被混写 | 拆分为独立决策，不允许连带批准 |
| `placeholder-conflict` | 出现 `REPLACE_WITH_...`、`YYYY-MM-DD`、`pending`、`missing`、`unassigned` 或空字符串 | 返回 reviewer 重新提交 |
| `format-approval-conflict` | 把格式通过、预检通过或汇总完成当作 release approval | 明确撤回该结论并重新审查 |

## 5. 冲突解决记录要求

每个被发现的冲突都必须记录：

- `conflictId`
- `evidenceId`
- `conflictType`
- `detectedBy`
- `detectedAt`
- `resolutionOwner`
- `resolutionDecision`
- `resolutionReason`
- `supersededEvidenceRef`
- `acceptedEvidenceRef`
- `residualRisk`

如果 `resolutionDecision` 仍为 `accepted-with-risk`、`deferred-with-owner` 或 `blocked`，必须继续补齐，并进入 `阶段30风险处置后续台账.md`：

- `followUpOwner`
- `recoveryCondition`
- `targetReviewDate`

## 6. 与转换预检的关系

`phase30:live-submission-conversion-preflight` 只有在冲突审查满足以下条件后，才可以继续讨论维护者转换：

1. 没有 unresolved conflict。
2. 每个冲突都有真实 `resolutionOwner` 和 `resolutionReason`。
3. 被覆盖的证据引用已经保留在 `supersededEvidenceRef`。
4. 最终采纳的证据引用已经写入 `acceptedEvidenceRef`。
5. 没有把 `format-valid`、`conversion-ready-for-maintainer-action` 或 `summaryStatus` 当作 release approval。
6. 所有风险类处置已通过 `phase30:risk-disposition-followup-ledger` 检查。

当前没有外部真实 reviewer 证据，因此冲突审查默认阻断转换。

## 7. 禁止事项

- 不自动创建 `data/phase30-human-evidence-submission.json`。

- 不把冲突审查完成当作 runtime go/no-go。
- 不伪造 reviewer、detectedBy、resolutionOwner、resolutionDecision 或 evidenceRef。
- 不设置 `releaseReady=true`。
- 不设置 `phase29ExitReady=true`。
- 不设置 `phase30EntryReady=true`。
- 不启用 `runtimeExecution`。
- 不启用 `thirdPartyExecution`。

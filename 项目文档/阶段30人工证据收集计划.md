# 阶段30人工证据收集计划

更新日期：2026-07-05

Baseline: `2.0.10 / phase30-human-evidence-intake-plan`

Phase 30 human evidence intake plan: active

本计划用于指导人工 reviewer 填写证据，不自动生成证据，不替代 reviewer 签核，不改变只读执行台账。

配套登记表：`阶段30人工证据收集登记表.md`，Baseline: `2.0.11 / phase30-human-evidence-intake-register`。

## 1. 当前状态

当前所有 evidence slots 仍保持：

- `currentDisposition=pending`
- `evidenceRef=missing`
- `reviewer=unassigned`
- `reviewedAt=missing`
- `decisionReason=missing`
- `residualRisk=missing`

因此仍阻断：

- release approval
- Phase 29 exit
- Phase 30 entry
- runtime execution
- third-party execution

## 2. 收集顺序

建议按以下顺序收集人工证据：

1. `release-blocker-disposition`
2. `transition-redline-disposition`
3. `signoff-evidence-reconciliation`
4. `release-runtime-separation`
5. `runtime-owner-go-no-go`
6. `rollback-strategy-review`
7. `sandbox-acceptance-review`
8. `secret-boundary-review`
9. `private-memory-boundary-review`
10. `audit-dry-run-review`

## 3. 每个槽位的最低填写要求

每个 evidence slot 必须包含：

- `evidenceRef`
- `reviewer`
- `reviewedAt`
- `disposition`
- `decisionReason`
- `residualRisk`

当 `disposition` 为 `accepted-with-risk`、`deferred-with-owner` 或 `blocked` 时，还必须包含：

- `followUpOwner`
- `recoveryCondition`
- `targetReviewDate`

## 4. 允许状态

只允许以下人工状态：

- `approved-with-evidence`
- `accepted-with-risk`
- `deferred-with-owner`
- `rejected`
- `blocked`

`pending` 只能表示尚未完成填写，不能作为通过状态。

## 5. 审查边界

人工证据收集完成后，仍需单独执行 release 审批和 runtime go/no-go。

禁止把以下结果自动转换为批准：

- check pass
- dry-run pass
- RC freeze
- closure review pass
- evidence worksheet exists

## 6. 当前结论

当前阶段只建立 intake 计划。所有 evidence slots 仍为 pending，不能进入 release、Phase 29 exit、Phase 30 entry 或 runtime enablement。

下一步进入 `阶段30人工证据收集登记表.md` 的人工登记准备；登记表存在本身仍不是 approval。

# 阶段30人工证据提交门禁

更新日期：2026-07-05

Baseline: `2.0.12 / phase30-human-evidence-submission-gate`

Phase 30 human evidence submission gate: active

本门禁用于校验未来真实 reviewer 提交的人工证据文件。它只检查格式、字段完整性、处置状态和保护线，不生成证据，不替代 reviewer，不批准 release，不退出 Phase 29，不进入 Phase 30 implementation，也不启用 runtime execution 或 third-party execution。

## 1. 当前状态

当前没有 live human evidence submission 文件。

- `submissionStatus=not-submitted`
- `pendingSlots=10`
- `approvedSlots=0`
- `noAutomaticApproval=true`
- `releaseReady=false`
- `phase29ExitReady=false`
- `phase30EntryReady=false`
- `runtimeExecution=false`
- `thirdPartyExecution=false`

当前结论：`format-gate-ready-but-evidence-not-submitted`。

## 2. 未来提交路径

真实人工证据只允许进入以下路径：

- `data/phase30-human-evidence-submission.json`

该文件默认不存在。文件存在时，`phase30:evidence-submission-gate` 会校验它；文件不存在时，门禁通过但继续保持 pending。

离线填写样例位于：

- `项目文档/阶段30人工证据提交样例.json`

样例由 `phase30:evidence-submission-template` 校验。样例文件不能直接视为 live submission，必须由真实 reviewer 填写后，才可由维护者明确复制为 `data/phase30-human-evidence-submission.json`。

Reviewer 责任拆分见：

- `阶段30人工Reviewer交接包.md`

交接包由 `phase30:reviewer-handoff` 校验。交接包只能分配人工任务，不能作为 live submission。

```json
{
  "source": "human-reviewer-submission",
  "noAutomaticApproval": true,
  "releaseReady": false,
  "phase29ExitReady": false,
  "phase30EntryReady": false,

  "thirdPartyExecution": false,
  "records": [
    {
      "evidenceId": "runtime-owner-go-no-go",
      "owner": "runtime owner",
      "reviewer": "real reviewer identity",
      "reviewedAt": "2026-07-05",
      "evidenceRef": "external-review-record-or-ticket",
      "disposition": "approved-with-evidence",
      "decisionReason": "human reviewed the runtime go/no-go record",
      "residualRisk": "none after review",
      "followUpOwner": "",
      "recoveryCondition": "",
      "targetReviewDate": ""
    }
  ]
}
```

## 4. 必需证据槽位

提交文件必须覆盖 10 个 evidence slots：

1. `release-blocker-disposition`
2. `transition-redline-disposition`
3. `signoff-evidence-reconciliation`
4. `release-runtime-separation`
5. `runtime-owner-go-no-go`
6. `rollback-strategy-review`
7. `sandbox-acceptance-review`
8. `secret-boundary-review`
9. `private-memory-boundary-review`
10. `audit-dry-run-review`

每个槽位必须包含真实 `reviewer`、`reviewedAt`、`evidenceRef`、`disposition`、`decisionReason` 和 `residualRisk`。

## 5. 允许处置状态

允许状态：

- `approved-with-evidence`
- `accepted-with-risk`
- `deferred-with-owner`
- `rejected`
- `blocked`

不允许把 `pending` 写入 live submission。`pending` 只表示尚未提交。

当状态为 `accepted-with-risk`、`deferred-with-owner` 或 `blocked` 时，还必须填写：

- `followUpOwner`
- `recoveryCondition`
- `targetReviewDate`

## 6. 门禁结论

门禁只产生格式结论：

- `format-gate-ready-but-evidence-not-submitted`
- `format-valid-but-not-release-approval`
- `format-valid-with-blocking-disposition`
- `format-invalid`

即使所有 records 都通过格式检查，也只是 `format-valid-but-not-release-approval`，仍需独立 release approval 与 runtime go/no-go。

## 7. 禁止事项

- `auto-approve-human-signoff`
- `convert-format-valid-to-release-approval`
- `convert-format-valid-to-runtime-go`
- `mark-releaseReady-true`
- `mark-phase29ExitReady-true`
- `mark-phase30EntryReady-true`
- `enable-runtimeExecution`
- `enable-thirdPartyExecution`
- `execute-real-third-party-plugin`

本门禁存在本身不代表任何 evidence slot 已经通过人工审查。

提交样例存在本身也不代表任何 evidence slot 已经通过人工审查。

Reviewer 交接包存在本身也不代表任何 evidence slot 已经通过人工审查。

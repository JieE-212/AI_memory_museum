# 当前候选交付变更索引

状态：`rc-reviewable-but-not-releasable`

这份索引用来汇总当前候选交付的本地体验、SQLite 初始化、人工证据闭环、普通用户路径和验收检查。它是交接索引，不是 release approval，不是 Phase 29 exit，不是 Phase 30 entry，也不是 runtime 或 third-party execution 放行。

## 一、交付边界

- 当前版本保持 `APP_VERSION=1.9.48`，当前阶段保持 `PHASE=29`。
- `releaseReady=false`。
- `phase29ExitReady=false`。
- `phase30EntryReady=false`。
- `runtimeExecution=false`。
- `thirdPartyExecution=false`。
- 默认不得创建 `data/phase30-human-evidence-submission.json`。
- 只有拿到真实 reviewer 输出和维护者显式转换授权后，才允许进入 live submission 转换流程。

## 二、已完成的候选交付项

1. 人工证据闭环状态审计：新增 no-live-submission 状态检查，明确 10 个证据槽位仍缺失，不能自动放行。
2. 首次 SQLite / 示例数据体验：把浏览器示例和真实 SQLite 数据边界显性化，按钮统一为“写入 SQLite”。
3. 首页普通用户入口瘦身：普通路径保留记录、回看、讲解、批量导入、报告、数据与同步；维护者入口默认折叠。
4. 维护者治理红线：维护者入口展示 release、Phase 30 entry、runtime、third-party、human evidence 状态。
5. 候选交付总览：维护者入口展示当前候选可体验、可复核、不可发布。
6. 轻量验收说明：新增一页式验收说明，覆盖启动、SQLite 边界、维护者总览、检查命令和禁做项。
7. 候选 UI 结构化检查：新增 `candidate:delivery-ui-check` 并纳入 `npm.cmd run check`。
8. 普通用户主路径打磨：新增保存状态条，折叠照片/附件线索，优化展品详情阅读结构。

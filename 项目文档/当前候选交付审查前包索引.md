# 当前候选交付审查前包索引

状态：`pre-review-package-only`

本索引用于人工审查会前打包当前候选交付材料，帮助 reviewer 按固定顺序阅读、演示和记录缺口。它不是 reviewer 结论，不是 live submission，不是 release approval，不是 Phase 29 exit，不是 Phase 30 entry，也不启用 runtime execution 或 third-party execution。

## 一、审查前结论

当前候选交付只能进入人工审查准备：

- `rc-reviewable-but-not-releasable`
- `APP_VERSION=1.9.48`
- `PHASE=29`
- `releaseReady=false`
- `phase29ExitReady=false`
- `phase30EntryReady=false`
- `runtimeExecution=false`
- `thirdPartyExecution=false`
- `approvedSlots=0`
- `missingSlots=10`
- `liveSubmissionExists=false`

## 二、Reviewer 阅读顺序

建议 reviewer 按以下顺序阅读，不要跳过证据缺口状态：

1. `README.md`：确认当前冻结阶段、候选状态和所有治理红线。
2. `项目文档/当前候选交付变更索引.md`：理解第 1 到第 9 项候选变更范围。
3. `项目文档/当前候选交付验收说明.md`：按本地验收方式确认普通用户路径和维护者入口。
4. `项目文档/当前候选交付人工审查执行顺序总表.md`：确认人工审查不可跳步的执行顺序。
5. `项目文档/当前候选交付人工审查链完整性索引.md`：确认人工审查链材料、命令和上下游没有断链。
6. `项目文档/当前候选交付人工审查演练Dry-run记录模板.md`：在正式会议前模拟命令、断点和退回项。
7. `项目文档/阶段30人工Reviewer交接包.md`：确认 reviewer 角色和各自负责的 evidence slots。
8. `项目文档/当前候选交付Reviewer证据槽位工作台.md`：按 reviewer role 分配 10 个 evidence slots。
9. `项目文档/当前候选交付Reviewer输出接收门禁.md`：确认 reviewer 输出被接收前的拒收条件。
10. `项目文档/当前候选交付Reviewer输出离线汇总模板索引.md`：确认接收通过后的离线汇总字段和分流规则。
11. `项目文档/阶段30人工证据闭环状态审计.md`：确认当前没有 live submission，10 个槽位仍缺失。
12. `项目文档/阶段30人工审查会议包.md`：作为会议议程和记录模板。
13. `项目文档/阶段30会后证据汇总包.md`：仅在真实 reviewer 输出产生后使用。
14. `项目文档/阶段30LiveSubmission转换预检包.md`：仅在维护者显式授权转换前使用。
15. `项目文档/当前候选交付维护者转换授权前置模板.md`：确认维护者显式转换授权的前置字段和否决条件。
16. `项目文档/当前候选交付负向用例防误用检查.md`：确认模板、dry-run、会议记录、检查命令和决策包不能被误用为证据、授权或审批。
17. `项目文档/当前候选交付人工审查包最终锁定索引.md`：交付 reviewer 前锁定材料范围、必跑命令和红线状态。
18. `项目文档/当前候选交付Reviewer实际交付执行清单.md`：交付 reviewer 时逐项勾选材料、签收、退回和记录交付状态。
19. `项目文档/当前候选交付Reviewer交付签收与退回回执模板.md`：记录材料包签收、退回、缺失材料、范围不清和补发动作。
20. `项目文档/当前候选交付Reviewer审查启动确认模板.md`：材料签收后确认 reviewer 已开始审查且理解槽位、红线和输出字段。
21. `项目文档/当前候选交付Reviewer审查中阻塞与澄清问题台账.md`：审查启动后、正式 reviewer 输出前记录阻塞、澄清、补发和恢复条件。
22. `项目文档/当前候选交付Reviewer输出退回与补交闭环记录.md`：记录 reviewer 输出或材料被退回后的补交、重新提交和再次收件条件。
23. `项目文档/当前候选交付Reviewer输出接收尝试记录.md`：记录每次 Reviewer 输出接收尝试、来源补交包、门禁结果引用和拒收原因。
24. `项目文档/当前候选交付Reviewer输出字段缺口定位矩阵.md`：接收尝试失败后定位 evidence slot、字段、缺口类型、阻塞等级和修复责任。
25. `项目文档/当前候选交付Reviewer输出字段修复责任分派单.md`：把字段缺口转成修复 owner、期限、修复路线、退回闭环和重试条件。
26. `项目文档/当前候选交付Reviewer输出字段修复回执与重试准备记录.md`：记录修复回执、补交包引用、回执完整性、退回闭环和下一次接收尝试准备条件。
27. `项目文档/当前候选交付Reviewer输出重试批次登记表.md`：把具备重试条件的修复回执归组为 retry batch，记录批次来源、目标 slots、执行命令和失败回滚路线。
28. `项目文档/当前候选交付Reviewer输出重试批次与接收尝试对账记录.md`：核对 retry batch、receipt attempt 和 receipt gate reference 的对应关系。
29. `项目文档/当前候选交付Reviewer输出对账失败回流处置单.md`：把对账失败状态回流到登记、尝试、门禁、字段缺口、修复分派或退回补交闭环。
30. `项目文档/当前候选交付Reviewer输出对账失败回流结果复核记录.md`：复核回流处置是否有 owner 确认、目标材料修正引用和重新对账条件。
31. `项目文档/当前候选交付Reviewer输出对账失败回流结果再对账准备记录.md`：整理重新对账或下一轮 retry batch 前的修正引用、recheck 命令和目标引用。
32. `项目文档/当前候选交付Reviewer输出再对账执行记录.md`：登记再对账执行的 source readiness、rerun command、target refs、execution status 和 result route。
33. `项目文档/当前候选交付Reviewer输出再对账结果接收记录.md`：接收再对账执行后的 result signal、matched/gate/field gap/blocker state 和下一步分流。
34. `项目文档/当前候选交付Reviewer输出再对账结果复核记录.md`：复核再对账结果接收后的 decision、route、correction target 和下一步引用。

## 三、演示路径

本地演示建议使用以下顺序：

1. 启动项目：

```powershell
npm.cmd start
```

2. 打开：

```text
http://localhost:3000
```

3. 普通用户路径：

- 首页普通入口：写一段记忆、回看展品墙、问 AI 讲解员。
- 添加记忆表单：确认保存状态条、展品信息折叠区、照片和附件线索折叠区。
- 展品详情：确认摘要、元信息网格、原始记忆、媒体线索和整理历史可读。
- 外部资料导入：确认先生成导入预览，不静默写入。
- 数据与同步：确认 JSON 备份先生成同步预览，写入 SQLite 与同步数据库刷新读取保持分离。

4. 维护者路径：

- 展开 `维护者入口`。
- 查看 `治理红线`。
- 查看 `当前候选交付` 总览。
- 确认 release、Phase 30 entry、runtime、third-party、human evidence 仍然阻断或禁用。

## 五、证据槽位缺口

当前仍缺失 10 个 evidence slots：

- `release-blocker-disposition`
- `transition-redline-disposition`
- `signoff-evidence-reconciliation`
- `release-runtime-separation`
- `runtime-owner-go-no-go`
- `rollback-strategy-review`
- `sandbox-acceptance-review`
- `secret-boundary-review`
- `private-memory-boundary-review`
- `audit-dry-run-review`

每个槽位仍需真实 reviewer 提供：

- `reviewer`
- `reviewedAt`
- `evidenceRef`
- `disposition`
- `decisionReason`
- `residualRisk`

如 disposition 为 `accepted-with-risk`、`deferred-with-owner` 或 `blocked`，还需要 `followUpOwner`、`recoveryCondition` 和 `targetReviewDate`。

## 六、禁止转换条件

出现以下任一情况时，不得创建 `data/phase30-human-evidence-submission.json`：

- 没有真实 reviewer 输出。
- 没有维护者显式转换授权。
- reviewer 输出尚未通过 `当前候选交付Reviewer输出接收门禁.md`。
- reviewer 输出尚未按 `当前候选交付Reviewer输出离线汇总模板索引.md` 完成离线汇总分流。
- 人工审查未按 `当前候选交付人工审查执行顺序总表.md` 完成不可跳步检查。
- 人工审查链未按 `当前候选交付人工审查链完整性索引.md` 完成断链检查。
- 人工审查演练 dry-run 发现断点且未处理。
- 维护者转换授权前置条件未按 `当前候选交付维护者转换授权前置模板.md` 确认。
- 未按 `当前候选交付负向用例防误用检查.md` 排除模板、dry-run、会议记录、检查命令、转换预检、准备信封或决策包的误用。
- 未按 `当前候选交付人工审查包最终锁定索引.md` 锁定审查材料和必跑命令。
- 未按 `当前候选交付Reviewer实际交付执行清单.md` 明确交付对象、材料、签收或退回状态。
- 未按 `当前候选交付Reviewer交付签收与退回回执模板.md` 区分材料签收、材料退回和 Reviewer 输出接收门禁。
- 未按 `当前候选交付Reviewer审查启动确认模板.md` 确认 reviewer 已理解范围、槽位、红线和输出字段。
- 未按 `当前候选交付Reviewer审查中阻塞与澄清问题台账.md` 收口审查中阻塞、澄清或补发事项。
- 未按 `当前候选交付Reviewer输出退回与补交闭环记录.md` 收口退回、补交、重新提交和再次收件条件。
- 未按 `当前候选交付Reviewer输出接收尝试记录.md` 记录每次接收尝试和门禁结果引用。
- 未按 `当前候选交付Reviewer输出字段缺口定位矩阵.md` 定位字段级缺口、阻塞等级和修复责任。
- 未按 `当前候选交付Reviewer输出字段修复责任分派单.md` 明确修复 owner、期限、路线和重试条件。
- 未按 `当前候选交付Reviewer输出字段修复回执与重试准备记录.md` 记录修复回执、补交包引用和接收重试准备。
- 未按 `当前候选交付Reviewer输出重试批次登记表.md` 归组 retry batch、接收尝试命令和失败回滚路线。
- 未按 `当前候选交付Reviewer输出重试批次与接收尝试对账记录.md` 对齐 retry batch、receipt attempt 和 gate ref。
- 未按 `当前候选交付Reviewer输出对账失败回流处置单.md` 处理 unmatched、orphan、missing gate ref 或 blocked 对账失败状态。
- 未按 `当前候选交付Reviewer输出对账失败回流结果复核记录.md` 复核处置结果、修正引用和下一次 recheck 条件。
- 未按 `当前候选交付Reviewer输出对账失败回流结果再对账准备记录.md` 确认重新对账或下一次 retry batch 输入齐备。
- 未按 `当前候选交付Reviewer输出再对账执行记录.md` 登记再对账执行命令、目标引用和结果分流。
- 未按 `当前候选交付Reviewer输出再对账结果接收记录.md` 接收再对账结果入口和下一步分流。
- 未按 `当前候选交付Reviewer输出再对账结果复核记录.md` 复核再对账结果路线和下一步处置引用。
- 任一 evidence slot 缺少 required fields。
- 任一 evidence slot 存在冲突且未完成人工冲突审查。
- 任一 accepted-with-risk / deferred-with-owner / blocked 缺少 follow-up 字段。
- 仅有模板、会议包、会后汇总草稿或 dry-run 输出。
- 仅有 `npm.cmd run check` 通过。

## 七、审查会输出要求

审查会后只能产出人工记录或会后汇总，不得自动放行。维护者需要在真实 reviewer 输出齐备后，按以下顺序处理：

1. 汇总 reviewer 输出。
2. 运行 `npm.cmd run phase30:post-review-evidence-summary`。
3. 处理冲突和风险后续项。
4. 运行 `npm.cmd run phase30:live-submission-conversion-preflight`。
5. 维护者显式授权后，才可创建 live submission。
6. 创建后再运行 `npm.cmd run phase30:evidence-submission-gate`。

本索引完成时，只表示审查前材料已打包；不表示任何 evidence slot 已通过人工审查。

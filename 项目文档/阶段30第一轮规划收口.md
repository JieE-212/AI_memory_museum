## 6. 交接包使用方式

人工复核时建议按以下顺序处理：

1. 对照 `阶段30运行时GoNoGo看板.md` 确认默认 `no-go-blocked` 是否仍成立。
2. 对照 `阶段30入口基线.md` 处理 blocker、redline、signoff evidence 和 release/runtime separation。
3. 对照 `阶段30运行时沙箱验收框架.md` 与 `阶段30密钥边界计划.md` 审核沙箱和密钥边界是否需要补充测试。
4. 对照 `阶段30审计持久化演练.md` 审核审计导出、关联 ID 和回滚线索是否足以进入真实实现评审。
5. 只有所有 pending 项有人工证据后，才允许另起 Phase 30 implementation entry 设计。

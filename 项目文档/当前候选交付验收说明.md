# 当前候选交付验收说明

更新日期：2026-07-07

本文用于验收当前候选交付是否可以进入人工复核和本地体验。它不是 release approval，不批准 Phase 29 exit，不进入 Phase 30 entry，不启用 runtime execution，也不允许 third-party execution。

## 1. 验收目标

当前候选交付的目标是确认：

- 普通用户可以本地打开应用，记录、回看、导入和讲解记忆；
- 首次运行时，SQLite 空库和浏览器示例展品的边界清楚；
- 维护者可以在首页看到候选交付总览和治理红线；
- `npm.cmd run check` 仍然通过；
- Phase 30 人工证据仍然是缺口状态，不会被脚本或页面自动补齐。

当前候选结论应保持为：

```text
rc-reviewable-but-not-releasable
releaseReady=false
phase29ExitReady=false
phase30EntryReady=false
runtimeExecution=false
thirdPartyExecution=false
```

## 2. 启动验收

在 `项目工程` 目录运行：

```powershell
npm.cmd start
```

打开：

```text
http://localhost:3000
```

如果 `3000` 端口被占用，可以临时指定其他端口：

```powershell
$env:PORT = "3001"
npm.cmd start
```

启动后可检查：

- `GET /api/health` 返回 `phase=29`；
- `GET /api/version` 返回 `version=1.9.48`；
- SQLite 数据库路径仍为 `data/memory-museum.sqlite`。

## 3. 首次运行和 SQLite 空库验收

当 SQLite 已连接但数据库里没有真实展品时，首页展品墙可能显示浏览器示例展品。此时页面必须明确显示：

- 当前展示的是浏览器示例或浏览器本地备份；
- SQLite 仍然为空或需要写入；
- 操作按钮文案为 `写入 SQLite`，不是旧文案 `迁移本地`。

验收要点：

- `最近展品墙` 下方出现同步提示条；
- 提示条说明示例只有点击 `写入 SQLite` 后才会成为数据库展品；
- 不应把浏览器示例误读为已经存在于 SQLite 的真实展品；
- 保存真实记忆、导入真实数据或完成写入后，示例状态应被清除。

## 4. 首页普通路径验收

首页默认应优先展示普通用户路径：

- `写一段记忆`
- `回看展品墙`
- `问 AI 讲解员`
- `批量导入旧资料`
- `时间线和回忆报告`
- `数据与同步`

维护、治理和插件信息应默认收在 `维护者入口` 中，不应压住普通记录体验。

## 5. 维护者入口验收

展开 `维护者入口` 后，应看到两个只读状态区：

- `治理红线`
- `当前候选交付` 总览

`治理红线` 应显示：

- 发布审批：阻断；
- Phase 30 入口：阻断；
- 运行时：禁用；
- 第三方执行：禁用；
- 人工证据：`0/10` 或等价缺口状态。

`当前候选交付` 总览应显示：

- 普通体验可本地预览；
- SQLite 初始化状态需要按当前数据库情况确认；
- 治理状态不可发布；
- 人工证据没有完成；
- 推荐验收命令包括 `npm.cmd run check` 和 `npm.cmd run phase30:evidence-closure-status`。

该总览只读展示当前候选状态，不得改变任何 gate。

## 6. 命令验收

必须通过：

```powershell
npm.cmd run check
```

建议单独运行：

```powershell
npm.cmd run phase30:evidence-closure-status
```

该命令的期望状态：

- `liveSubmissionExists=false`
- `noAutomaticApproval=true`
- `approvedSlots=0`
- `missingSlots=10`
- `readyForReleaseApproval=false`
- `readyForPhase29Exit=false`
- `readyForPhase30Entry=false`
- `readyForRuntimeExecution=false`
- `readyForThirdPartyExecution=false`

如果检查通过，只能说明候选交付可复核；不能说明发布、入口、运行时或第三方执行已经获批。

## 7. 禁止事项

在没有真实 reviewer 输出和维护者显式转换授权前，不得：

- 不得创建 `data/phase30-human-evidence-submission.json`；
- 把 `npm.cmd run check` 通过解释为人工批准；
- 把 dry-run 通过解释为 runtime go；
- 修改 `releaseReady=false`；
- 修改 `phase29ExitReady=false`；
- 修改 `phase30EntryReady=false`；
- 不得启用 `runtimeExecution`；
- 不得启用 `thirdPartyExecution`；
- 执行真实第三方插件代码；
- 制作正式 release commit。

## 8. 交接清单

交接给 reviewer 或 maintainer 时，应同时说明：

- 当前版本是 `1.9.48 / phase29-release-exit-final-archive-manifest-preview`；
- 当前仍处于 Phase 29 frozen review state；
- 当前候选交付可本地体验、可人工复核、不可发布；
- Phase 30 人工证据槽位仍未完成；
- live submission 默认不存在；
- release approval、Phase 29 exit、Phase 30 entry、runtime go/no-go 和 third-party execution 是独立 gate，不会互相自动级联。

本文档只用于验收说明，不是 Phase 30 live submission，不是 release approval。

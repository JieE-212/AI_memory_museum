# 阶段30发布审批决策包

更新日期：2026-07-06

Baseline: `2.0.22 / phase30-release-approval-decision-packet`

Phase 30 release approval decision packet: active

本决策包用于定义 release owner 的独立发布审批记录格式。它只是 release-approval-decision-packet-only package，不是 live submission，不自动批准发布，不自动退出 Phase 29，不进入 Phase 30 entry，不启用 runtime execution，也不允许 third-party execution。

- `releaseApprovalStatus=blocked-until-release-owner-decision`
- `releaseApprovalDecision=not-recorded`
- `releaseDecisionReadiness=not-ready`
- `entryReadinessRedline=active`
- `gateCascadeAllowed=false`
- `liveSubmission=data/phase30-human-evidence-submission.json`
- `liveSubmissionStatus=not-created`
- `releaseReady=false`
- `phase29ExitReady=false`
- `phase30EntryReady=false`
- `runtimeExecution=false`
- `thirdPartyExecution=false`
- `persistedMutations=0`

This package is not release approval.

This package is not Phase 29 exit.

This package is not Phase 30 entry.

## 2. 允许决策结论

发布审批决策包只允许 release owner 记录以下结论：

- `approved`
- `rejected`
- `blocked`
- `hold`

当前默认结论必须保持 `blocked`，因为 live submission 不存在，10 个 evidence slots 仍为 pending，且 release owner decision 尚未记录。

## 3. 决策输入

release owner 只能在以下输入都可审阅后记录发布审批：

- `阶段30发布决策准备信封.md`
- `阶段30入口准备红线.md`
- `阶段30人工证据提交门禁.md`
- `阶段30人工证据冲突审查包.md`
- `阶段30风险处置后续台账.md`
- `阶段30会后证据汇总包.md`
- `data/phase30-human-evidence-submission.json`
- 外部真实 reviewer 证据和 release owner 审阅记录

如果 `data/phase30-human-evidence-submission.json` 不存在，本包只能保持 `releaseApprovalStatus=blocked-until-release-owner-decision`。

## 4. 必需字段

未来 release owner 决策记录必须包含以下字段：

- `releaseApprovalDecision`
- `releaseApprovalOwner`
- `releaseApprovalReviewedAt`
- `releaseApprovalEvidenceRef`
- `releaseApprovalDecisionReason`
- `releaseApprovalResidualRisk`
- `releaseApprovalRollbackCondition`
- `releaseApprovalScope`
- `releaseApprovalExclusions`
- `releaseApprovalFollowupOwner`

缺少任一字段时，不允许把决策解释为 release approval。

## 6. 与入口准备红线的关系

本决策包必须继承 `阶段30入口准备红线.md` 的边界：

- `gateCascadeAllowed=false`
- `release-approval-to-phase29-exit` remains blocked
- release approval 不自动触发 Phase 29 exit
- release approval 不自动触发 Phase 30 entry
- release approval 不自动触发 runtime go/no-go
- release approval 不自动触发 third-party execution

即使未来 `releaseApprovalDecision=approved`，也只能表示 release owner 对发布作出独立批准记录；它不能设置 `phase29ExitReady=true`，不能设置 `phase30EntryReady=true`，不能打开 runtime。

## 7. 禁止事项

- 不自动创建 `data/phase30-human-evidence-submission.json`。
- 不伪造 release owner 决策。
- 不把 `ready-for-human-release-review` 当作 `approved`。
- 不把格式校验通过当作 `approved`。
- 不把冲突解决当作 `approved`。
- 不把风险 followup 完成当作 `approved`。
- 不设置 `releaseReady=true`。
- 不设置 `phase29ExitReady=true`。
- 不设置 `phase30EntryReady=true`。
- 不启用 `runtimeExecution`。
- 不启用 `thirdPartyExecution`。

## 8. 完成判定

本决策包完成时，只表示项目具备 release owner 独立发布审批记录的结构、字段和反级联边界。它不代表 release owner 已经批准发布，也不代表 Phase 29 exit、Phase 30 entry、runtime go/no-go 或 third-party execution 已经通过。

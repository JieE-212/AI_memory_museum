# 时屿（TIME ISLE）｜简历项目描述

## 推荐版

**时屿（TIME ISLE）｜AI 私人记忆策展工具｜Node.js · SQLite · Vanilla JavaScript · Vercel**

- 独立完成本地优先的 AI 记忆整理应用，将日记、聊天片段等原始叙述转换为可编辑的结构化展品，支持展厅、人物、地点、标签、情绪和重要度管理。
- 设计档案、策展、编辑三步 Agent 流程，持久化 run、step 与事件记录；AI 输出只作为草稿，用户确认后才写入 SQLite，保证原文和生成说明可区分、整理依据可回看。
- 实现可解释的混合检索与 AI 讲解员，对标题、正文、人物、地点、标签和情绪进行加权匹配，返回命中原因、置信提示和展品引用，减少无依据生成。
- 提供时间线、主题聚合、回顾摘要、完整/脱敏 JSON 导出与恢复；无 AI Key 时自动切换到本地 Mock，保证离线可用。
- 部署 Vercel 面试 Demo，使用临时 SQLite 和四件示例展品，禁用删除与清空操作，避免暴露私人数据和真实密钥。

Demo: https://ai-memory-museum-demo.vercel.app
GitHub: https://github.com/JieE-212/AI_memory_museum

## 一行版

独立开发 AI 记忆整理应用，以 Node.js + SQLite 保存原始记忆、结构化展品和可追踪 Agent 流程，实现带命中依据与展品引用的混合检索/讲解、时间线与主题回顾，并通过临时数据和删除保护部署安全的 Vercel 面试 Demo。

## 面试口述版

这个项目解决的不是“让 AI 改写一段文字”，而是私人记忆如何长期保存、重新找到，以及怎样判断 AI 的整理和回答有没有依据。

用户先写原始记忆，系统通过档案、策展和编辑三个 Agent 步骤生成可修改草稿，确认后才保存到 SQLite。原文与展品说明分开保留，Agent run 也会持久化。之后可以按正文、人物、地点、标签和情绪做混合检索；讲解员的回答会引用具体展品，证据不足时明确说明。

为了让面试官可以直接体验，我还做了独立 Demo 模式：使用示例数据和临时 SQLite，关闭删除类接口，并在没有真实 AI Key 时使用 Mock fallback。这样项目可以公开访问，但不会泄露私人记忆或产生模型费用。

## 可主动说明的工程取舍

- 选择 Node 原生 HTTP 和 Vanilla JS：项目规模不需要框架，依赖更少，面试时可以直接讲清数据流。
- 选择 SQLite：适合个人、本地优先场景；公开 Demo 的 `/tmp` 只用于体验，不伪装成生产持久化方案。
- 当前“语义检索”是词汇扩展和字段加权，不声称使用向量数据库。
- 多用户认证、云端长期存储和文件上传留作真实需求出现后的扩展，而不是继续堆在默认产品里。

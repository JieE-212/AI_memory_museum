# 时屿（TIME ISLE）｜简历项目描述

## 推荐版

**时屿（TIME ISLE）｜本地优先的 AI 私人记忆策展与加密分享工具｜Node.js · SQLite · Vanilla JavaScript · Web Crypto**

- 独立开发本地优先的记忆整理应用，将日记、聊天片段与照片组织为可编辑的结构化展品；一次模型调用（无 Key 时回退为本地规则）被拆解为可回看的档案提取、策展标注、草稿生成三阶段，草稿仅在用户确认后写入馆藏。
- 使用 Node 标准库实现 JPEG / PNG / WebP 魔数、容器结构和尺寸校验，拒绝伪 MIME、SVG / GIF、截断文件及超限像素图；原图流式进入暂存区，浏览器 Canvas 生成 1600px 展示图和 480px 缩略图 WebP，避免服务端图片处理依赖。
- 以 SHA-256 建立内容寻址媒体资产和精确去重：保留原图时锚定原图，仅保存安全派生图时锚定规范展示图；图片、派生版本、展品关联与观察记录在 SQLite 中分层建模，并通过 staging、quarantine 和引用保护完成失败清理与垃圾回收。
- 实现规范化图片区域证据：圈选坐标使用 `canonical-preview-v1`，证据锚定当前隐私模式下的内容 SHA-256，并区分“来源可校验”与“用户已确认”，使照片中的人物、物件、文字等线索可回到确定图片和区域复核。
- 将 dHash 仅用于用户显式触发的“可能相似照片”候选检索，非 SHA 相同绝不视为同一文件，也不提供自动合并或删除；EXIF 拍摄时间只作为待确认线索，无时区时间保持 floating，GPS 默认按敏感信息处理且不自动写入地点。
- 构建隐私优先的照片文字摘录：仅在浏览器支持时调用本机 `TextDetector` 识别用户圈选区域，结果先进入可编辑草稿，必须由用户核对确认后才保存为区域证据；不支持时诚实回退为手动摘录，不上传第三方，也不声称具备普遍 OCR 能力。
- 实现可访问的双图“时光对照”与两点手动相似变换，支持透明度、图片点选、0–1 键盘坐标、撤销和重置；它只帮助用户目视叠影，不包装成自动视觉识别、地点判断或自动对齐。
- 使用 Node 标准库实现无第三方归档依赖的 `.time-isle` gzip / tar 全量备份，严格拒绝目录穿越、重复项、大小写碰撞、软硬链接和展开炸弹；恢复前全量校验，展品、媒体、证据、展览、语音与胶囊在单次 SQLite 事务中写入，失败时清理已移动文件。
- 在不增加顶层导航的前提下，将主题展览、记忆回访、可解释语义线索、人物档案、语音记忆和时间胶囊收进既有四项任务流；新能力默认折叠或懒加载，移动端仍保持固定底部导航与单列阅读节奏。
- 设计只来源于已确认展览的时间胶囊，将外壳、内容快照和 display WebP 关联分表保存；未到期内容接口固定返回 `423` 与公开外壳，服务路径不读取正文载荷，日期只承担仪式门槛。
- 实现浏览器端加密离线分享：用户显式选择 display WebP 与已确认文字稿，素材读取完成后不再联网；Web Crypto 以 PBKDF2-SHA-256（310,000 次）+ AES-256-GCM 生成自包含 HTML，口令不上送、不持久化、不写入文件，导出结构物理排除原图、EXIF/GPS、草稿、Agent 日志、内部 ID、未选内容与原始音频。
- 建立真实 HTTP smoke 与具名模块回归，覆盖迁移、媒体与语音、主题展览、回访、线索、胶囊未到期零泄漏、浏览器加密信封、断网阅读、完整归档原子恢复及 Demo 强制 Mock / 禁写边界。

## 一行版

独立开发本地优先的 AI 私人记忆策展工具，以 Node.js + SQLite 保存可追踪整理流程、图文声音证据和已确认主题展览；实现外壳/载荷分离的时间胶囊，以及 PBKDF2-SHA-256 + AES-256-GCM 浏览器端加密、自包含且断网可读的最小化分享文件，并以真实 HTTP smoke、模块回归和可原子恢复的 `.time-isle` 归档验证核心闭环。

## 面试口述版

这个项目解决的不是“让 AI 改写一段文字”，而是私人记忆如何长期保存、重新找到，以及同一往事的文字与照片怎样留下可复核的来源。

文字进入系统后，一次模型调用会生成结构化草稿；没有 Key 时使用本地规则回退。为了让处理过程可以回看，我把结果组织成档案提取、策展标注和草稿生成三个阶段并保存处理快照，但不把它描述成三个独立 Agent。分析轨迹可以先保存，结构化展品必须由用户确认后才进入馆藏。

V4 增加了完整图片链路。服务端不依赖 Sharp 或 ImageMagick，而是自己校验 JPEG、PNG 和 WebP 的魔数、结构、尺寸和像素上限；浏览器把原图生成静态展示图和缩略图。SHA-256 是精确身份与去重依据：保留原图时使用原图，选择仅安全展示图时则使用规范展示图。

照片不是只作为装饰。用户可以圈出画面区域并保存人物、物件或文字线索，坐标锚定规范展示空间，证据同时保存该媒体资产的内容 SHA-256。保留原图时它锚定原图，仅安全图模式则锚定规范展示图；系统明确区分“哈希和坐标可校验”与“这条说明由用户确认”，前者不能替代后者，更不能验证记忆真假。

相似检索使用 dHash、画面比例和颜色等本地特征，但它只返回“可能相似、需人工核对”的候选。只有 SHA-256 相同才算同一内容，系统不会自动合并或删除照片。EXIF 时间也只是线索；缺失时区时不会擅自换算，GPS 默认敏感，不会自动反查或写入展品地点。

OCR 同样设置了确认门禁：只有浏览器本机 `TextDetector` 可用时，才对用户圈选区域生成可编辑草稿；用户逐字核对后才能保存。不支持时就回退为手动摘录。我不会把单一浏览器实验能力描述成通用中文 OCR，也没有把图片发送给外部 AI 服务。

“时光对照”把两段记忆的照片并排展示，用户可以用两组对应点手动计算缩放、旋转和平移并调节透明度。界面有完整键盘坐标后备，但它仍是人工叠影工具，不声称自动识别同一地点或同一事件。

为了让文字、照片和声音能够一起迁移，我设计了 `.time-isle` 全量归档。解包会在任何业务写入前检查路径穿越、链接、重复项、碰撞和体积上限；恢复使用单次数据库事务，失败时清理已经落盘的媒体目录。真实 HTTP smoke 与具名模块断言覆盖从上传、关联、证据、主题展览和语音，到胶囊未到期零泄漏、损坏归档零写入和公开 Demo 强制 Mock / 禁写的完整路径。

V5 到 V6.1 没有继续堆导航，而是把主题展览、单件记忆回访、可解释语义线索、人物档案和语音记忆放进原有四项任务流。所有模型或规则结果仍先进入可核对状态，展览只有用户确认后才成为时间胶囊与分享的来源。

V7 把“留给未来”和“交给别人”拆成两套边界。时间胶囊将外壳、内容快照和图片关联分表保存，未到期接口只返回外壳且不读取载荷；日期只负责仪式感。真正的分享保密发生在浏览器：先读取用户显式选择的 display WebP 和已确认文字稿，再用 PBKDF2-SHA-256 + AES-256-GCM 生成断网可读的自包含 HTML。口令从不上传或写入文件，原图、EXIF/GPS、草稿、内部 ID、Agent 日志、未选内容和原始音频则从分享结构中物理缺席。

公开 Demo 使用临时 SQLite，不部署开发者的私人馆藏、图片或真实模型密钥。预置展品、媒体写入、导入、完整归档恢复、删除和清空均受保护；访客不应向共享 Demo 提交私人内容。

## 可主动说明的工程取舍

- 选择 Node 原生 HTTP、SQLite 和 Vanilla JS：项目规模不需要前端框架，依赖更少，媒体状态、事务和异步清理路径可以直接讲清。
- “零依赖”特指图片格式校验、归档打包与解包没有引入第三方图片或 tar 库，不代表整个软件生态没有运行时依赖。
- 浏览器 Canvas 只负责生成静态 WebP 派生图；服务端仍重新验证真实容器和尺寸，不能信任前端声明的 MIME。
- SHA-256 用于精确内容身份；dHash 只用于召回待复核候选，绝不作为事实判断或自动去重依据。
- EXIF、OCR、相似图片和手动叠影都是辅助线索：没有自动视觉识别、云端图片分析、云同步或 GPS 反向地理编码。
- 本机 OCR 依赖浏览器 `TextDetector`，兼容性不足时提供手动摘录，不声称覆盖所有浏览器、语言、字体和复杂场景。
- `.time-isle` 恢复在写入前完成全量归档校验，并将业务数据放在单次 SQLite 事务中；这不等同于跨设备分布式事务。
- 公开 Demo 的 `/tmp` 仅用于体验，不是生产持久化或用户会话隔离方案；媒体写接口全部禁用。
- “混合/语义检索”仍是本地词表扩展和字段加权，不是向量数据库；记忆考古仍是确定性规则，不验证记忆真假。
- 时间胶囊日期不是可信时间锁；拥有本机数据或设备控制权的人可以绕过仪式门槛，离线分享的保密性取决于 Web Crypto、文件完整性和足够强的口令。
- 离线分享首版没有账号、云端撤回、远程失效或口令找回；口令应走另一条安全渠道，分享文件只包含用户明确选择的安全材料。

# 时屿（TIME ISLE）｜简历项目描述

## 推荐版

**时屿（TIME ISLE）｜本地优先的 AI 私人记忆策展、受限 Agent、设备内语义回忆与加密信笺｜Node.js · SQLite · Vanilla JavaScript · Transformers.js · ONNX Runtime Web · MediaRecorder · Web Crypto · PWA**

> 版本边界（2026-07-22）：简历网址、GitHub/Gitee `main`、CloudBase 与 Vercel 已发布 V17.0.0（schema 19）；功能提交为 `413f78640baad7eae6324ef14bc291f05325fbf6`，热修复为 `38d3450b9f26efdc59df4860317cdf9513e77e65`。CloudBase 部署 `002` 与 Vercel 已通过桌面/接口生产核验，V17 手机真机仍待复核。V18 仅完成 Windows 原生驱动 canary，因缺少 Linux / CloudBase 同等证据总体为 NO-GO，项目没有上线数据库或媒体静态加密。

- 独立开发本地优先的记忆整理应用，将日记、聊天片段与照片组织为可编辑的结构化展品；一次模型调用（无 Key 时回退为本地规则）被拆解为可回看的档案提取、策展标注、草稿生成三阶段，草稿仅在用户确认后写入馆藏。
- 为展品正文与结构化字段构建“记忆年轮”：规范快照以 SHA-256 父链连续追加，旧版恢复会复制为新的最新版本而不删除中间历史；编辑与恢复同时使用 `ETag / If-Match` 乐观并发控制，过期页面返回 `412`，避免静默覆盖较新的修改。
- 使用 Node 标准库实现 JPEG / PNG / WebP 魔数、容器结构和尺寸校验，拒绝伪 MIME、SVG / GIF、截断文件及超限像素图；原图流式进入暂存区，浏览器 Canvas 生成 1600px 展示图和 480px 缩略图 WebP，避免服务端图片处理依赖。
- 以 SHA-256 建立内容寻址媒体资产和精确去重：保留原图时锚定原图，仅保存安全派生图时锚定规范展示图；图片、派生版本、展品关联与观察记录在 SQLite 中分层建模，并通过 staging、quarantine 和引用保护完成失败清理与垃圾回收。
- 实现规范化图片区域证据：圈选坐标使用 `canonical-preview-v1`，证据锚定当前隐私模式下的内容 SHA-256，并区分“来源可校验”与“用户已确认”，使照片中的人物、物件、文字等线索可回到确定图片和区域复核。
- 将 dHash 仅用于用户显式触发的“可能相似照片”候选检索，非 SHA 相同绝不视为同一文件，也不提供自动合并或删除；EXIF 拍摄时间只作为待确认线索，无时区时间保持 floating，GPS 默认按敏感信息处理且不自动写入地点。
- 构建隐私优先的照片文字摘录：仅在浏览器支持时调用本机 `TextDetector` 识别用户圈选区域，结果先进入可编辑草稿，必须由用户核对确认后才保存为区域证据；不支持时诚实回退为手动摘录，不上传第三方，也不声称具备普遍 OCR 能力。
- 实现可访问的双图“时光对照”与两点手动相似变换，支持透明度、图片点选、0–1 键盘坐标、撤销和重置；它只帮助用户目视叠影，不包装成自动视觉识别、地点判断或自动对齐。
- 使用 Node 标准库实现无第三方归档依赖的 `.time-isle` gzip / tar 流式备份：服务端打包到私有临时文件后流式响应，浏览器支持时直接写入用户选择的文件；恢复采用两遍扫描与隔离暂存，严格拒绝目录穿越、链接、碰撞、截断流、超限和 future schema，验真通过后才进入数据库事务与文件补偿。
- 在“数据与项目”内提供只读馆藏体检与备份验真：核对 SQLite、迁移账本、检索索引、修订链、图片/声音文件及策展审计结构，并可在不恢复业务数据的前提下完整检查 `.time-isle`；结构损坏与人工待复核分级，报告只暴露安全计数和固定问题码。
- 实现真实一次性恢复演练：把完整 `.time-isle` 写入全新的 SQLite、图片和声音副本，恢复后独立复核 quick check、外键、FTS、业务关系及媒体 SHA-256，再关闭数据库并销毁副本；当前馆藏能力不进入 runner，清理失败会闭锁后续演练。该能力只证明当前机器/版本可恢复，不冒充异机灾备或 RTO/RPO；公开 Demo 在读取归档正文前拒绝上传。
- 在不增加顶层导航的前提下，将主题展览、受限策展助手、记忆回访、字段与线索检索、人物档案、语音记忆和时间胶囊收进既有四项任务流；新能力默认折叠或懒加载，移动端仍保持固定底部导航与单列阅读节奏。
- 新增 V17“按意思找回”：用户主动点击后才同源加载约 47 MB 的 Transformers.js 3.8.1、ORT WASM 与 `Xenova/bge-small-zh-v1.5` q8，使用真实 512 维 embedding 在 Module Worker 内建立会话索引；最多 500 件/1 MiB 快照，真实 tokenizer 核对 512-token 边界，远程模型关闭，查询、正文、索引和向量不发往第三方、不写浏览器持久化存储。
- 将“以后怎样再见”建模为用户明确确认的长期回访意愿：支持自然回访、欢迎主动出现、指定本地日期以后和暂停主动出现；确定性筛选只影响主动回访，不隐藏、删除或降低搜索可见性，也不从浏览行为推断心情、关系或重要程度。
- 为同一往事的日期冲突设计“来源校准台”：汇总当前日期、修订、原文日期锚点与照片 EXIF，用户可确认单日、时间范围、保留多种记录或继续不确定；稳定来源摘要仅用于检测线索变化，旧判断会保留并进入待复核，系统绝不回写原文或替用户裁决哪段记忆更“真实”。
- 实现事件级“一问一答口述史”：只为未解决且双侧有依据的日期冲突生成一个问题，支持单段 MediaRecorder / 本地 WebM、M4A、毫秒级人工选段、手填文字稿与显式 day/range/uncertain；回答以 draft/confirmed/superseded/withdrawn 追加保存，`ETag / If-Match + questionSetSha256 + submissionId` 处理并发与幂等，且不自动转写、识别说话人、判断情绪或改写原记忆。
- 实现受限本地策展助手：固定白名单仅含 4 项只读馆藏工具，单次最多 6 步/4 次工具/2 秒/6 件来源，只生成绑定来源哈希的 proposal/preview；保存草稿、确认候选关系和发布必须分别通过 `ETag / If-Match + Idempotency-Key + confirm` 人工决定，分享不在工具白名单，且整个执行器不调用外部大模型。
- 构建可验证记忆收件箱与来源护照：TXT/Markdown 只在浏览器临时读取，服务端重算原文件、规范文本、UTF-16 片段区间和多级 SHA-256，来源片段与新展品同事务人工入馆；主张以不可变来源快照和追加事件表达支持/补充/不同记录，来源变化只进入 `needsReview`，不自动推断人物关系、日期或真假。
- 设计“共忆信笺”离线见证链路：浏览器以 PBKDF2-SHA-256（310,000 次）+ AES-256-GCM 生成自包含邀请和请求摘要绑定的加密回信；服务端重新校验目标、问答与来源锚点，只有用户预览确认后才把 `self-asserted-unverified / unsigned` 回信保存为独立 provenance 来源，冲突时整包回滚，不自动合并原记忆或推断人物关系。
- 实现 GET-only 的 V16“多视角记忆对照”，把当前记录、记忆年轮、已确认共忆回信和来源护照并排呈现；只展示人工确认且来源仍有效的支持/补充/不同记录关系，未绑定回信、withdrawn 主张、`needsReview` 与自述未核验身份都有明确边界，零新增表、零模型、零持久化，也不自动判断谁更可信。
- 实现设备内可解释镜片：GET-only API 只接收 2–20 个明确展品 ID，并在服务端重读当前馆藏，用确定性规则生成时间、共同出现、证据和线索四种带来源投影；固定零外部模型、零工具调用、零持久化，只有 2–6 件可显式交接策展助手，7–20 件不会被静默截断。
- 构建 schema 19 锁馆写门禁与结构恢复演练：单例锁状态使用 revision CAS、校验和及派生 verifier，锁馆等待在途 mutation/维护结束后让新写请求在读取正文前返回 423；GET、导出和只读验真保持可用。结构演练仅核对完整归档 manifest、哈希与引用，明确不执行恢复、不证明容灾；锁馆也不冒充 SQLite、媒体或磁盘静态加密。
- 为“真正加密锁馆”设置 V18.0 fail-closed 可行性门禁：Windows x64 上固定 `better-sqlite3-multiple-ciphers@12.11.1` 的 WAL、FTS5、JSON、外键、事务、错钥拒绝与重开 canary 已通过，但因缺少 Linux `node:24-bookworm-slim` / CloudBase 同等证据，整体主动判定 NO-GO；实验驱动未接入正式依赖，SQLite3MC 也未被误写成官方 SQLCipher。
- 补齐隐私安全的可安装 PWA 外壳：Service Worker 只缓存离线边界页、对应样式与公开品牌图标；首页、主应用、API、图片、声音、归档和用户数据均不进入缓存，所有图片变体统一返回 `private, no-store`，断网时只展示不含私人馆藏的边界说明。
- 设计只来源于已确认展览的时间胶囊，将外壳、内容快照和 display WebP 关联分表保存；未到期内容接口固定返回 `423` 与公开外壳，服务路径不读取正文载荷，日期只承担仪式门槛。
- 实现默认拒绝的三层分享隐私编辑台：公开层使用通用外壳与固定立即开启时间，解密层允许编辑本次叙事副本并逐章、逐件、逐条选择引用、confirmed 文字稿和 display WebP，最终层生成精确内容收据；投影阶段连续匿名化并物理排除原图、EXIF/GPS、Agent 日志、内部 ID/URL/SHA、未选内容与原始音频，再以 PBKDF2-SHA-256（310,000 次）+ AES-256-GCM 生成断网可读 HTML。
- 将 schema 19 来源模块纳入 JSON / `.time-isle` 严格恢复：完整归档按收件箱、来源护照、共忆回信与 provenance 依赖顺序验真和映射，脱敏只保留固定安全计数；锁状态、salt、digest 与 verifier 物理排除。V17 已于 2026-07-22 发布，GitHub / Gitee `main` 均已包含 `38d3450` 代码基线；CloudBase 部署 `002` 与 Vercel 均完成 V17 桌面生产核验。2026-07-20 的手机 Wi-Fi / 蜂窝结果属于 V14 历史发布，V17 真机仍待复核。

## 面试口述版

这个项目解决的不是“让 AI 改写一段文字”，而是私人记忆如何长期保存、重新找到，以及同一往事的文字与照片怎样留下可复核的来源。

文字进入系统后，一次模型调用会生成结构化草稿；没有 Key 时使用本地规则回退。为了让处理过程可以回看，我把结果组织成档案提取、策展标注和草稿生成三个阶段并保存处理快照，但不把它描述成三个独立 Agent。分析轨迹可以先保存，结构化展品必须由用户确认后才进入馆藏。

V4 增加了完整图片链路。服务端不依赖 Sharp 或 ImageMagick，而是自己校验 JPEG、PNG 和 WebP 的魔数、结构、尺寸和像素上限；浏览器把原图生成静态展示图和缩略图。SHA-256 是精确身份与去重依据：保留原图时使用原图，选择仅安全展示图时则使用规范展示图。

照片不是只作为装饰。用户可以圈出画面区域并保存人物、物件或文字线索，坐标锚定规范展示空间，证据同时保存该媒体资产的内容 SHA-256。保留原图时它锚定原图，仅安全图模式则锚定规范展示图；系统明确区分“哈希和坐标可校验”与“这条说明由用户确认”，前者不能替代后者，更不能验证记忆真假。

相似检索使用 dHash、画面比例和颜色等本地特征，但它只返回“可能相似、需人工核对”的候选。只有 SHA-256 相同才算同一内容，系统不会自动合并或删除照片。EXIF 时间也只是线索；缺失时区时不会擅自换算，GPS 默认敏感，不会自动反查或写入展品地点。

OCR 同样设置了确认门禁：只有浏览器本机 `TextDetector` 可用时，才对用户圈选区域生成可编辑草稿；用户逐字核对后才能保存。不支持时就回退为手动摘录。我不会把单一浏览器实验能力描述成通用中文 OCR，也没有把图片发送给外部 AI 服务。

“时光对照”把两段记忆的照片并排展示，用户可以用两组对应点手动计算缩放、旋转和平移并调节透明度。界面有完整键盘坐标后备，但它仍是人工叠影工具，不声称自动识别同一地点或同一事件。

为了让文字、照片和声音能够一起迁移，我设计了 `.time-isle` 全量归档。解包会在任何业务写入前检查路径穿越、链接、重复项、碰撞和体积上限；恢复使用单次数据库事务，失败时清理已经落盘的媒体目录。真实 HTTP smoke 与具名模块断言覆盖从上传、关联、证据、主题展览和语音，到胶囊未到期零泄漏、损坏归档零写入和公开 Demo 强制 Mock / 禁写的完整路径。

V5 到 V6.1 没有继续堆导航，而是把主题展览、单件记忆回访、字段与线索检索、人物档案和语音记忆放进原有四项任务流。所有模型或规则结果仍先进入可核对状态，展览只有用户确认后才成为时间胶囊与分享的来源。

V7 把“留给未来”和“交给别人”拆成两套边界。时间胶囊将外壳、内容快照和图片关联分表保存，未到期接口只返回外壳且不读取载荷；日期只负责仪式感。真正的分享保密发生在浏览器：先读取用户显式选择的 display WebP 和已确认文字稿，再用 PBKDF2-SHA-256 + AES-256-GCM 生成断网可读的自包含 HTML。口令从不上传或写入文件，原图、EXIF/GPS、草稿、内部 ID、Agent 日志、未选内容和原始音频则从分享结构中物理缺席。

V7.1 让时屿可以安装到桌面或主屏幕，但没有借 PWA 缓存私人馆藏。Service Worker 使用精确白名单，只保留离线边界页、样式和公开品牌图标；首页、主应用、API、用户数据及媒体均不缓存，所有图片变体统一使用 `private, no-store`。因此断网时看到的是边界说明，而不是可能过期、残缺或泄漏隐私的“空馆藏”；这也与加密分享文件本身可断网解密阅读的能力明确区分。

V7.2 处理长期编辑中的另一类风险：误改和并发覆盖。展品正文及其结构化字段被规范化为 SHA-256 快照父链；旧版可以预览，但恢复时不会把指针倒退，而是把所选快照追加为新的最新版本。编辑和恢复都必须匹配当前 `ETag`，旧页面提交会得到 `412` 和新的版本标识，不会把较新的正文带回错误响应，也不会静默覆盖。

“数据与项目”新增的馆藏体检只读核对数据库、检索索引、年轮、图片与声音，不会自动修复或删除；备份验真复用真实恢复前的全量校验，但只在隔离暂存区展开并返回可恢复摘要。归档导出、HTTP 传输与兼容浏览器文件写入改为流式路径，中止会清理本次临时文件；启动维护也只清扫超过一小时、名称受控的归档残留，避免误删仍在使用或不属于应用的目录。

V7.3 继续把决定权留给用户。一件展品可以明确设置“欢迎主动出现、指定日期以后或暂停主动出现”；系统只按这项确认意愿调整主动回访顺序与过滤，不保存选择原因，也不从浏览次数猜测心理。恢复“自然回访”会删除这条长期意愿，馆藏、搜索和原文始终不受影响。

分享链路也从“选择候选材料”升级为三层隐私编辑台：带入编辑台不等于分享，章节、展品、引用、确认文字稿和安全展示图默认全部不选；受众、用途和叙事副本位于密文内，最终收据精确列出分享数量与下载后不可撤回的边界。V2 文件执行严格字段、计数、媒体归属和连续匿名键校验，同时保留 V1 离线文件兼容；编辑、取消、关闭和下载后都会清空口令与明文会话。

V8 处理的是“多条记录时间不一致时怎么办”。系统只在已由用户确认属于同一往事、且确有可验证日期差异时才展示默认折叠的校准台；当前日期、修订、原文日期锚点和照片 EXIF 都以独立来源卡呈现。用户可以确认一个日期、一个范围、保留多种记录或继续不确定；来源集合变化后，旧判断不会被删除或偷偷改写，只会标记为待复核。这里的哈希证明的是来源集合变了，不是证明哪段记忆为真。

V9 进一步处理“书面来源仍解决不了冲突时怎么办”。系统只为符合条件的确认事件生成一个问题；用户录音或选择本地音频后，亲手划定片段、填写文字稿并明确 day/range/uncertain。只有人工确认的 day/range 会成为事件级 `oral-history` 时间来源，uncertain 只保留证据，draft 不进入候选。新回答、重答和撤回都通过追加状态保留历史，并使引用旧来源的校准进入待复核，而不是覆盖原文。系统不自动转写、识别说话人、判断情绪、发布或分享。

V10 处理“怎样让 Agent 帮忙，但不把决定权交出去”。助手只有四项本地只读工具，不能联网、读文件、执行任意 SQL 或调用外部大模型；固定步骤、工具、时间、结果字节和来源数量预算后，只生成带来源引用的未保存展览提案。保存草稿、确认候选关系和发布是三次独立人工决定，分享仍进入既有三层隐私编辑台。运行回执可以离线重放评测；来源改变时旧提案只读保留并待复核，归档恢复后也被强制降为不可再次决定的历史记录。

V11 先解决散落文档怎样进入馆藏。TXT/Markdown 全文只在浏览器临时读取，用户逐段选择后，服务端重新校验原始字节、规范文本、UTF-16 区间与片段哈希；来源锚点和新展品在同一事务确认。来源护照再把支持、补充、不同记录三种关系做成人工主张与追加事件链，来源变化只进入待复核，不计算真假分数。

V12 进一步解决“怎样请别人补充记忆，但不做账号社交”。邀请和回信在浏览器端加密并由请求摘要绑定；馆主解密预览后，服务端再次校验目标、问答、身份边界和来源锚点，才把它保存为独立见证来源。这里我刻意保留“身份自述未验证、文件未签名”：加密能保护文件内容，不能证明是谁写的；确认入馆后的问答也仍是本机 SQLite 明文。

V13 的设备内镜片没有引入大模型。客户端只发送明确展品 ID，服务端重读当前馆藏后，以固定规则生成时间、共同出现、证据和线索四种投影，回执明确为零模型、零工具、零保存。2–6 件来源可由用户带入策展助手，更多来源则要求重选而不是静默截断。

V14 处理写入保护和恢复表述的真实性。锁馆使用 schema 19 单例、revision CAS 和派生 verifier，在等待在途写入结束后让新的 mutation 在读取正文前返回 423，同时保留导出和只读验真。它不是数据库或磁盘静态加密；结构演练也只核对 manifest、哈希和引用，没有执行恢复，更不能证明已经具备灾难恢复能力。

V15 才把“结构能验”推进到“备份真的恢复过”：完整归档被写进全新的一次性 SQLite、图片与声音目录，随后由副本独立核对数据库、关系和文件哈希，关闭并销毁后才返回成功回执。这个 runner 不拿当前馆藏能力，旧副本清理失败还会阻止读取下一份备份；但它仍只证明当前机器与当前版本，不是异机灾备证明。

V16 把当前记录、过去修改、亲友回信和来源护照放在一张只读对照里。只有用户已经确认且来源仍有效的支持、补充、不同记录关系会出现；未绑定回信、已撤回主张和待复核来源不会被包装成结论，亲友身份也始终标明自述未核验。系统不计算可信度，更不会替用户判断谁记得更准。

V17 将早期容易误解的“语义线索检索”纠偏为“字段与线索检索”，另做真正的“按意思找回”。约 47 MB 的中文模型和 WASM 运行库只有主动点击才从同源加载，真实 512 维 embedding 在 Worker 内生成和排序，最多索引 500 件、1 MiB 受控短片段；查询、正文、向量和索引不发往第三方也不落浏览器数据库。结果只是相似度，最接近也可能不相关，不能解释成事实或概率。

V18 没有为了追版本号而把旧写门禁包装成加密。Windows 固定原生 SQLite3MC canary 虽通过，但 Linux/CloudBase 尚无同等证据，因此门禁总体 NO-GO，项目没有接入实验驱动，也没有实现“真正的加密锁馆”。这部分可以用来说明我在安全功能上采用证据不足即停止的工程决策。

公开 Demo 使用临时 SQLite，不部署开发者的私人馆藏、图片或真实模型密钥。预置展品、媒体写入、导入、完整归档恢复、删除和清空均受保护；访客不应向共享 Demo 提交私人内容。

## 可主动说明的工程取舍

- 选择 Node 原生 HTTP、SQLite 和 Vanilla JS：项目规模不需要前端框架，依赖更少，媒体状态、事务和异步清理路径可以直接讲清。
- “零依赖”特指图片格式校验、归档打包与解包没有引入第三方图片或 tar 库，不代表整个软件生态没有运行时依赖。
- 浏览器 Canvas 只负责生成静态 WebP 派生图；服务端仍重新验证真实容器和尺寸，不能信任前端声明的 MIME。
- SHA-256 用于精确内容身份；dHash 只用于召回待复核候选，绝不作为事实判断或自动去重依据。
- EXIF、OCR、相似图片和手动叠影都是辅助线索：没有自动视觉识别、云端图片分析、云同步或 GPS 反向地理编码。
- 本机 OCR 依赖浏览器 `TextDetector`，兼容性不足时提供手动摘录，不声称覆盖所有浏览器、语言、字体和复杂场景。
- `.time-isle` 恢复在写入前完成全量归档校验，并将业务数据放在单次 SQLite 事务中；这不等同于跨设备分布式事务。
- V15 一次性恢复会真实写入并销毁独立 SQLite/图片/声音副本，但只验证当前机器、当前程序版本，不代表另一设备恢复、RTO/RPO 或进程隔离已经成立。
- 公开 Demo 的 `/tmp` 仅用于体验，不是生产持久化或用户会话隔离方案；媒体写接口全部禁用。
- PWA 只提供可安装外壳，不承诺离线浏览馆藏；Service Worker 的缓存白名单不包含首页、应用脚本、API、媒体或用户数据，断网边界页也不会伪装成空馆藏。
- 记忆年轮只覆盖展品正文和结构化字段，不回滚图片、声音、展览或胶囊关系；SHA-256 父链能发现链内不一致，但不是外部签名、可信时间戳或记忆真实性证明。
- 回访意愿是用户明确设置的产品偏好，不是心理画像；“欢迎”只在当前回访类型的合格候选中优先，“延期/暂停”只约束主动回访，不隐藏搜索结果或删除展品。
- 时间校准的来源键与来源集合 SHA-256 只用于识别证据集合变化；它不是真相评分、可信时间戳或记忆真实性证明，GPS 也不会进入时间来源。
- 口述史的音频片段、文字稿与时间含义都依赖用户明确确认；回答是事件级来源而非任意挂靠成员展品，普通 JSON 不携带声音字节，因此非空口述史必须用 `.time-isle` 无损迁移。
- “受限策展助手”不是后台自治 Agent 或大模型工作流：它是确定性的本地证据执行器，工具只有只读查询；哈希用于来源变化和回执一致性，不证明提案正确，人工批准也不会自动扩展到分享。
- 共忆信笺的 PBKDF2/AES-GCM 只保护馆外文件；回信身份未验证、文件未签名，确认入馆后的问答仍以普通明文结构保存在本机 SQLite。
- 设备内镜片不是 embedding、向量数据库或本地大模型；它只是按保存字段与确认来源运行的确定性投影，并且不持久化结果。
- 多视角对照只派生已有记录和人工确认关系；它不会计算可信度或自动识别矛盾，亲友回信身份仍是自述未核验、文件未签名。
- 锁馆是应用写门禁而非 SQLite/媒体/磁盘静态加密；结构演练是只读结构验真而非已执行恢复或灾难恢复证明。
- 修订账本按完整 JSON 合同设置 20 MiB 总预算；旧 JSON 导入使用独立的 64 MiB 请求上限，数据库、JSON 和 `.time-isle` 遇到高于当前程序能力的 future schema 都会拒绝继续，而不是猜测性降级。
- 馆藏体检和备份验真只报告问题，不自动修复；流式归档仍需要本机临时空间，崩溃清扫也只处理超过一小时且路径命名符合规则的残留。
- 展品库的“字段与线索检索”仍是 FTS5、词表扩展和字段加权；V17“按意思找回”才使用真实设备 embedding，但不落向量库，也不验证记忆真假。
- V17 模型与运行库自托管同源资源，约 47 MB 只在主动点击后加载；向量只在 Worker 会话内，但模型静态响应可能进入浏览器 HTTP 缓存，“释放”不等于擦除浏览器网络缓存。
- V18 Windows canary 使用 SQLite3MC 而非 Zetetic 官方 SQLCipher；缺少 Linux/CloudBase 同等验证时总体 NO-GO，正式项目依赖和数据格式都没有切换到静态加密。
- 时间胶囊日期不是可信时间锁；拥有本机数据或设备控制权的人可以绕过仪式门槛，离线分享的保密性取决于 Web Crypto、文件完整性和足够强的口令。
- 离线分享没有账号、云端撤回、远程失效或口令找回；下载后接收者仍可复制、转发或截图，口令应走另一条安全渠道。

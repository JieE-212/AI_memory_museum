# V17 浏览器发布门禁

这是一套与根项目运行依赖完全隔离的 Playwright 门禁。它会启动一份临时的 `INTERVIEW_DEMO`，依次验证：

- 1265×720、390×844、320×700 三档视口；
- 四项主导航及移动端固定底栏；
- 四个主视图没有页面级横向溢出；
- 策展助手只读取合成 sample，写入接口返回 403，运行记录始终为空；
- 设备内镜片默认折叠、不会自动勾选展品，四种镜片只以 GET 读取明确选择的合成展品并返回零模型、零工具、零保存预览；
- 锁馆与完整备份结构演练在公开 Demo 中默认折叠且不可输入，两个 POST 均在读取正文前返回 403，锁馆 revision 保持不变；
- 记忆收件箱保持合成、只读、零持久化，并在 390×844 与 320×700 两档移动视口中四边贴合屏幕；
- V15 隔离恢复在公开 Demo 中于正文前拒绝，V16 多视角保持合成只读、零模型、零保存；
- V17 同源 ONNX/WASM 真实推理、Worker 会话释放、第三方请求负例和浏览器持久化负例；
- CloudBase 独立静态唤醒入口在点击前不会请求云托管探针，点击后仅 3 次固定 PNG 探针、失败停止、手动重试与 Vercel 兜底，并在三档视口保持无横向溢出；
- 所有场景均检查页面级横向溢出和浏览器控制台错误。

首次使用时，在本目录安装独立依赖和 Chromium：

```powershell
npm.cmd install
npm.cmd run install:browser
```

之后可以在项目根目录执行：

```powershell
npm.cmd run test:browser
```

当前门禁共 24 项：8 个场景分别在桌面、390×844 与 320×700 三档执行。门禁会动态选择空闲端口，并在系统临时目录创建专用数据库、媒体目录、截图和 trace；无论通过或失败，正常退出时都会清理这些临时产物。它不会读取或修改项目的私人 `data/`，也不会被根目录的 `npm run check` 隐式触发。
